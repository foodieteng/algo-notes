完整實作 · 末位顏色 DP + 頭尾修正// Full C++

忠於 DP 思路的寫法：固定第一個 coin 的顏色，對每個首色各跑一次線性 DP， 末位再依「頭尾不一藍一綠」取捨。一次把 ans[1..maxN] 全填好。

p126.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const int MOD = 1000007;
const int MAXN = 100005;

int ans[MAXN];   // ans[N] = N 個 coin 的塗色方法數 (mod)

// 顏色：0 紅(R)  1 綠(G)  2 藍(B)。限制：G 與 B 不相鄰（含頭尾）
// dp[last] = 「第一個塗 first、目前末位塗 last」的方法數
void build() {
    for (int first = 0; first < 3; first++) {
        int r = (first == 0);          // N=1：末位 = first
        int g = (first == 1);
        int b = (first == 2);

        for (int n = 1; n < MAXN; n++) {
            // 累計這個 first 對 ans[n] 的貢獻（末位不可與 first 形成一藍一綠）
            int add = r;                                  // 末位 R 一定合法
            if (!(first == 2 && 1)) {}                  // (佔位，見下方條件)
            if (!(first == 2)) add = (add + g) % MOD;  // 末位 G：首色不能是 B
            if (!(first == 1)) add = (add + b) % MOD;  // 末位 B：首色不能是 G
            ans[n] = (ans[n] + add) % MOD;

            // 轉移到 n+1（線性：G 不接 B、B 不接 G）
            int nr = (r + g + b) % MOD;
            int ng = (r + g) % MOD;
            int nb = (r + b) % MOD;
            r = nr; g = ng; b = nb;
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    build();                       // 一次預處理所有 N

    int T;
    cin >> T;
    while (T--) {
        int n;
        cin >> n;
        cout << ans[n] << '\n';   // 查表 O(1)
    }

    return 0;
}

更短的等價寫法：答案數列滿足三項遞迴 a(n) = 3·a(n−1) − a(n−2) − a(n−3)（base 3,7,15）。預處理只要一條迴圈：

p126_recur.cpp C++17 
long long a[MAXN];
a[1] = 3; a[2] = 7; a[3] = 15;
for (int n = 4; n < MAXN; n++)
    a[n] = ((3 * a[n-1] - a[n-2] - a[n-3]) % MOD + MOD) % MOD;
// 注意減法後可能變負 ⇒ + MOD 再取餘

邊界與陷阱// Pitfalls

陷阱說明 / 解法
模數是 1000007 不是 1e9+7 題目要求對 1000007 取餘 — 別反射性寫成 1000000007。而且 1000007 不是質數（= 29 × 34483），但本題只有加減、用不到除法 / 逆元。
遞迴減法變負 3a−a−a 取餘後可能 < 0 ⇒ 寫 (x % MOD + MOD) % MOD。
多筆要預處理 T 筆各自 O(N) 會 TLE；一次 build 全部 N、查表回答。
頭尾限制別漏 只做線性 DP 會多算「頭尾一藍一綠」的情形 ⇒ N=3 會得 17 而非 15。要固定首色取捨 / 扣掉。
N = 1 / N = 2 邊界 N=1 沒有相鄰也沒頭尾配對 ⇒ 3；N=2 兩端相鄰，{G,B} 兩種被擋 ⇒ 9−2 = 7。遞迴 base 要正確。
10⁵ 筆 I/O sync_with_stdio(false); cin.tie(nullptr);，輸出用 '\n'。
04

細節走讀// Section Walkthrough

① 線性轉移（核心）

// transition
nr = r + g + b      // 末位 R：前一位什麼都行
ng = r + g          // 末位 G：前一位不能是 B
nb = r + b          // 末位 B：前一位不能是 G
// base: 第一個 coin → (r,g,b) 只有 first 那一格是 1

② 頭尾取捨（把環補回來）

// endpoint
固定 first 後，末位累進 ans[n] 時：
  末位 R → 一定合法
  末位 G → 只有 first ≠ B 才算（否則頭尾 = 一藍一綠）
  末位 B → 只有 first ≠ G 才算
∴ 三種 first 各自掃一遍、累加，就自動排除頭尾違規

③ 為什麼可以「線性總數 − 頭尾{G,B}」

// 等價觀點
線性 DP 數的是「相鄰都合法」的所有塗法 L(N)
其中違反頭尾規則的，恰好是「第1個與第N個 = 一藍一綠」
這些在線性裡是合法的（頭尾不相鄰）所以被算進去了
⇒ 答案 = L(N) − #(頭尾為 {G,B})
N=3: 17 − 2 = 15 ✓

④ 三項遞迴怎麼來的

// recurrence
答案數列 3, 7, 15, 35, 83, 199, 479, ...
設 a(n) = p·a(n-1) + q·a(n-2) + r·a(n-3)
用前幾項解出 (p,q,r) = (3, -1, -1)
驗證後續項皆成立 ⇒ a(n) = 3a(n-1) − a(n-2) − a(n-3)
