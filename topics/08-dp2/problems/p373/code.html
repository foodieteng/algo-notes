01

完整實作 · 兩候選值 DP// Full C++

每筆測資讀 N M 與 N 個數。每個數字有兩個候選值 v[i][0]=aᵢ、v[i][1]=2aᵢ（只有 ≥ M 可用）。 dp[i][c] = 以第 i 個數用候選 c 結尾的最長非遞減鏈，從前面所有值 ≤ 它的狀態接 +1。答案是所有 dp 的最大值。

p373.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) {
        int N;
        long long M;
        cin >> N >> M;

        vector<long long> a(N);
        for (int i = 0; i < N; i++) cin >> a[i];

        // v[i][0] = a[i]（維持），v[i][1] = 2*a[i]（變兩倍）
        // dp[i][c] = 以「第 i 個數、用候選值 v[i][c]」結尾的最長非遞減鏈
        vector<array<long long, 2>> v(N);
        vector<array<int, 2>>      dp(N, {0, 0});
        int ans = 0;

        for (int i = 0; i < N; i++) {
            v[i][0] = a[i];
            v[i][1] = 2 * a[i];
            for (int c = 0; c < 2; c++) {
                if (v[i][c] < M) { dp[i][c] = 0; continue; }   // 此候選值 < M，不可用
                int best = 1;                                  // 至少自己一個
                for (int j = 0; j < i; j++)
                    for (int cc = 0; cc < 2; cc++)
                        if (dp[j][cc] > 0 && v[j][cc] <= v[i][c])
                            best = max(best, dp[j][cc] + 1);
                dp[i][c] = best;
                ans = max(ans, best);
            }
        }
        cout << ans << '\n';
    }
    return 0;
}

非遞減 ⇒ 比較用 ≤：允許相鄰相等（例如範例的 [2, 2, 3]）。若寫成嚴格 < 就變成「嚴格遞增」，會把合法的相等鏈排除、答案偏小。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
非遞減用 ≤ 題目是「非嚴格遞增」⇒ 允許相等。轉移條件 v[j][cc] ≤ v[i][c]，寫成 < 會少算相等鏈、WA。
候選值要 ≥ M 只有 ≥ M 的候選值能用。若 2aᵢ < M 連兩倍都不夠 ⇒ 這個數字完全不可用，兩個 dp 都 0。
全程 long long 2aᵢ 最大 2×10⁹ > int 上限（~2.1×10⁹ 也很接近）⇒ 候選值與比較一律 long long。
答案可能是 0 若沒有任何數字的任何候選值 ≥ M（如範例2 a=[7,8,9], M=42，最大 2×9=18<42）⇒ 答案 0。
T 筆要清空 每筆測資 dp/v 重新配置或清零；用 vector 每筆新建最省事。
同一數字不能用兩次 dp[i][c] 只從 j < i 轉移，天生不會把同一個 i 的兩個候選串進同一條鏈。
04

細節走讀// Section Walkthrough

① 操作 → 候選值

// candidates
維持原樣 → 值 = a[i]
變成兩倍 → 值 = 2*a[i]
移除     → 不選（不進子序列）
⇒ 每個數字最多兩個「可用值」：{a[i], 2a[i]} ∩ [M, ∞)

② 狀態與轉移

// dp
dp[i][c] = 以「第 i 個數、用 v[i][c]」結尾的最長非遞減鏈
dp[i][c] = 1 + max{ dp[j][cc] : j<i, v[j][cc] ≤ v[i][c] }   (v[i][c] ≥ M)
答案 = max over all dp[i][c]

③ 升級到 O(N log N)：BIT 取代內層掃描

// speedup (N up to 1e5)
內層「max 前面值 ≤ v[i][c] 的 dp」= 值域上的前綴最大值查詢。
把所有候選值離散化，開一棵 BIT（Fenwick）存「該值結尾的最大鏈長」。
處理 i 時：對它的兩個候選值先各 query 一次（前綴 max +1），
          再把兩個結果各 update 回 BIT（先查後改，避免同 i 自我串接）。
每次 query/update O(log N) ⇒ 總 O(N log N)，可吃滿 N=1e5。

④ 為什麼「先查兩個、再改兩個」

// batch update
若邊查邊改，第 i 個數的小候選值 update 後，
它的大候選值 query 會把「自己」算進去 → 同一數字被串兩次（錯）。
所以兩個候選值都 query 完，才一起 update。
