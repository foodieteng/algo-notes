// 線性 DP · 最大不相鄰子集和

★ ★ ☆ ☆ ☆ 核心 一維 DP（House Robber） · 時間 O(N)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

給一個正整數陣列 a₁, a₂, …, a_N，要從中取出一些數字， 但任兩個被取的數字不能相鄰，求取出數字的總和最大值。 多筆 T ≤ 10，1 ≤ N ≤ 10⁵，1 ≤ aᵢ ≤ 1000。

經典的「打家劫舍 / House Robber」：一排房子不能搶相鄰兩間，求最大金額。 範例：[1,2,3,4] → 6（取 2+4）、[1,3,4,2] → 5（取 3+2 或 1+4）。

02

為什麼用 DP// Why DP

A 枚舉子集 O(2ᴺ)
試每種取法再檢查相鄰 ⇒ N=10⁵ 完全不可能（連 N=40 都爆）。
× REJECT · TLE
B 貪心取大 O(N log N)
每次挑最大的、刪掉它的鄰居 ⇒ 會選錯：[3,4,3,4,3] 貪心拿不到最佳。
△ REJECT · WRONG
BOTH FAIL · ADOPT
C 線性 DP O(N)
每個位置只有「取 / 不取」兩種，遞迴關係只看前一、前二格。一遍掃完。
● ADOPT · STABLE

為什麼貪心錯？ 局部最大不保證全域最佳 — 拿了一個大數會封鎖兩個鄰居，可能擋掉更好的組合。 DP 把「取 / 不取」的後果完整往後傳遞，才正確。

03

狀態與轉移// State & Transition

dp[i] = 只看 a[0..i] 時的最大不相鄰和。每個位置兩個選擇：

dp[i] = max( dp[i−1] , dp[i−2] + a[i] )
跳過 a[i] ⇒ 沿用 dp[i−1]；取 a[i] ⇒ 不能用 a[i−1]，接 dp[i−2] 再加 a[i]
  1. base · dp[0] = a[0]、dp[1] = max(a[0], a[1])。
  2. 答案 · dp[N−1]。
  3. 滾動優化 · 只用到前兩格 ⇒ 兩個變數 take / skip 即可，空間 O(1)。

05

為什麼這個轉移是對的// Correctness

// 最優子結構 + 無後效性
  1. 看最後一個被考慮的 a[i]，最佳解只有兩種可能：沒取它 或 取了它。
  2. 沒取 a[i] ⇒ 問題退化成「a[0..i−1] 的最佳解」= dp[i−1]。
  3. 取了 a[i] ⇒ a[i−1] 不能取（相鄰），剩下自由的是 a[0..i−2] ⇒ dp[i−2] + a[i]。
  4. 兩種互斥且涵蓋所有情況 ⇒ 取 max 即 dp[i]。子問題的最佳解能組出大問題的最佳解（最優子結構）。
  5. dp[i] 只依賴更小的 i（無後效性）⇒ 由小到大一次填完即可。

因為 aᵢ ≥ 1 全為正，「多取一個合法的數」永遠不會變差，所以不需要考慮「故意少取」 — 轉移取 max 就涵蓋了。

06

複雜度// Complexity

每格 O(1) × N 格 = O(N) 每筆
滾動兩變數 ⇒ 空間 O(1)（不算讀入陣列）；N = 10⁵、T ≤ 10 輕鬆過
項目數量備註
TimeO(N)單次掃描，每格常數工作
SpaceO(1)滾動 take/skip 兩變數即可
遞迴深度0（迭代）純迴圈
Constant factor低一次比較一次加法
07

一句洞察// One-Sentence Insight

每個位置只有「取 / 不取」兩條路。
取就跳過鄰居接 dp[i−2]，不取就沿用 dp[i−1]，兩者取大 —
dp[i] = max(dp[i−1], dp[i−2] + a[i])，一遍 O(N) 解決。
