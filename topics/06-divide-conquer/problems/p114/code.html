完整實作// Full C++

互動式題目：實作 solve(N, X, Y)，由 judge 提供 Report(...)。不需 main()。

p114.cpp C++17 
#include "lib0114.h"

// 把 3 格 (lx[0..2], ly[0..2]) 串成一塊 L 回報給 judge
void put(int lx[3], int ly[3]) {
    Report(lx[0], ly[0], lx[1], ly[1], lx[2], ly[2]);
}

// 鋪 n×n 子板，左上角 (x0, y0)，已被佔的格子 = (bx, by)
void tile(int x0, int y0, int n, int bx, int by) {
    // ── BASE: 2×2 扣掉 blocked 剩 3 格本身就是 L
    if (n == 2) {
        int lx[3], ly[3], cnt = 0;

        for (int dx = 0; dx < 2; dx++) {
            for (int dy = 0; dy < 2; dy++) {
                int x = x0 + dx;
                int y = y0 + dy;

                if (x == bx && y == by) continue;

                lx[cnt] = x;
                ly[cnt] = y;
                cnt++;
            }
        }

        put(lx, ly);
        return;
    }

    // ── 切 4 象限：mx, my 是上半／左半的最後一行 (列)
    int h  = n / 2;
    int mx = x0 + h - 1;
    int my = y0 + h - 1;

    // 4 象限「最靠近中心」的格子 (TL, TR, BL, BR)
    int cx[4] = {mx,     mx,     mx + 1, mx + 1};
    int cy[4] = {my,     my + 1, my,     my + 1};

    // blocked 落在哪個象限？  0=TL, 1=TR, 2=BL, 3=BR
    int bq = (bx > mx) * 2 + (by > my);

    // ── 中央 L：放在「不含 blocked 的 3 個象限」最靠近中心的角
    int lx[3], ly[3], cnt = 0;

    for (int q = 0; q < 4; q++) {
        if (q == bq) continue;

        lx[cnt] = cx[q];
        ly[cnt] = cy[q];
        cnt++;
    }

    put(lx, ly);

    // ── 對 4 個子象限遞迴；blocked 不在的象限以「剛種的角」當新 blocked
    int sx[4] = {x0, x0,     x0 + h, x0 + h};
    int sy[4] = {y0, y0 + h, y0,     y0 + h};

    for (int q = 0; q < 4; q++) {
        int nbx = (q == bq) ? bx : cx[q];
        int nby = (q == bq) ? by : cy[q];

        tile(sx[q], sy[q], h, nbx, nby);
    }
}

void solve(int N, int X, int Y) {
    tile(1, 1, N, X, Y);
}

複雜度分析// Complexity Derivation

每呼叫一次 tile() 的工作量（不含遞迴）：

步驟內容單呼叫成本
base case 判斷n == 2 時掃 4 格O(1)
算 mx / my / bq4 個位元運算 + 比較O(1)
列舉 4 角 + Report固定 4 次迴圈，1 次 ReportO(1)
遞迴 4 子板4 個 n/2 大小的子問題4 · T(N/2)
T(N) = 4 · T(N/2) + O(1)
Master Theorem case 1：a = 4, b = 2, log_b a = 2；f(N) = O(N⁰) < N² ⇒ T(N) = Θ(N²)
項目數量備註
TimeΘ(N²)總 (N² − 1)/3 個 L，每個 O(1)。
SpaceO(log N)只用遞迴堆疊；無額外陣列。
Recursion Depthlog₂ N = KN = 1024 ⇒ 10 層，stack 很淺。
Constant factor低純整數比較與算術；無 STL 容器。

下限：要放 (N² − 1)/3 = Θ(N²) 個 L，每個必須 ≥ O(1) ⇒ 演算法已達理論下限。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
1-indexed 座標 題目給的 (X, Y) 從 1 開始，最外層呼叫必須是 tile(1, 1, N, X, Y)，不是 (0, 0, ...)。
mx 是上半最後一列 mx = x0 + h − 1，不是中線。下半從 mx + 1 開始；4 個角是 {mx, mx+1} × {my, my+1}。
bq 編號用比較 bq = (bx > mx) * 2 + (by > my) — 高位是 X、低位是 Y，剛好 TL=0, TR=1, BL=2, BR=3。注意是 > mx（嚴格大於）。
base case 不需算 bq n = 2 時不切象限；雙重 for 跳過 blocked 即可。提早判斷 n == 2 是必須的，否則 h = 1、mx = x0、會繼續往下遞迴變成無限迴圈。
中央 L 必須先 Report 放完中央 L 才能讓其他 3 個象限「有 blocked」可遞迴；順序顛倒會導致子象限沒有 blocked 而行為未定義。
用 put() 包 Report put(lx, ly) 把 3 格寫成「Report(lx[0], ly[0], lx[1], ly[1], lx[2], ly[2])」— 中央 L、base case 都共用一個出口，少寫一次 Report 列舉，可讀性高。
不用 std::array 用原始 int cx[4] 等平行陣列，索引 0..3 直接對應 TL..BR — 比 struct 清楚。
04

細節走讀// Section Walkthrough

① base case (n == 2)

// n == 2
double for-loop 掃 4 格 (x0..x0+1) × (y0..y0+1)
跳過 (bx, by)，把剩 3 格塞進 lx / ly
put(lx, ly) 一次回報 ─ stack 在這停。

② 算 mx / my (子板的中心邊界)

// split
h  = n / 2
mx = x0 + h - 1   ← 上半最後一列
my = y0 + h - 1   ← 左半最後一行

4 個「最靠近中心」的角 (TL, TR, BL, BR)：
  cx = { mx,   mx,   mx+1, mx+1 }
  cy = { my,   my+1, my,   my+1 }

③ 判定 blocked 在哪個象限

// bq
bq = (bx > mx) * 2 + (by > my)

  TL = 0 :  bx ≤ mx  且  by ≤ my
  TR = 1 :  bx ≤ mx  且  by > my
  BL = 2 :  bx > mx  且  by ≤ my
  BR = 3 :  bx > mx  且  by > my

④ 放中央 L

// place center L
for q in 0..3:
    if q == bq: skip
    else: 把 (cx[q], cy[q]) 收進 lx / ly

put(lx, ly)

→ 這 3 格從此被「占用」 ─ 在各自象限的遞迴裡會當新 blocked。

⑤ 遞迴 4 個子象限

// recurse 4 quadrants
sx = { x0, x0,     x0 + h, x0 + h }   ← 子板左上 X
sy = { y0, y0 + h, y0,     y0 + h }   ← 子板左上 Y

for q in 0..3:
    if q == bq:
        nbx, nby = bx, by             ← 用原 blocked
    else:
        nbx, nby = cx[q], cy[q]       ← 用剛種的角
    tile(sx[q], sy[q], h, nbx, nby)

→ 每個子板都是「n/2 × n/2 + 1 個 blocked」 ─ 與原問題同形。
