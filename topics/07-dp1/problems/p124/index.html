円円數磁磚

// 計數 DP · 3×N 骨牌鋪法

★ ★ ★ ☆ ☆ 核心 耦合狀態 DP（profile） · 時間 O(maxN + T)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

一條寬 3 塊磁磚、長 N 塊磁磚的走廊（即 3 × N 方格）， 要用 1×2 的長方形骨牌鋪滿，問共有幾種鋪法？答案對 1000007 取餘。 多筆 T ≤ 10⁵，1 ≤ N ≤ 10⁵。

範例：N=1 → 0、N=2 → 3、N=4 → 11、N=100000 → 232803。 關鍵觀察：3×N 共 3N 格，每塊骨牌蓋 2 格 ⇒ N 為奇數時 3N 是奇數、鋪不滿 ⇒ 0。

02

為什麼用 DP// Why DP

A 枚舉所有鋪法 指數級
逐格嘗試放骨牌再回溯 ⇒ 方法數本身就指數成長，N=100000 完全不可能列出。
× REJECT · TLE
B 每筆各自 DP O(T · N)
每筆測資重跑 O(N) ⇒ T·N 最壞 10⁵×10⁵ = 10¹⁰，TLE。
△ REJECT · TLE
BOTH FAIL · ADOPT
C 耦合狀態 DP + 預處理 O(maxN + T)
用「逐欄填、邊界只有兩種型態」推遞迴，一次算好 f[1..10⁵]，每筆查表 O(1)。
● ADOPT · STABLE

核心想法：一欄一欄往右鋪，欄與欄的交界只可能是「平的」或「缺一角（凹進去）」少數幾種型態。 把這些型態當狀態，就能寫出線性遞迴。多筆 ⇒ 一次預處理所有 N。

03

狀態設計 · 兩種邊界// State Design

逐欄鋪，看「目前已鋪區域的右邊界」長什麼樣。3 寬的走廊，邊界本質上只有兩型：

  • f[n] = 把 3×n 完整鋪滿（平邊界）的方法數。
  • g[n] = 鋪滿前 n−1 欄，最後一欄只鋪了底部兩格、頂端那格還空著（缺一角／凹進去的鋸齒邊界）的方法數。
f[n] = f[n−2] + 2·g[n−1]
g[n] = f[n−1] + g[n−2]
base f[0]=1, f[1]=0, g[0]=0, g[1]=1；答案 = f[N]

（f[n]=f[n−2]+2g[n−1]：最左欄三橫填滿、或留下一個缺角邊界； 兩式聯立消去 g 後得單一遞迴 f[n]=4f[n−2]−f[n−4]。）

05

遞迴怎麼來的// Deriving the Recurrence

先把 g 講清楚，再看 f[n]=f[n−2]+2·g[n−1] 的每一項長什麼樣。

STEP 1 g[n] 是什麼？「鋪到最後一欄時，缺了一角（凹進去）」

g 之所以重要：逐欄往右鋪時，「最後一欄」要嘛三格都滿（f，平邊界）， 要嘛缺頂端或底端一格（g，凹進去），不會有別種形狀。所以只要追蹤這兩個量。

注意 · g 只算「缺一側」

g[n] 固定算「缺某一側」（例如固定缺頂端那格）。 缺頂端、缺底端是兩組不同的鋪法，但上下鏡像、數量剛好相等，所以都記成同一個數 g[n]。
—— 這就是後面 2·g[n−1] 那個 2 的來源：一種缺在頂、一種缺在底，兩組不重疊、各一份 ⇒ g + g = 2g。

STEP 2 關鍵：看「最左欄」第一塊骨牌怎麼放

不要去數 3×2 有幾種。要問：3×n 最左邊那一欄（3 格）的第一塊骨牌怎麼放？ 只有 3 種互斥的放法，每種都把問題縮小成一個更小的 f 或 g：

A 和 B 都是 g[n−1]，那不是重複嗎？
不重複。 A 留下的是「缺底角」的 g、B 留下的是「缺頂角」的 g —— 是兩組不同的鋪法， 只是上下鏡像、數量剛好相等，才都寫成 g[n−1]。兩份相加 ⇒ g[n−1] + g[n−1] = 2·g[n−1]。
而「沒有 +3」是因為我們不是在數 3×2 有幾種，是把鋪法依「最左欄第一塊怎麼放」分成 A/B/C，加總就是全部、沒有第 4 類。

套到 n = 2 驗證（g[1]=1, f[0]=1）：

// n = 2
情況 A：上直 + 下橫 → 剩 g[1]（缺底）= 1 種
情況 B：下直 + 上橫 → 剩 g[1]（缺頂）= 1 種
情況 C：三條橫      → 剩 f[0] = 1 種
合計 f[2] = 1 + 1 + 1 = 3  ✓（剛好就是那 3 種 3×2 鋪法）

看到了嗎：那「3 種」是結果（1+1+1），不是要再「乘 3 / 加 3」的係數。 係數只有 A+B 合成的 2。把 n 換大，三類各自縮成的子問題 g[n−1], g[n−1], f[n−2] 就不再都是 1，但分類方式完全一樣。

STEP 3 那 g[n] 自己呢？看「缺角正下方那格」怎麼蓋

g[n] 的盤面 = 前 n−1 欄全滿，最後一欄缺頂端一格（下圖畫最後 2 欄，左邊「…」代表前面已鋪滿）。 缺角那格暫時管不到（要靠它左邊或下面的骨牌伸過來補），所以改看缺角正下方那格怎麼蓋，只有兩種：

套到 n = 3 驗證（f[2]=3, g[1]=1）：

// g[3]（最後一欄缺頂角）
看缺角正下方那格怎麼蓋
  A 直放 → 最後一欄收乾淨 → 剩 f[2] = 3
  B 橫放 → 又冒出一個缺角   → 剩 g[1] = 1
g[3] = f[2] + g[1] = 3 + 1 = 4  ✓
// 整理 · 兩條遞迴怎麼來的
  1. f[n] 的 (a) 項：最右邊用「三條橫」把整塊 3×2 收掉，前面剩一個平邊界 ⇒ 接 f[n−2]。
  2. f[n] 的 (b) 項：最左欄擺成「直 + 橫」，留下一個缺一角的邊界；缺在頂 / 缺在底兩種鏡像 ⇒ 2·g[n−1]。
  3. 合起來 f[n] = f[n−2] + 2·g[n−1]。（n=2 驗證：1 + 2×1 = 3 ✓）
  4. g[n]（STEP 3）：看缺角正下方那格，直放剩 f[n−1]、橫放剩 g[n−2] ⇒ g[n] = f[n−1] + g[n−2]。（n=3 驗證：3 + 1 = 4 ✓）
  5. 兩式消去 g ⇒ f[n] = 4·f[n−2] − f[n−4]（實作最省事）。
f[n] = 4·f[n−2] − f[n−4]
base f[0]=1, f[2]=3；奇數 N → 0；對 1000007 取餘（減法後 +MOD）

複雜度// Complexity

預處理 O(maxN) + 每筆 O(1) = O(maxN + T)
maxN = 10⁵、T = 10⁵ ⇒ 約 2×10⁵ 步，秒過；對照每筆重算 O(T·N)=10¹⁰
項目數量備註
TimeO(maxN + T)預處理一次、查表 O(1)
SpaceO(maxN)f[] 一條陣列
遞迴深度0（迭代）純迴圈填表
Constant factor低整數加減取餘
07

一句洞察// One-Sentence Insight

逐欄鋪，交界只有「平」或「缺一角」兩種型態。
把型態當狀態 ⇒ f / g 互相遞推 ⇒ 消去後 f[n]=4f[n−2]−f[n−4]；
奇數 N 直接 0；多筆一次預處理查表。
