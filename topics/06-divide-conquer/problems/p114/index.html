Divide & Conquer Recursion Construction Interactive O(N²)

題意拆解

王老先生有一塊 N × N 的正方形土地，其中 N = 2ᴷ（K 為正整數）。 土地被切成 N × N 個邊長為 1 的小方格， 位置以 (i, j) 表示：i 為由上到下第 i 列、j 為由左至右第 j 行（皆從 1 開始）。

王老先生先挑一格 (X, Y) 作為自己的養老用地， 剩下 N² − 1 格要平分給 (N² − 1) / 3 個子孫。 每個子孫剛好分到 3 格相連，且必須排成 L 型（不能是 1 × 3 直線）。 任兩個子孫不得有重疊。

輸入 / 輸出

本題採 互動式評測。請實作以下函式（lib0114.h 已宣告）：

lib0114.h C++ 
void solve(int N, int X, int Y);   // 你要實作這個
void Report(int x1, int y1,
            int x2, int y2,
            int x3, int y3);       // 由 judge 提供，回報一個 L

solve() 內部對每個 L 呼叫一次 Report， 參數為該 L 的三個格子座標。呼叫順序與三格的順序皆不影響判定， 只要分配滿足規則即視為正確。

限制

  • 30% 資料：1 < N ≤ 8
  • 100% 資料：1 < N ≤ 1024，1 ≤ X, Y ≤ N
  • N 保證為 2 的正整數次方，存在正整數 K 使 N = 2ᴷ

範例

judge 呼叫 solve(4, 3, 4)，你的程式可以依照分治 DFS 順序回報 5 次 Report：

// JUDGE
solve(4, 3, 4);
// YOUR REPORTS
Report(2,2, 2,3, 3,2); Report(1,1, 1,2, 2,1); Report(1,3, 1,4, 2,4); Report(3,1, 4,1, 4,2); Report(3,3, 4,3, 4,4);
N = 4、特殊格 (3, 4) 共需 (16 − 1) / 3 = 5 個 L-tromino。 下一節「Concept」說明為什麼這樣的分配一定存在， 再下一節「Visualization」可以看 5 個 L 一步一步鋪上來。
