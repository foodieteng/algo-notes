01

完整實作 · 二維 0/1 背包// Full C++

二維 dp[j][l]（j = 已選個數、l = 飽足）。逐物品，內層 j 從 K 倒到 1、l 從 M 倒到 Aᵢ。 每筆測資先把 dp 歸零，答案 dp[K][M]。

p144.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const int MAXK = 105;
const int MAXM = 1005;
long long dp[MAXK][MAXM];   // dp[j][l] = 選 ≤ j 個、飽足 ≤ l 的最大滿足感

void solve() {
    int N, M, K;
    cin >> N >> M >> K;

    for (int j = 0; j <= K; j++)
        for (int l = 0; l <= M; l++) dp[j][l] = 0;

    for (int i = 0; i < N; i++) {
        int satiety, joy;
        cin >> satiety >> joy;
        for (int j = K; j >= 1; j--)              // 已選個數倒序
            for (int l = M; l >= satiety; l--)      // 飽足感倒序
                dp[j][l] = max(dp[j][l], dp[j - 1][l - satiety] + joy);
    }

    cout << dp[K][M] << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) solve();

    return 0;
}

和 P143 的差別：只是把一維 dp[l] 變成二維 dp[j][l]，內層多一層 j（從 K 倒掃）、轉移看 dp[j−1][·]。 其餘（倒序、max 轉移）完全一樣。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
兩維都要倒序 j 和 l 都由大到小。任一維寫成正序，dp[j−1][l−Aᵢ] 可能已含本物品 ⇒ 重複拿、答錯。
別漏讀 K 每筆第一行是 三個整數 N M K（原題只有 N M）。漏讀 K 會整個對不上。
答案是 dp[K][M] dp 定義為「≤ j 個、≤ l 飽足」，所以直接取上界 dp[K][M] 就是「最多 K 個、最多 M 飽足」的最佳。
每筆重置 dp 多筆測資 K、M 會變，務必每筆把 dp[0..K][0..M] 歸零，別讓上一筆殘值汙染。
型別 Bᵢ ≤ 10⁶、選 ≤ 100 個 ⇒ 總和 ≤ 10⁸。int 上限 ~2.1×10⁹ 雖夠，long long 最保險。
K = N 退化 30% 測資 K=N，等於沒件數限制 ⇒ 退化回原題（P143）。同一份二維程式照樣正確。
04

細節走讀// Section Walkthrough

① 狀態定義

// state
dp[j][l] = 選 ≤ j 個高棕櫚、飽足感 ≤ l 時的最大滿足感
初值 dp[j][l] = 0
答案 = dp[K][M]

② 轉移（吃 or 不吃第 i 個）

// transition
for i in 0..N-1:
  for j from K down to 1:           // 個數維倒序
    for l from M down to A[i]:       // 容量維倒序
      dp[j][l] = max( dp[j][l],                  // 不吃
                      dp[j-1][l-A[i]] + B[i] )   // 吃（用掉 1 名額 + A[i] 飽足）

③ 為什麼來源是 dp[j−1][l−A]

// 為何 j-1
吃下第 i 個 ⇒ 已選個數 +1、飽足 +A[i]
所以「吃之後的 dp[j][l]」要從「吃之前、少一個名額少 A[i] 飽足」轉移
  → dp[j-1][l-A[i]]，再加上這顆的 B[i]

ONE-LINE SUMMARY
二維背包：dp[j][l]=max(dp[j][l], dp[j−1][l−Aᵢ]+Bᵢ)，j、l 倒序，答案 dp[K][M]，O(N·K·M)。
和 P143 只差多一維 j；兩維都倒序；別漏讀 K；long long 保險；K=N 退化回原題。
