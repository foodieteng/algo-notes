重點回顧// Concept Recap

關鍵點內容
成本型態 折扣 ⇒ 邊際遞減 ⇒ 凹成本（規模經濟）。
核心洞察 凹 + 過原點 ⇒ 次可加 ⇒ 拆流不划算 ⇒ 全部走同一條路。
邊權公式 w = (F ≤ D) ? F·C : D·C + (F−D)·C'，整批 F 箱的花費。
演算法 壓好邊權後跑 Dijkstra（邊權非負）求 S→E 最短路。
常見錯 套 MCMF：凹成本拆邊會偷拿折扣價 ⇒ 答案偏低、錯。
數值 F·C ~5×10¹³ ⇒ 全程 long long。

觀念題// Concept Questions

Q1 建模

為什麼可以把「每箱分段收費」的方案壓成一條固定權重的邊？權重怎麼算？

Q2 凹 vs 凸

折扣讓成本變凹。為什麼凹成本可以「集中單路徑 + 最短路」，而凸成本不行？

Q3 反例

若直接把每條邊拆成「容量 D 成本 C」「容量 ∞ 成本 C'」兩條邊跑 MCMF，會算出什麼？為什麼錯？

Q4 演算法選擇

邊權都非負，為什麼用 Dijkstra 而不是 Bellman-Ford 或 SPFA？這題能用 Floyd 嗎？

Q5 資料型別

哪些變數一定要用 long long？舉一個用 int 會溢位的具體數字。

範例 Trace · 跑一次 code 看會輸出什麼// Worked Trace

用樣本第 2 組：N=4, M=4, S=1, E=4, F=2。

// INPUT4 4 1 4 2
1 2 1 1 1      (A=1 B=2 C=1 D=1 C'=1)
2 4 5 1 3      (A=2 B=4 C=5 D=1 C'=3)
1 3 1 1 1      (A=1 B=3 C=1 D=1 C'=1)
3 4 6 1 1      (A=3 B=4 C=6 D=1 C'=1)

STEP 1 · 壓邊權（F=2 箱），每條都 F=2 > D=1 ⇒ 走折扣分支 D·C + (F−D)·C'：

邊CDC'w = 1·C + 1·C'
1→21111+1 = 2
2→45135+3 = 8
1→31111+1 = 2
3→46116+1 = 7 ← 折扣猛

STEP 2 · Dijkstra 從 S=1：

// INIT
dist = [_, 0, ∞, ∞, ∞]    (索引 1..4)
pq = {(0,1)}
// POP (0, 1)
鬆弛 1→2 (w=2): dist[2] = 2  push (2,2)
鬆弛 1→3 (w=2): dist[3] = 2  push (2,3)
dist = [_, 0, 2, 2, ∞]
// POP (2, 2)
鬆弛 2→4 (w=8): dist[4] = 2+8 = 10  push (10,4)
dist = [_, 0, 2, 2, 10]
// POP (2, 3)
鬆弛 3→4 (w=7): 2+7 = 9 < 10 ⇒ dist[4] = 9  push (9,4)
dist = [_, 0, 2, 2, 9]      ← 走 1→3→4 比 1→2→4 便宜
// POP (9, 4)
u = 4 = E，dist[4] 已定 = 9
(後續 pop (10,4) 時 d=10 > dist[4]=9 ⇒ 跳過)
// OUTPUT9
觀察：兩箱全走 1→3→4 = 2 + 7 = 9，贏過 1→2→4 = 2 + 8 = 10。 勝負手是 3→4 的折扣（第 2 箱從 6 降到 1）。沒有拆箱、整批走同一條路 —— 這正是凹成本的最優型態。

應用題// Quick Calculations

App 1 手算

樣本第 1 組同圖但 F=1，請手算輸出。第 3 組 N=2, F=10⁹, 邊 C=C'=50216, D=1000 又是多少？

App 2 變形

若改成「運越多越貴」（漲價，C' ≥ C，凸成本），這個單路徑最短路還對嗎？該換什麼演算法？

App 3 無解

若 S 根本到不了 E（沒有路徑），程式會輸出什麼？這樣合理嗎？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

A1 · 建模

因為最優解一定「整批 F 箱走同一條路」（見 A2），所以每條邊上的運量要嘛 0、要嘛 F， 不會是中間值。既然只有「運 F 箱」這一種情況，就能事先把它的花費算成固定權重： 

// 邊權
w = (F <= D) ? F*C : D*C + (F-D)*C'

之後在這張固定權重的圖上找最便宜路徑，就是最短路。

A2 · 凹 vs 凸

折扣 ⇒ 每條邊的花費函數 g(x) 是凹的、過原點 ⇒ 次可加： g(a+b) ≤ g(a)+g(b)。意思是「合併送」永遠不比「分開送」貴。

把任何分流方案沿每條邊合併，總花費只減不增 ⇒ 一定能合併成「全部走一條路」。 於是只要找最便宜的單一路徑 ⇒ 最短路。

反之，凸成本（漲價、邊際遞增）是超可加：g(a+b) ≥ g(a)+g(b) ⇒ 分開送才便宜 ⇒ 要拆流 ⇒ 這才是最小成本流（MCMF）的場景。

A3 · MCMF 反例

把邊拆成「容量 D 成本 C」+「容量 ∞ 成本 C'」後，MCMF 會優先用便宜的 C' 那條 （因為 C' ≤ C），等於沒先付前 D 箱的全額就偷拿折扣價 ⇒ 算出比實際更低的花費。 

// 樣本 2 套 MCMF（錯）
正確答案 = 9
兩邊拆 MCMF 算出 = 4   ← 偷拿折扣，嚴重偏低

根因：MCMF 的正確性建立在凸成本上；凹成本拆邊後模型本身就錯了。

A4 · 演算法選擇

壓完邊權後所有邊權都 ≥ 0（C, D, C', F 都正），所以 Dijkstra 正確且最快 O(M log N)。Bellman-Ford / SPFA 也會對但較慢、無負邊也沒必要。

Floyd 也可以（N ≤ 100 ⇒ O(N³) = 10⁶，× T=20 = 2×10⁷ 仍可過）， 且寫起來更短；但只需要單源最短路時 Dijkstra 更貼題。

A5 · 資料型別

邊權 w、dist[]、以及讀入的 F, C, D, C' 都建議用 long long。 

// int 溢位例
F = 10⁹, C = 50216
F * C = 50216 × 10⁹ ≈ 5.02 × 10¹³
int 上限 ≈ 2.1 × 10⁹  ⇒  乘法當下就溢位 ❌
long long 上限 ≈ 9.2 × 10¹⁸  ⇒  OK ✓

樣本第 3 組答案 50215999949784 本身就 > 5×10¹³，用 int 直接爆。

A · App 1

樣本 1（F=1）：F=1 ≤ D=1 ⇒ 每邊全額 C。路徑 1→2→4 = 1+5 = 6， 1→3→4 = 1+6 = 7。最短 = 6。

樣本 3：F=10⁹ > D=1000，邊 1→2 權重 = 1000·50216 + (10⁹−1000)·50216 = 10⁹ · 50216（因 C=C' 折扣無差）= 50215999949784。

A · App 2

不對了。漲價 = 凸成本 = 超可加 ⇒ 把流量拆到多條路徑才便宜，單路徑反而可能更貴。 這時要改用最小成本最大流（MCMF）：每條邊拆成多段遞增單價的子邊（容量 D 成本 C、其餘成本 C'≥C）， 逐單位沿最短增廣路送 F 箱。凸成本下這個拆邊模型才正確。

A · App 3

S 到不了 E 時 dist[E] 仍是初始的 INF (LLONG_MAX)，程式會原樣印出這個巨大數字。

本題保證有解（測資不會出無路徑的情況），所以這樣寫沒問題；但若題目可能無解，正式做法應特判 dist[E] == INF 輸出指定的「無解」字樣，而不是印出 LLONG_MAX。
