// 有數量上限的 0/1 背包 · 加一維「已選個數」

★ ★ ★ ☆ ☆ 核心 二維背包 dp[已選幾個][飽足] · 時間 O(N·K·M)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

承「高棕櫚農場」：吃高棕櫚，第 i 個有飽足感 Aᵢ（重量）、滿足感 Bᵢ（價值）， 總飽足感不能超過 M。這次多一個限制：最多只能吃 K 個。

求滿足感最大值。多筆 T ≤ 10，1 ≤ K ≤ N ≤ 100，1 ≤ M ≤ 1000， 1 ≤ Aᵢ ≤ 100，1 ≤ Bᵢ ≤ 10⁶。（30% 測資 K = N，即退化回原題）

就是 0/1 背包再多一條「件數上限」。範例 M=6, K=3、物品 (2,4)(3,5)(4,6)： 選 (2,4)+(4,6) 飽足 6≤6、用 2 個 ≤ 3、滿足 10。若 K=2 不變仍可（剛好 2 個）。

02

為什麼要多一維「已選個數」// Why an Extra Dim

A 枚舉吃哪些 O(2ᴺ)
每個拿/不拿、再數個數是否 ≤ K，N=100 ⇒ 2¹⁰⁰ 子集，爆炸。
× REJECT · TLE
B 直接套原題一維 dp[l] O(N·M)
一維 dp[l] 沒辦法限制件數——它會把能塞進容量的全拿，可能超過 K 個。少了「選幾個」這個狀態。
△ REJECT · WRONG
BOTH FAIL · ADOPT
C 二維 dp[已選 j 個][飽足 l] O(N·K·M)
多開一維記「已選幾個」。轉移 dp[j][l] = max(dp[j][l], dp[j−1][l−Aᵢ]+Bᵢ)，j、l 都倒序，答案 dp[K][M]。
● ADOPT · STABLE

核心思想：背包問題遇到「額外的計數限制」就多開一維去記那個計數。 這裡限制「件數 ≤ K」⇒ 多一維 j（已選個數），其餘和 0/1 背包一模一樣。

03

狀態與轉移// State & Transition

dp[j][l] = 選 ≤ j 個、飽足感 ≤ l 時的最大滿足感。

dp[j][l] = max( dp[j][l] , dp[j−1][l−Aᵢ] + Bᵢ )
dp[j][l] = 不吃第 i 個；dp[j−1][l−Aᵢ]+Bᵢ = 吃第 i 個（個數 −1、飽足 −Aᵢ 再加 Bᵢ）。j、l 都由大到小。
  1. 初始 · dp[j][l] = 0（什麼都沒選）。
  2. 逐物品 · 外層 i；對每個物品，j 從 K 到 1、l 從 M 到 Aᵢ（兩維都倒序）。
  3. 兩維倒序的理由 · 和一維背包同理——保證 dp[j−1][l−Aᵢ] 還是「沒考慮第 i 個」的值 ⇒ 每個物品只被拿一次。
  4. 答案 · dp[K][M]（dp 定義是 ≤，所以直接取上界即可）。

05

為什麼 j、l 都要倒序// Why Both Descending

// 兩維倒序 = 每個物品只拿一次
  1. 更新第 i 個物品時，dp[j−1][l−Aᵢ] 必須是「還沒考慮第 i 個」的值（即上一輪的表），否則同一物品會被重複拿。
  2. j 倒序（K→1）：更新 dp[j] 時，dp[j−1]（較小的 j）本輪還沒被改 ⇒ 仍是上一輪值。✓
  3. l 倒序（M→Aᵢ）：同一維背包的理由，dp[·][l−Aᵢ] 還沒被本輪改到。✓
  4. 兩維只要有一維寫成正序，dp[j−1][l−Aᵢ] 就可能已含第 i 個 ⇒ 重複拿、答案錯。所以兩維都倒序。

其實「個數維 j」和「容量維 l」在這裡角色一樣：都是有限資源（最多 K 個、最多 M 飽足）， 拿一個物品同時消耗「1 個名額」和「Aᵢ 飽足」，所以兩維一起 −1 / −Aᵢ、一起倒序。

06

複雜度// Complexity

N 物品 × K 個數 × M 容量 = O(N·K·M)
N,K ≤ 100、M ≤ 1000 ⇒ 10⁷ 每筆、T ≤ 10 ⇒ 10⁸，但每格一次 max，實測 < 0.05s
項目數量備註
TimeO(N·K·M) 每筆三層迴圈 i / j / l
SpaceO(K·M)二維 dp[K+1][M+1]
遞迴深度0（迭代）純填表
Constant factor低每格一次 max

注意型別：Bᵢ ≤ 10⁶、最多選 K ≤ 100 個 ⇒ 滿足感總和 ≤ 10⁸，雖然 int 勉強夠，用 long long 最保險。

07

一句洞察// One-Sentence Insight

多一個「最多 K 件」限制 ⇒ 背包多開一維記「已選個數」。
dp[j][l] = max(dp[j][l], dp[j−1][l−Aᵢ]+Bᵢ)，j、l 兩維都倒序 —
拿物品同時吃掉「1 個名額 + Aᵢ 飽足」，答案 dp[K][M]，O(N·K·M)。
