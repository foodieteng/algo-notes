最近點對

★ ★ ★ ★ ★ Closest Pair of Points O(N log² N)
PAGE 1 / 3  ·  概念
01

題目在問什麼 // What's the question?

QUESTION ✨

平面上有 N 個整數座標點 (N ≤ 2×10⁵, |x|, |y| ≤ 10⁹)， 請找出距離最近的兩個點，輸出它們的距離平方 d²。

為什麼輸出「距離平方」而不是距離本身？
因為 a < b 蘊含 a² < b²（兩數都非負時）， 比大小用平方一樣準，可以省下開根號 + 浮點誤差的痛苦。整題能用純整數 (long long) 跑完。

02

為什麼選分治 // Why Divide & Conquer?

把「所有點兩兩比」拆掉之前，先想看看其他做法的缺點：

A
暴力 O(N²)
兩層 for-loop 比所有 C(N, 2) 對。 N = 2×10⁵ ⇒ 約 2×10¹⁰ 運算，TLE。
B
Hash 格子法
把平面切 d×d 格、每點只查鄰格。 期望 O(N) 但最壞退化（點全擠一起）；浮點精度也要 care。
C
分治 O(N log² N) ✓
按 x 切一半 → 各自遞迴 → 合併時只看中線附近的「跨界候選」strip。 穩定且純整數。
03

想法是怎麼演化來的 // Thought Evolution

分治不是憑空想到的，是「想要避開全部比，但又不能漏掉答案」一步步演化出來：

想法 1
直接比所有 pair 兩個 for-loop。對 N = 2×10⁵ 是 4×10¹⁰ 次運算，秒爆。
→ 必須避開「全部比」。
↓
想法 2
切一半 → 各自遞迴 按 x 切兩半，左半遞迴算最小 d_L、右半算最小 d_R。
但問題：如果最近 pair 一個在左一個在右？ 直接切丟掉這個情況，會錯。
↓
想法 3
中線附近補抓「跨界候選」 設 d = min(d_L, d_R)。若有跨界 pair 比 d 更近， 它們的 x 距離必 < d — 所以候選只在中線左右各 d 寬的 strip 內。
但 worst case strip 可能裝下全部 N 個點。
↓
想法 4 ✓
strip 按 y 排序 + 每點只看 7 個鄰居 幾何 + Pigeonhole：strip 內任意 d-高度視窗最多 8 個點。 所以按 y 排序後，每點只需跟後面 7 個比 — 合併 O(N)！
→ 總 T(N) = 2·T(N/2) + O(N log N) = O(N log² N) ✓
04

演算法 · 三步驟 // The Algorithm in 3 Steps

對先按 x 排好序的點陣列，遞迴函式 closest(lo, hi) 做：

1
切一半
取中位 index mid = (lo + hi) / 2， 記中線 mid_x = pts[mid].x。
2
遞迴左右
d_L = closest(lo, mid)
d_R = closest(mid+1, hi)
d = min(d_L, d_R)
3
合併 strip
收 |x − mid_x| < √d 的點 → 按 y 排序 → 每點看下 7 個鄰居 → 更新 d。
Base case 🌱

n ≤ 3 時直接 brute force（最多 3 對）— 遞迴最深處沒有任何巧思， 所有聰明都在第 3 步合併。

07

為什麼 strip 內每點只看 7 個鄰居？ // Pigeonhole Proof

把 2d × d 高度視窗切成 4 個 (d/2) × (d/2) 子格：

  • 每個子格對角線 = d/√2 ≈ 0.71 d < d
  • 左半已遞迴出 d_L ≥ d ⇒ 左半 2 個子格各最多 1 點 ⇒ 左半最多 2 點
  • 右半同理 ⇒ 2d × d 內最多 4 點
  • 擴成 2d × 2d 視窗（上下兩個矩形疊）⇒ 最多 8 點（含自己）
關鍵推論

所以按 y 排序的 strip 中，每點之後至多 7 個 鄰居可能 < d。 實作再加 if (Δy² ≥ d) break; 提前終止，常數更低。

08

複雜度 // Complexity

T(N) = 2 · T(N/2) + O(N log N)
⇒ O(N log² N)
合併階段 strip 排序貢獻一個 log；遞迴展開共 log N 層

若改用 merge sort 在遞迴中同步維護 y 排序的副本，可降到 O(N log N)。 多數教學版採 O(N log² N) — 程式更直觀，N = 2×10⁵ 仍秒過。

09

一句洞察 // Key Insight

記住這一點 ✦

D&C 解這題的價值不在「切半」，而在「合併」 — 左右遞迴得到 d 後，畫一條寬 2d 的 strip 就把跨界候選縮到 O(N)； strip 按 y 排序 + 每點看 7 個鄰居又把比對縮到 O(N)。 整題的關鍵是 幾何 + pigeonhole，不是遞迴本身。
