重點回顧// Concept Recap

關鍵點內容
問題本質 最大「任兩數距離 ≥ K」子集和 = House Robber 的 K 推廣。
狀態 dp[i] = 看 a[1..i] 的最大合法和（1-indexed，dp[0]=0）。
轉移 dp[i] = max(dp[i−1], dp[i−K] + a[i])。
核心差別 「取」接 dp[i−K]（不是 dp[i−2]）⇒ 保證距離 ≥ K。
邊界 i−K ≤ 0 ⇒ 接 dp[0]=0（只取這一個）。
退化 K=2 = 取數字1；K=1 = 全取總和。

範例 Trace · 跑 dp 看 [1,3,4,2], K=3// Worked Trace

1-indexed，dp[0]=0。每步取 max(dp[i−1], dp[i−3]+a[i])（K=3）：

// INPUT4 3
1 3 4 2     // a[1]=1 a[2]=3 a[3]=4 a[4]=2
// i = 1
skip = dp[0] = 0
take = dp[1-3]=dp[-2] → i-K<0 → 0,  + a[1]=1  = 1   // 只取 a[1]
dp[1] = max(0, 1) = 1
// i = 2
skip = dp[1] = 1
take = dp[2-3]=dp[-1] → i-K<0 → 0,  + a[2]=3  = 3   // 只取 a[2]
dp[2] = max(1, 3) = 3
// i = 3
skip = dp[2] = 3
take = dp[3-3] = dp[0] = 0,  + a[3]=4  = 4          // 取 a[3]（前面湊不到距離 3）
dp[3] = max(3, 4) = 4
// i = 4
skip = dp[3] = 4
take = dp[4-3] = dp[1] = 1,  + a[4]=2  = 3          // 取 a[1] 與 a[4]（距離 3 ✓）
dp[4] = max(4, 3) = 4
// OUTPUTdp[4] = 4
觀察：最佳是只取 a[3]=4。雖然「取 a[1]+a[4]」距離合法，但和只有 3 < 4。 和取數字1（同陣列、K=2 → 答案 5，取 a[1]+a[3] 或 a[2]+a[4]）對照：K 變大 ⇒ 能同時取的變少 ⇒ 答案不增反減。

應用題// Quick Calculations

App 1 空間優化

N 很大、記憶體吃緊時，怎麼把 O(N) 空間壓到 O(K)？

App 2 輸出方案

若要輸出「取了哪些 index」，怎麼從 dp[] 回溯？

App 3 K 單調性

固定陣列、令 f(K) = 答案。f(K) 隨 K 變大是遞增、遞減、還是不一定？為什麼？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · 轉移推導

「同時取 aᵢ 與 aⱼ 須 |i−j| ≥ K」。若取了 a[i]，設前一個被取的 index 為 j，則 i − j ≥ K， 即 j ≤ i − K。所以取了 a[i] 後，剩下能自由選的前綴是 a[1..i−K]，其最佳和 = dp[i−K] ⇒ take = dp[i−K] + a[i]。「2」只是 K=2 的特例。

Q2 · 與取數字1

唯一差別：「取」那項從 dp[i−2] 換成 dp[i−K]。其餘狀態、轉移結構、base 全相同。 代入 K = 2 ⇒ dp[i] = max(dp[i−1], dp[i−2] + a[i])，完全變回取數字1。 所以取數字1 是本題 K=2 的特例。

Q3 · 為何夠

因為 dp[] 是「前綴最佳和」，且 aᵢ ≥ 1 全為正 ⇒ dp 單調不減 （多看一格的前綴最佳不會變小）。所以對任何 j ≤ i − K，dp[j] ≤ dp[i−K]， 前一個被取的落在更早的位置不會比落在 i−K 更好。接 dp[i−K] 一項就已涵蓋「上一個取在 ≤ i−K 任意位置」的所有情況。

Q4 · 邊界

i − K < 0 表示「取了 a[i]、但前面根本湊不出一個距離 ≥ K 的位置」⇒ 只能單獨取 a[i] 這一個。 code 用 (i−k >= 0) ? dp[i−k] : 0 回退到 0，等價於接 dp[0]=0 ⇒ take = 0 + a[i] = a[i]，正是「只取它」。

Q5 · 滾動可行？

取數字1 的轉移只用到 dp[i−1]、dp[i−2]，固定兩格 ⇒ take/skip 兩變數即可。 本題用到 dp[i−K]，K 不固定且可大到 N ⇒ 至少要能查到 K 格之前的 dp 值。

所以最少要保留 K+1 格（dp[i−K..i]）⇒ 用一個大小 K+1 的環狀緩衝把空間壓到 O(K)； 但 N ≤ 1000，直接開整條 dp[] 最簡單也夠用，不必為了 O(K) 增加出錯風險。

App 1 · 空間優化

只需 dp[i−1]（skip）與 dp[i−K]（take 的 feeder）。維護一個長度 K+1 的環狀陣列 ring[]， ring[i % (K+1)] 存 dp[i]，要查 dp[i−1]、dp[i−K] 時用 ring[(i−1)%(K+1)]、 ring[(i−K+K+1)%(K+1)]。空間 O(K)。N ≤ 1000 其實不需要，但 N 大時這招把記憶體從 O(N) 降到 O(K)。

App 2 · 輸出方案

留整條 dp[]，從 i = N 往回走： 

// backtrack
i = N
while i >= 1:
    if dp[i] == dp[i-1]:        // 沒取 a[i]（沿用前一格）
        i -= 1
    else:                       // 取了 a[i] = dp[i-K]+a[i]
        記錄 i;  i -= K

反序記錄的 index 即為取法（之一）。需 O(N) 空間存 dp。注意若 dp[i]==dp[i-1] 又恰好也==dp[i-K]+a[i]， 優先當「沒取」走即可（任一條最佳解都行）。

App 3 · K 單調性

遞減（非嚴格，f(K) ≥ f(K+1)）。K 越大、距離限制越嚴 ⇒ 合法的取法集合越小（K+1 的任何合法解，任兩數距離 ≥ K+1 ≥ K，也是 K 的合法解）。 在更小的可行集合上取最大，最大值不會更大。

本例印證：同陣列 [1,3,4,2]，K=2 答案 5、K=3 答案 4 ⇒ 5 ≥ 4。極端 K=1（無限制）給全取總和（最大），K=N 只能取一個（最小）。
