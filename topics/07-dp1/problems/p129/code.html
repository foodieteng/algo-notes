01

完整實作 · 區間 DP 按長度填表// Full C++

1-indexed。先算前綴和讓 sum(i..j) O(1)；外層跑區間長度 len由小到大， 內層枚舉左端 i（右端 j = i+len−1），最內層枚舉切點 k。

p129.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 105;
long long a[MAXN];            // a[1..n] 各円円糟糕值
long long prefix[MAXN];       // prefix[i] = a[1]+...+a[i]
long long dp[MAXN][MAXN];     // dp[i][j] = 合成 a[i..j] 的最小高棕櫚

void solve() {
    int n;
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> a[i];

    prefix[0] = 0;
    for (int i = 1; i <= n; i++) prefix[i] = prefix[i - 1] + a[i];

    for (int i = 1; i <= n; i++) dp[i][i] = 0;   // 單一円円不必合成

    for (int len = 2; len <= n; len++) {        // 區間長度由小到大
        for (int i = 1; i + len - 1 <= n; i++) {
            int j = i + len - 1;
            long long sum = prefix[j] - prefix[i - 1];  // 合 [i..j] 那一步固定付的成本
            long long best = LLONG_MAX;
            for (int k = i; k < j; k++) {          // 枚舉最後一步的切點
                long long cand = dp[i][k] + dp[k + 1][j] + sum;
                if (cand < best) best = cand;
            }
            dp[i][j] = best;
        }
    }

    cout << dp[1][n] << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) solve();

    return 0;
}

填表順序為何用 len？ 算 dp[i][j] 需要更短的子區間 dp[i][k]、dp[k+1][j]（長度都 < len）， 所以先把所有較短區間算完，長區間才能正確取用 — 這正是「由小到大」的無後效性。

複雜度分析// Complexity Derivation

步驟內容成本
前綴和一遍掃出 prefix[]O(N)
區間數(len, i) 對 ≈ N²/2 個O(N²)
每個區間枚舉切點k 從 i 到 j−1，最多 N 個O(N)
sum(i..j)前綴和相減O(1)
全部 T 筆—O(T · N³)
O(N²) 區間 × O(N) 切點 = O(N³) 每筆
N ≤ 100 ⇒ ~10⁶、T ≤ 10 ⇒ ~10⁷，秒過
項目數量備註
TimeO(N³) 每筆三層迴圈 len / i / k
SpaceO(N²)dp 表；前綴和 O(N)
遞迴深度0（迭代）純填表
Constant factor低每步一次加法一次比較
03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
填表順序必須按 len 若直接 for i for j 順序填，算 dp[i][j] 時 dp[k+1][j] 可能還沒算好 ⇒ 錯。一定要由短到長（或記憶化遞迴）。
sum 加在外層、與 k 無關 「合 [i..j] 那一步付 sum(i..j)」是固定的，對每個 k 都要加。常見錯誤是漏加或只在某些 k 加。
切點範圍 i ≤ k < j k=j 會讓右半 [j+1..j] 空、左半就是整段 ⇒ 無效切。k 必須 < j 才能真的二分。
用前綴和 每個 (i,j,k) 都要 sum(i..j)。若每次重算 O(N) ⇒ 總 O(N⁴) 可能超時。前綴和讓它 O(1)。
貪心不可 「相鄰才能合」≠ Huffman（可任意配對）。最小相鄰對先合未必最佳，務必 DP。
型別 aᵢ ≤ 1000、N ≤ 100 ⇒ 總和 ≤ 10⁵，成本上界約 N×總和 ≈ 10⁷，int 夠；用 long long 保險。
04

細節走讀// Section Walkthrough

① 狀態定義

// state
dp[i][j] = 把連續區段 a[i..j] 全部合成成「一隻」的最小總高棕櫚
dp[i][i] = 0          // 單一隻，不必合
答案 = dp[1][n]

② 轉移（枚舉最後一步切點 k）

// transition
sum = prefix[j] - prefix[i-1]            // 合 [i..j] 那步必付
dp[i][j] = min over k in [i, j-1] of
             dp[i][k] + dp[k+1][j] + sum
// 左半先合成一隻、右半先合成一隻、再合兩者(付 sum)

sum 與 k 無關 ⇒ 也可寫成 sum + min_k(dp[i][k]+dp[k+1][j])，一樣的意思。

③ 為什麼按長度填

// fill order
len = 2,3,...,n            // 先短後長
  i = 1..n-len+1, j = i+len-1
算 dp[i][j] 用到的 dp[i][k]、dp[k+1][j]
長度都 < len → 已經算好 → 無後效性成立

ONE-LINE SUMMARY
前綴和 + 區間 DP：len 由小到大填，dp[i][j]=min_k(dp[i][k]+dp[k+1][j])+sum(i..j)，O(N³)。
填表務必按長度；sum 對每個 k 都加；k 範圍 [i,j−1]；前綴和讓 sum 為 O(1)。
