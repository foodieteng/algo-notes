最小生成樹

// Minimum Spanning Tree · Kruskal + DSU

★ ★ ★ ☆ ☆ 核心 MST · Kruskal + 並查集 · 時間 O(M log M)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

給一張連通的無向帶權圖（N 個點、M 條邊）， 求最小生成樹（MST）的權重和 —— 也就是挑出 N−1 條邊把所有點連起來、 且邊權總和最小。

N ≤ 2×10⁵ · N−1 ≤ M ≤ 2×10⁵ · 0 ≤ wᵢ ≤ 10⁹ · 圖保證連通。 生成樹 = 用 N−1 條邊連通全部 N 個點且不成環；MST 就是其中權重和最小的那棵。

02

為什麼用 Kruskal// Why Kruskal

A 枚舉所有生成樹 指數級
生成樹數量可達 N^(N−2)（Cayley），完全不可能枚舉。
× REJECT · TLE
B Prim + 樸素 O(N²)
稠密圖 OK，但本題用邊表 + 稀疏，鄰接矩陣 N=2×10⁵ 開不下。
△ REJECT · 不適稀疏
BOTH FAIL · ADOPT
C Kruskal + DSU O(M log M)
邊按權排序，由小到大貪心加入；用並查集判環，連成 N−1 條就停。
● ADOPT · OPTIMAL

貪心成立的理由（cut property）：對任意把點切成兩半的方式，跨越這個切口的最小邊 一定屬於某棵 MST。Kruskal 由小到大加邊、只跳過會成環的，等於每次都安全地選了一條 cut 上的最小邊。

註：被淘汰的是樸素 O(N²) Prim（鄰接矩陣，N=2×10⁵ 開不下）。 換成 min-heap 版 Prim（O(M log N)）同樣正確、同樣能過 —— 它也靠 cut property，只是改成「以點為中心、從一點長出樹」。 Code 頁兩種寫法都附上，可對照學習。

03

核心策略 · 三步驟// Algorithm Outline

  1. 排序 · 把 M 條邊按邊權 w 由小到大排序。
  2. 貪心加邊 + 判環 · 依序看每條邊 (u, v, w)：若 u, v 還不在同一個連通塊 （並查集 find(u) ≠ find(v)）就加入、union 起來、累加 w；否則跳過（會成環）。
  3. 連滿就停 · 加滿 N−1 條邊即構成生成樹，輸出權重和。

並查集（DSU）是判環的關鍵：find 查代表元，union 合併兩塊。 搭配路徑壓縮 + 按秩合併，每次操作近乎 O(1)（反阿克曼 α）。

05

為何貪心加最小邊一定對// Cut Property

Kruskal 每加一條邊，其實是在「目前兩個連通塊之間」選一條最小的橋。 它的正確性由 cut property（切割性質）保證。

CUT PROPERTY · 切割上的最小邊必在 MST

把點集任意切成兩堆 (A, B)。所有「一端在 A、一端在 B」的邊叫跨切邊。

  1. 設 e 是所有跨切邊中權重最小的那條。宣稱：存在一棵 MST 包含 e。
  2. 反證：設某棵 MST T 不含 e。把 e 加進 T 會形成一個環， 這個環必定還有另一條跨切邊 e'（因為環要從 A 跨到 B 再回來）。
  3. 因 e 是跨切最小 ⇒ w(e) ≤ w(e')。把 e' 換成 e 仍是生成樹， 且權重不增 ⇒ 得到一棵含 e 的 MST。矛盾解除。
  4. Kruskal 每次加的邊，正是「連接兩個不同連通塊」的當前最小邊 —— 把「已連好的那塊」當作 A，其餘當 B，這條邊就是該切口的最小跨切邊 ⇒ 安全。

⇒ 由小到大加邊、跳過成環者，每一步都選了一條安全邊 ⇒ 最終得到 MST。

跳過成環的邊為何安全：若 u, v 已同塊，這條邊兩端在同一堆，不是跨切邊， 加了只會製造環、不會連通新東西 —— 丟掉它不損失任何連通性。

06

複雜度// Complexity

瓶頸是排序 O(M log M)；DSU 的 M 次 find/union 近乎線性 O(M α(N))，可視為常數倍。

T = O(M log M)
M ≤ 2×10⁵ ⇒ 排序約 2×10⁵ × 18 ≈ 4×10⁶，輕鬆通過
項目數量備註
TimeO(M log M)排序主導；DSU 近乎 O(M·α)
SpaceO(N + M)邊陣列 + 並查集 par/rank
數值型別long long權和最大 (N−1)·10⁹ ≈ 2×10¹⁴，必用 64-bit
DSU 優化路徑壓縮 + 按秩單次操作攤銷 α(N) ≈ 常數

一句洞察// One-Sentence Insight

邊權排序 + 由小到大貪心加邊 + 並查集判環 = MST。
cut property 保證「跨切口的最小邊一定安全」，
所以每次選當前最小的「連接兩塊」之邊都不會錯。
加滿 N−1 條邊即停；權和要開 long long。
