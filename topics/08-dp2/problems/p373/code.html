01

完整實作 · tails + upper_bound// Full C++

這是能 AC 的寫法：用 LIS 的 tails 技巧（耐心排序）。 tails[k] = 長度 k+1 的非遞減子序列裡「最小的結尾值」。每個數字有兩個候選值 {2aᵢ, aᵢ}（須 ≥ M）， 大的先放、小的後放，各做一次 upper_bound：能接長就 push、否則把對應 tail 壓小。答案 = tails.size()。

p373.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) {
        int N;
        long long M;
        cin >> N >> M;

        vector<long long> a(N);
        for (int i = 0; i < N; i++) cin >> a[i];

        // tails[k] = 長度為 k+1 的非遞減子序列，所有結尾值裡「最小的那個」
        // 每個數字有兩個候選值 {2*a[i], a[i]}（須 >= M）。
        // 先放「大候選 2*a[i]」再放「小候選 a[i]」，避免同一個數字被串兩次。
        vector<long long> tails;
        for (int i = 0; i < N; i++) {
            long long cand[2] = { 2 * a[i], a[i] };   // 大的先、小的後
            for (long long x : cand) {
                if (x < M) continue;                  // 此候選值 < M，不可用
                // 非遞減 => upper_bound：找第一個「嚴格大於 x」的位置
                auto it = upper_bound(tails.begin(), tails.end(), x);
                if (it == tails.end()) tails.push_back(x);   // x 比所有 tail 都大 => 接長
                else *it = x;                                // 否則把那個 tail 壓小成 x
            }
        }
        cout << tails.size() << '\n';
    }
    return 0;
}

為什麼是 upper_bound 不是 lower_bound？ 非遞減（允許相等）⇒ 一個值 x 可以接在「等於它」的結尾後面， 所以要替換的是第一個嚴格大於 x 的 tail ⇒ upper_bound。若改 lower_bound 就變成嚴格遞增，相等鏈被排除、答案偏小。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
upper_bound 不是 lower_bound 非遞減（允許相等）⇒ 用 upper_bound（替換第一個嚴格大於 x 的 tail）。用 lower_bound 會變嚴格遞增、WA。
大候選先、小候選後 同一個數字的兩個候選值 {2aᵢ, aᵢ} 要先放大的再放小的，避免同一數字被串進同一條鏈兩次。
候選值要 ≥ M 只有 ≥ M 的候選能進 tails。若 2aᵢ < M 連兩倍都不夠 ⇒ 兩個候選都 skip。
全程 long long 2aᵢ 最大 2×10⁹ > int 上限 ⇒ 候選值、tails、比較一律 long long。
答案可能 0 若所有候選值都 < M（如範例2 a=[7,8,9], M=42），tails 一直空 ⇒ size()=0。
T 筆要清空 每筆測資 tails 重新宣告（或 clear）；用區域 vector 每筆新建最省事。
04

細節走讀// Section Walkthrough

① tails 是什麼

// tails meaning
tails[k] = 「長度為 k+1 的非遞減子序列」中，所有可能結尾值裡最小的那個。
tails 永遠是遞增的（嚴格遞增），長度 = 目前能組出的最長鏈。
直覺：結尾越小，後面越容易接 → 貪心地讓每種長度的結尾都最小。

② 一個值 x 怎麼更新 tails

// one value
it = upper_bound(tails, x)        // 第一個 > x 的位置
if (it == end) tails.push_back(x) // x ≥ 所有 tail → 接出更長一格
else           *it = x            // 把第一個 > x 的 tail 壓小成 x
非遞減用 upper_bound：x 可接在「= x」的結尾後，所以只動「嚴格大於」的。

③ 兩候選值：大先小後

// two candidates
cand = {2*a[i], a[i]}             // 大的先
對每個 ≥ M 的候選做一次上面的更新。
為何大先小後？小候選 a[i] 之後處理時，雖然會看到剛放進去的 2*a[i]，
但 2*a[i] ≥ a[i]，upper_bound(a[i]) 只會動到「> a[i]」的位置，
不會把同一數字的大候選接到小候選之後 → 同一數字不被串兩次。

④ 從 O(N²) DP 到這裡

// from the DP
概念頁的 dp[i][c] = 1 + max{前面值 ≤ 它的 dp} 是 O(N²)。
tails 是同一個「最長非遞減鏈」問題的 O(N log N) 標準寫法：
不記每個位置的 dp，而是維護「每種長度的最小結尾」，二分搜定位。
兩者答案一定相同（已對拍驗證），tails 版才吃得滿 N=10⁵。
