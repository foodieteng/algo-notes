01

重點回顧// Concept Recap

關鍵點內容
問題本質 數樓層序列：每步 |y−x| < |b−x|，求長度 K 的序列數。
關鍵觀察 合法落點 y ∈ [x−d+1, x+d−1]（d=|x−b|）是連續區間且碰不到 b。
狀態 cur[f] = 搭到目前、停在樓層 f 的方法數。
加速 區間加 ⇒ 差分陣列，每趟 O(n)。
細節 扣 y=x 自身、減法補 MOD、答案 Σ cur[f]。
複雜度 O(N·K)；MOD 1e9+7（質數，但只用加減）。
02

觀念題// Concept Questions

Q1 建模

條件 |y−x| < |b−x| 展開後，合法落點 y 是什麼形狀？為什麼這保證跳不到禁止樓層 b？

Q2 為何差分

樸素 DP 每趟為什麼是 O(n²)？差分陣列怎麼把它降到 O(n)？關鍵前提是什麼？

Q3 扣自身

為什麼前綴和算完後要把每個 nxt[x] −= cur[x]？不扣會多算什麼？

Q4 嚴格不等

條件是嚴格 < 還是 ≤？若不小心寫成 ≤，會發生什麼？（提示：想 y=b）

Q5 答案位置

為什麼答案是 Σ cur[f] 而不是某一個 cur[?]？搭完 K 趟後可以停在哪些樓層？

03

範例 Trace · n=8, a=2, b=5, K=2// Worked Trace

cur[f] 從起點 a=2 出發。樓層 5 是禁止的（紅）。每趟把每個 cur[x] 灑到 [x−d+1, x+d−1]（扣 y=x）：

// INPUT8 2 5 2     // n=8, a=2, b=5, K=2
// init (s=0)
樓層 f :  1   2   3   4  [5]  6   7   8
cur    :  0   1   0   0   -   0   0   0     ← 只有 a=2 有 1 種
// 第 1 趟：從 x=2 散播
x=2: d=|2-5|=3 → 落點 [2-2, 2+2]=[0,4] 夾成 [1,4]，扣掉 y=2
   → 加到 {1,3,4} 各 +1
cur    :  1   0   1   1   -   0   0   0     （sum 還沒當答案，繼續第 2 趟）
// 第 2 趟：從 x=1,3,4 各自散播
x=1: d=|1-5|=4 → [1-3,1+3]=[-2,4]→[1,4]，扣 y=1 → 加 {2,3,4}
x=3: d=|3-5|=2 → [3-1,3+1]=[2,4]，扣 y=3 → 加 {2,4}
x=4: d=|4-5|=1 → [4-0,4+0]=[4,4]，扣 y=4 → 加 {} （唯一落點就是自己，無效）
累加：
cur    :  0   2   1   2   -   0   0   0
              ↑   ↑   ↑
            f=2:從1,3   f=3:從1   f=4:從1,3
// OUTPUTΣ cur[f] = 2 + 1 + 2 = 5
觀察：第 2 趟時 x=4 的 d=1，合法區間只剩它自己 ⇒ 扣掉後沒有任何落點，這條路「卡住」了。 離 b 越近、d 越小、可走的範圍越窄，這正是 |y−x| < |b−x| 的幾何意義。最終 5 種搭法。
04

應用題// Quick Calculations

App 1 為何 0

範例2 n=7, a=1, b=2, K=2 答案是 0。為什麼從 a=1 連一趟都搭不出去？

App 2 前綴和重建

差分陣列 diff 做完所有區間加後，要怎麼還原出真正的 nxt[]？這一步的複雜度？

App 3 變體

若把條件改成 |y−x| ≤ |b−x|（允許等於、但仍 y≠b），落點區間會變怎樣？要多加什麼特判？

Q1 · 建模

|y−x| < d（d=|x−b|）展開為 −d < y−x < d ⟺ y ∈ [x−d+1, x+d−1]， 是以 x 為中心、半徑 d−1 的連續整數區間。

b 在距離 x 恰好 d 的位置，而區間最遠只到 d−1 ⇒ b 永遠落在區間外，所以每趟自動跳不到也跳不過 b，不用特別排除。

Q2 · 為何差分

樸素轉移：對每個 x，要把 cur[x] 加到 O(n) 個合法 y ⇒ 每趟 O(n²)、K 趟 O(n²K)=1.6×10¹⁰，TLE。

關鍵前提：合法 y 是連續區間。於是「把一個值加到一段區間」可用差分 diff[lo]+=v, diff[hi+1]−=v 做到 O(1)， 全部加完再一次前綴和 O(n) 還原 ⇒ 每趟 O(n)，總 O(nK)。

Q3 · 扣自身

合法區間 [x−d+1, x+d−1] 包含 x 本身（|x−x|=0 < d）。但電梯要動、y ≠ x， 所以區間加會多算一份「停在原地」的假搭法。

前綴和算完後，對每個出發樓層 x 做 nxt[x] −= cur[x] 把這份多算的扣掉。不扣的話每趟都會灌水。

Q4 · 嚴格不等

是嚴格 <。所以區間右端是 x+d−1（不含 d）。

若寫成 ≤，區間會延伸到 x±d，剛好包含 b（b 在距離 d 處）⇒ 會允許跳到禁止樓層、且 d=1 時還會允許停原地的相鄰，答案整個錯。範例2 用 ≤ 會算出非 0。

Q5 · 答案位置

題目只要求「連續搭 K 趟」，沒有規定終點。所以搭完 K 趟後停在任何樓層都算一種合法搭法。

因此答案是把 cur[] 全部加起來 Σ_f cur[f]，代表「所有可能終點的方法數總和」。

App 1 · 為何 0

a=1、b=2，距離 d = |1−2| = 1。合法落點區間 [1−0, 1+0] = [1,1] ＝ 只有樓層 1 自己。 扣掉 y=x 後沒有任何合法落點 ⇒ 第一趟就搭不出去 ⇒ 0 種。

直覺：你緊貼著禁止樓層（距離 1），任何移動都會碰到或越過它，所以動彈不得。

App 2 · 前綴和重建

差分還原就是做前綴和：nxt[y] = nxt[y−1] + diff[y]（從 y=1 掃到 n，邊掃邊累加 run）。 這樣每個 diff[lo]+=v / diff[hi+1]−=v 的「區間加」效果就被展開成 [lo,hi] 每格都 +v。

複雜度 O(n)：一次線性掃描即可還原整條 nxt[]。

App 3 · 變體

改成 |y−x| ≤ d ⇒ 區間變 [x−d, x+d]（兩端各多 1 格）。但 x−d 或 x+d 其中一個正好是 b。

所以要多一個特判把 y=b 那格扣掉（像扣 y=x 一樣，再做一次 nxt[b] −= cur[x]，但只在 b 落在該 x 的區間內時）。 本題用嚴格 < 就天生避開 b，不必這個特判 — 這也是原題刻意設計成嚴格不等的原因。
