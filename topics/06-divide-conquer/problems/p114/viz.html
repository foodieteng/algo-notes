動畫演示

Base Case · n = 2 直接解

在看主動畫之前，先把 base case 弄清楚。 當 tile() 遞迴到 n = 2 時： 2×2 板共 4 格，扣掉 blocked 後剩 3 格本身就是一個 L， 不需再切象限、不需放中央 L，直接 Report 三格座標即可。 以下是 4 種可能的 base case（黑格 ★ 是 blocked，橘格是 L）：

Case 01
blocked = (1,1)
Case 02
blocked = (1,2)
Case 03
blocked = (2,1)
Case 04
blocked = (2,2)
// 為什麼 base case 一定有解？

2×2 中 4 格扣掉任一格，剩下 3 格一定組成 L（兩個邊上格 + 一個對角格）— 不會出現 1×3 直線，因為 1×3 至少需要 3 個共線的格子， 而 2×2 內不可能找到 3 個共線非 blocked 格子。 這保證了遞迴在最深層一定能停下並合法輸出。

主動畫 · N = 4 範例

以 solve(4, 3, 4) 為例。★ 是王老先生選的特殊格 (3, 4)。 按 Play 自動播放，或用 Prev / Next 逐步檢視 5 個 L-tromino。 每個 L 用不同顏色實心填滿，剛放下的 L 邊框會加粗黑色高亮。 第 1 步是中央 L（遞迴步驟），第 2–5 步全部是 base case 直接解 — 跟上面 4 個 mini-grid 完全對應。

-- / --
INITIAL · 4×4 board · special = (3, 4)

遞迴順序對照

下面是 5 個 L 對應的 tile() 呼叫順序（在實際 code 中，遞迴是 DFS，所以中央 L 會「最先 Report」）：

StepL 名稱覆蓋的格子來自
1L1 · Center(2,2), (2,3), (3,2)tile(1,1,4,3,4) 中央放 L
2L2 · TL(1,1), (1,2), (2,1)tile(1,1,2,2,2) base case
3L3 · TR(1,3), (1,4), (2,4)tile(1,3,2,2,3) base case
4L4 · BL(3,1), (4,1), (4,2)tile(3,1,2,3,2) base case
5L5 · BR(3,3), (4,3), (4,4)tile(3,3,2,3,4) base case
// 觀察

L1 在 TL / TR / BL 三個象限的最角落格上各放一個 cell — 這 3 個 cell 在子問題裡就是該象限的「new special」。 BR 象限因為「原 special (3,4)」就在裡面，所以不需要中央 L 補一個 cell — 直接遞迴用原 special 解。

簡易 Trace 卡

下面把同一個 solve(4, 3, 4) 拆成講義式小步驟。 每張卡只標出「這一步剛放下的 L」與它對應的 Report， 方便先看懂遞迴修補中心，再回頭對照上面的動畫。

STEP 1 · Center Repair

先在正中央補一塊 L

原 special 在右下 BR，所以中央 L 覆蓋 TL/TR/BL 三個象限最靠近中心的角。 這三格之後會成為各自 2×2 子問題的 special。

L1L1 L1王 
Report(2,2, 2,3, 3,2);
STEP 2 · TL Base Case

左上 2×2：扣掉 (2,2)

中央 L 已佔住 (2,2)，所以左上子板剩下三格剛好形成一塊 L。

L2L2 L2hole 
Report(1,1, 1,2, 2,1);
STEP 3 · TR Base Case

右上 2×2：扣掉 (2,3)

右上子板把中心旁的 (2,3) 視為 special，其他三格回報成 L。

L3L3 holeL3 
Report(1,3, 1,4, 2,4);
STEP 4 · BL Base Case

左下 2×2：扣掉 (3,2)

左下子板把 (3,2) 視為 special，剩下三格就是一個 L。

L4hole L4L4 
Report(3,1, 4,1, 4,2);
STEP 5 · BR Base Case

右下 2×2：扣掉王老先生的 (3,4)

最後一個子板使用原本的 special。五次 Report 後，除了王老先生那格以外全部鋪滿。

L5王 L5L5 
Report(3,3, 4,3, 4,4);
