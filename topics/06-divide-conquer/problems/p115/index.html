逆序對和

// Inversion-Pair Sum · Merge Sort 順便算貢獻

★ ★ ★ ★ ★ 核心 Divide & Conquer · Merge Trick · 時間 O(N log N)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

長度 N ≤ 10⁶ 的正整數序列 a[1..N]（每個 ≤ 10⁹）。 對每組逆序對 (i, j)（i < j 且 a[i] > a[j]） 計算 a[i] + a[j]，加總後輸出 mod 10000019。

最壞情況（全降序）逆序對數量 ≈ 5 × 10¹¹ ⇒ 絕對不能 O(N²) 枚舉，但仍要把每對的「兩數和」都計到。

03

核心策略 · 三步驟// Algorithm Outline

merge_sort(left, right) 做 3 件事：

  1. 切半 · mid = (left + right) / 2
  2. 遞迴左右 · 各自排序 + 各自算內部逆序對和
  3. 合併 merge · 兩半已排序 → 雙指針合併時順便算跨界逆序對和

Base case · left ≥ right 直接 return（單一元素，無逆序對）。

05

跨界貢獻的數法// Cross-Boundary Counting

假設 merge 進行到某一步，左半游標 i、右半游標 j， 且 a[i] > a[j]，要取右半的 a[j]。為什麼這一動可以一次算完一群逆序對？

// MERGE STEP · 跨界對的數法
  1. 左半在 merge 前已排序，所以 a[i] ≤ a[i+1] ≤ … ≤ a[mid]。
  2. 既然 a[i] > a[j]，那 a[i+1], …, a[mid] 也全部 > a[j]。
  3. 左半下標全部 < 右半下標 ⇒ 這 mid − i + 1 個元素都跟 a[j] 形成逆序對。
  4. 本步貢獻 = (a[i] + a[i+1] + … + a[mid]) + a[j] × (mid − i + 1)。
  5. 用 remaining 滾動維護左半剩餘和：merge 開始算一次，取左半時就 remaining −= a[i] ⇒ O(1) 取用。

⇒ 排序與計貢獻共用一次掃描，merge 本來就要逐一比較，只是在「取右」時多加一筆 O(1) 計算。

06

複雜度分析// Complexity

每層 merge 掃過所有 N 個元素一次（O(N)），共切 log N 層：

T(N) = 2 · T(N/2) + O(N)  ⇒  Θ(N log N)
Master Theorem case 2：a = 2, b = 2, f(N) = O(N) = O(Nlogb a)
項目數量備註
TimeO(N log N)N = 10⁶ ⇒ ~2 × 10⁷ 次運算
SpaceO(N)全域 buf[] 暫存陣列
Recursion DepthO(log N)N = 10⁶ ⇒ 約 20 層
Constant factor低純整數比較、無浮點、無 sqrt
07

一句洞察// One-Sentence Insight

利用排序過程順便數，不需要額外資料結構。
Merge 的「取右半」這一動本來就要做，
我們只是在那一瞬間把「左半剩 k 個元素」這群逆序對一次性算進貢獻。
排序與計貢獻共用一條 O(N log N) 流水線。
