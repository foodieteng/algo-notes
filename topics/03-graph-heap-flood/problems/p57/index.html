01

題目簡介// Brief

QUESTION

處理 T ≤ 10⁶ 個操作，每個是兩種之一： 1 n 把數字 n 放入 heap；2 把 heap 中最小的數字 pop 出來並輸出。 若 pop 時 heap 已空，輸出一行 empty!。

要的就是一個支援「插入」與「取出最小」的資料結構 — 也就是 min-heap（最小堆 / 優先佇列）。 本題練的是自己用陣列把 heap 寫出來（或直接用 STL priority_queue）。

02

為什麼用 Heap// Why a Heap

A 未排序陣列 pop O(n)
insert O(1)，但每次 pop 要掃全部找最小 ⇒ 10⁶ 次 pop 最壞 O(n²) ≈ 10¹².
× REJECT · TLE
B 維護排序陣列 insert O(n)
pop O(1)（取頭），但每次 insert 要挪位插入 ⇒ 一樣 O(n²)。
△ REJECT · TLE
BOTH FAIL · ADOPT
C Binary Heap 兩者 O(log n)
完全二元樹存進陣列；insert 上浮、pop 下沉，各只走樹高 O(log n)。最小值永遠在根。
● ADOPT · STABLE

A、B 都把成本壓在某一邊、犧牲另一邊 ⇒ 10⁶ 操作必 TLE。Heap 讓 insert 與 pop 同時 O(log n)， 總時間 O(T log T) 才能在時限內跑完。

03

核心結構 · 陣列存樹// Array as Tree

把完全二元樹用 1-indexed 陣列存（不用指標）：

  1. 索引關係 · 節點 i 的左子 = 2i、右子 = 2i+1、父 = ⌊i/2⌋。
  2. heap 性質 · 每個節點都 ≤ 它的兩個子（min-heap）⇒ 最小值永遠在 heap[1]。
  3. 插入（sift-up） · 放到陣列尾端，與父比、比父小就交換上浮，直到 ≥ 父或到根。
  4. 取出最小（sift-down） · 輸出 heap[1]，把最後一個搬到根，與較小的子比、比子大就交換下沉。

樹高 = ⌊log₂ n⌋，sift-up / sift-down 各最多走這麼多層 ⇒ 每次操作 O(log n)。

05

為什麼每次操作只要 O(log n)// Why O(log n)

// LEMMA · 完全二元樹的高度是 ⌊log₂ n⌋
  1. heap 是完全二元樹：除最後一層外每層填滿，最後一層由左往右填。
  2. 第 d 層最多 2ᵈ 個節點 ⇒ 前 h 層共 2ʰ⁺¹ − 1 個。
  3. 所以 n 個節點的樹高 h = ⌊log₂ n⌋。
  4. sift-up 每次往父走一層、sift-down 每次往子走一層 ⇒ 最多走 h 層。
  5. ⇒ insert、pop 各 O(log n)；每層只做常數次比較與交換。

對照：未排序陣列 pop 要 O(n)、排序陣列 insert 要 O(n)。Heap 用「只維持局部父子關係、不要求全序」換來兩邊都 O(log n)。
