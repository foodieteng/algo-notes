重點回顧// Concept Recap

關鍵點內容
十字的容量 row i + col i 共用對角格 ⇒ 2N − 1 格，剛好放 {1..2N−1} ⇒ 每數恰好一次。
對角 = N 對角中心同屬 row i 與 col i，全放常數 N，剩 2N−2 格分給互補對。
互補對（和 2N） {k, 2N−k}，共 N−1 組。要求 a[i][c] + a[c][i] = 2N。
化約 = 1-因子分解 「每個十字碰每組互補對一次」= 把 K_N 用 N−1 色正常著色，每點看到全部顏色。
XOR 神兵 N = 2^K ⇒ color(i,j) = i ^ j 自動滿足雙射 + 對稱 + 完美配對，一行解決。
定方向 i==j → N；i<j → i^j；i>j → 2N−(i^j)。
複雜度 O(N²) 時間、O(1) 空間（不存陣列），row-major Report。
02

觀念題// Concept Questions

Q1 容量觀察

為何「row i ∪ col i = {1..2N−1}」代表這 2N−1 格裡每個數恰好出現一次？這個觀察是怎麼推出來的？

Q2 XOR 雙射

固定 i，為何 i XOR j（j 跑遍 j≠i）會剛好掃過 1..N−1 每個值各一次？這個性質為何需要 N = 2^K？

Q3 1-因子分解

把 index 當點、color(i,j) = i^j 當邊色，請說明「每色是一個完美配對」這件事，並寫出顏色 k 把誰配給誰。

Q4 方向不變量

為何要用 i<j / i>j 分方向？若 (i,j) 和 (j,i) 兩格都填 i^j，會違反哪一條性質？

Q5 對角選值

對角線為何選 N 而不是別的值（譬如 1 或 2N−1）？換成別的常數還能用嗎？

03

範例 Trace · 完整跑 solve(4)// Worked Trace

跑 solve(4)，N = 4、2N = 8。逐格套公式（0-indexed）：

// ROW 0 · i = 0
(0,0) i==j → 4
(0,1) i<j  → 0^1 = 1
(0,2) i<j  → 0^2 = 2
(0,3) i<j  → 0^3 = 3
→ row 0 = [4, 1, 2, 3]
// ROW 1 · i = 1
(1,0) i>j  → 8 - (1^0) = 8 - 1 = 7
(1,1) i==j → 4
(1,2) i<j  → 1^2 = 3
(1,3) i<j  → 1^3 = 2
→ row 1 = [7, 4, 3, 2]
// ROW 2 · i = 2
(2,0) i>j  → 8 - (2^0) = 8 - 2 = 6
(2,1) i>j  → 8 - (2^1) = 8 - 3 = 5
(2,2) i==j → 4
(2,3) i<j  → 2^3 = 1
→ row 2 = [6, 5, 4, 1]
// ROW 3 · i = 3
(3,0) i>j  → 8 - (3^0) = 8 - 3 = 5
(3,1) i>j  → 8 - (3^1) = 8 - 2 = 6
(3,2) i>j  → 8 - (3^2) = 8 - 1 = 7
(3,3) i==j → 4
→ row 3 = [5, 6, 7, 4]
// FINAL · A(4)
4 1 2 3
7 4 3 2
6 5 4 1
5 6 7 4

驗證每個十字（row i ∪ col i，去重後）：
i=0: {4,1,2,3} ∪ {4,7,6,5} = {1,2,3,4,5,6,7} ✓
i=1: {7,4,3,2} ∪ {1,4,5,6} = {1,2,3,4,5,6,7} ✓
i=2: {6,5,4,1} ∪ {2,3,4,7} = {1,2,3,4,5,6,7} ✓
i=3: {5,6,7,4} ∪ {3,2,1,4} = {1,2,3,4,5,6,7} ✓
// OUTPUT · row-major Report
Report(4); Report(1); Report(2); Report(3);
Report(7); Report(4); Report(3); Report(2);
Report(6); Report(5); Report(4); Report(1);
Report(5); Report(6); Report(7); Report(4);
觀察：每組互補對（和為 8）的兩半永遠分居對角線兩側 — (0,1)=1 與 (1,0)=7、(0,2)=2 與 (2,0)=6…。 對角的 4 是所有十字的共用中心；其餘三組互補對各被每個十字碰一次，剛好湊出全部七個數。

應用題// Quick Calculations

App 1 直接查詢

不展開整個 A(8)，直接算出 A(8)[1][6] 與 A(8)[6][1] 各是多少？兩者相加應該是？

App 2 記憶體比較

XOR 閉式解相對「2×2 block 遞迴版」省下哪一項資源？N = 1024 時具體省多少？兩版的時間複雜度相同嗎？

App 3 遷移思辨

若 N 不是 2 的次方（譬如 N = 6），i XOR j 構造為何失效？這時 1-因子分解還存在嗎？該用什麼方法？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、A 對應 A。

A1 · 容量觀察

格數 = 數量 ⇒ 一一對應。row i 有 N 格、col i 有 N 格，但對角格 (i, i) 同時屬於兩者、只算一次， 所以十字一共 N + N − 1 = 2N − 1 格。

而題目要求這些格的聯集等於 {1..2N−1} — 那是 2N − 1 個相異的數。 2N−1 個位置要裝下 2N−1 個相異值且全覆蓋 ⇒ 由鴿籠原理，每個值恰好佔一格、不重不漏。

這個觀察是整題的起點：它把「填陣列」變成「每個十字是 1..2N−1 的一個排列」， 後面才有空間去談配對與著色。

A2 · XOR 雙射

固定 i，函數 f(j) = i XOR j 是自己的反函數（f(f(j)) = i^(i^j) = j）， 所以是雙射。當 j 跑遍 [0, N)，i^j 也跑遍 [0, N) 每個值各一次。

排除 j = i 那一格（i^i = 0，對應對角，不參與十字的非對角配對）， 剩下的 i^j 恰好是 {1..N−1} 每個各一次 ⇒ row i 的非對角格碰到每組互補對編號各一次。 

// 為何需要 N = 2^K
i^j 落在 [0, N) ⟺ i, j 的二進位都在 K = log2(N) 位內
N = 2^K ⇒ [0, N) = 所有 K-bit 數 ⇒ XOR 封閉，i^j 一定 < N

若 N 不是 2 次方（如 N=6, 二進位有 3 bit 但 6,7 越界）：
   i^j 可能 = 7 ≥ 6 ⇒ 顏色越界、雙射壞掉
A3 · 1-因子分解

把 index 0..N−1 當圖的點，每條邊 {i, j} 染上顏色 i XOR j。 固定一個顏色 k ≠ 0，看「所有顏色 = k 的邊」：

  • 邊 {i, j} 是 k 色 ⟺ i XOR j = k ⟺ j = i XOR k。
  • 所以 k 色的邊就是把每個點 i 連到 i XOR k。
  • 因為 (i XOR k) XOR k = i，i 與 i XOR k 互相配對，且 i ≠ i XOR k（k≠0）⇒ 無自環、無重疊。
// PERFECT MATCHING for color k
i  ──k──  i XOR k     對所有 i 成立，兩兩配對覆蓋全部 N 個點
⇒ 顏色 k 的邊集 = 一個 1-因子（perfect matching）

N−1 個非零顏色 ⇒ K_N 被切成 N−1 個完美配對 = 1-因子分解

每個點都恰好碰到每種顏色一次 ⇒ 每個十字碰每組互補對一次 ⇒ 不重不漏。

A4 · 方向不變量

互補對 {k, 2N−k} 有兩個值，要分給對稱的兩格 (i,j) 與 (j,i)。 用 i<j / i>j 當分界，保證一格拿小半 i^j、另一格拿大半 2N−(i^j)： 

// INVARIANT
a[i][j] + a[j][i] = (i^j) + (2N - (i^j)) = 2N   ✓ 永遠成立

若兩格都填 i^j（小半）：

  • 互補對的大半 2N−(i^j) 永遠不出現在陣列裡。
  • 十字 i 只收得到 {1..N−1} 那一側的數 + 對角 N，完全缺 {N+1..2N−1}。
  • 違反 row i ∪ col i = {1..2N−1}。

方向是讓「互補對的兩半都現身」的唯一機制 — 缺了它，一半的值域整個消失。

A5 · 對角選值

對角線是十字的共用中心，所有十字都會碰到自己那一格的對角值。 所以對角必須放一個「對每個十字都合法」的常數。

選 N 是因為剩下的 2N−2 個數 {1..N−1} ∪ {N+1..2N−1} 剛好能配成 N−1 組「和為 2N」的互補對 — N 正好是這個對稱中心，互補對才漂亮。

  • 換成別的常數 v：剩下的數變成 {1..2N−1} \ {v}，要配「和為某定值」的互補對就不對稱了。
  • 譬如 v = 1：剩 {2..2N−1}，2N−1 個數扣掉 1 是偶數個，但配對中心不再是整數 ⇒ 構造得重新設計。
  • v = N 是唯一讓「互補對和為 2N、且 XOR 方向乾淨對接」的選擇。

對角 = N 不是隨便挑的，是讓整個互補對結構對稱的幾何中心。

A · App 1

N = 8，2N = 16。 

// A(8)[1][6] 與 A(8)[6][1]
(1, 6): i < j ⇒ 1 ^ 6 = 0b001 ^ 0b110 = 0b111 = 7
(6, 1): i > j ⇒ 16 - (6 ^ 1) = 16 - 7 = 9

驗證互補：7 + 9 = 16 = 2N  ✓

不需展開 64 格 — 看 (i, j) 的大小定方向、算一次 XOR 就 O(1) 查任意格的值。 對稱兩格相加必為 2N = 16。

A · App 2

省下的是「空間」。 

// N = 1024 資源對照
2×2 block 遞迴版：
   Space = int M[1100][1100] ≈ 4.8 MB（必須全域存整個矩陣）
   Time  = O(N²) ≈ 10⁶

XOR 閉式解：
   Space = O(1)（不存陣列，邊算邊 Report）  ← 省掉 4.8 MB
   Time  = O(N²) ≈ 10⁶  ← 與遞迴版同階

時間複雜度兩版相同（都是 O(N²)，受輸出量下限）， 但 XOR 版把空間從 O(N²) 降到 O(1)，且程式碼短到只剩一個雙層迴圈、沒有遞迴堆疊。

A · App 3

若 N 不是 2 的次方，i XOR j 可能 ≥ N（XOR 在 [0, N) 上不封閉）， 顏色越界、雙射壞掉 ⇒ XOR 構造失效。

  • 1-因子分解仍然存在：只要 N 是偶數，K_N 一定有 1-因子分解（這是經典定理）。
  • 此時要用 round-robin / 圓圈法：固定一個點在圓心，其餘 N−1 個點繞圈，每輪旋轉得到一個完美配對。程式碼要轉指標、取模，比 XOR 瑣碎。
  • N 為奇數時 K_N 沒有完美配對（點數奇），這題的「對角 + 互補對」框架也得改 — 但本題保證 N = 2^K，永遠偶數，不必煩惱。

一句話：XOR 是「N = 2^K」的專屬捷徑；一般偶數退回 round-robin，奇數則整個結構失效。 題目限制 N = 2^K 正是為了讓 XOR 這條最乾淨的路成立。
