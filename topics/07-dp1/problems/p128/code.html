01

完整實作 · dp[i] = max(dp[i−1], dp[i−K]+a[i])// Full C++

和「取數字1」的滾動版不同，這裡「取」要接 dp[i−K]（往回 K 格），K 不固定， 所以最直接就是留整條 dp[]、用 1-indexed 讓 dp[0]=0 自然當 base。

p128.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1005;
long long a[MAXN];    // a[1..n] 陣列元素
long long dp[MAXN];   // dp[i] = 只看 a[1..i] 的最大合法和

void solve() {
    int n, k;
    cin >> n >> k;
    for (int i = 1; i <= n; i++) cin >> a[i];

    dp[0] = 0;                         // 空集合
    for (int i = 1; i <= n; i++) {
        long long prev = (i - k >= 0) ? dp[i - k] : 0;  // 取 a[i] ⇒ 接前 K 格
        long long take = prev + a[i];  // 取這格：往回跳 K 格保證距離 ≥ K
        long long skip = dp[i - 1];    // 不取這格：沿用上一格最佳
        dp[i] = max(skip, take);
    }

    cout << dp[n] << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) solve();

    return 0;
}

為什麼 1-indexed？ 讓 dp[0]=0 直接當「空前綴」的 base — 當 i ≤ K（取了這格、前面湊不到 K 格）時，i−k 可能為 0，dp[0]=0 正好代表「只取這一個」。 (i − k >= 0) 的判斷只在 k > i 時擋掉負索引（其餘情況一定成立）。

複雜度分析// Complexity Derivation

步驟內容成本
讀入讀 N 個 a[i]O(N)
轉移每個 i 做一次 max、一次加法、一次查 dp[i−K]O(1)
掃完整列N 個元素各一次O(N)
全部 T 筆—O(T · N)
每筆 O(N)，總 O(T · N)
T ≤ 10、N ≤ 1000 ⇒ 約 10⁴ 次運算，秒過
項目數量備註
TimeO(N) 每筆單遍掃描
SpaceO(N)查 dp[i−K] ⇒ 留整條 dp[]（可壓到 O(K) 環狀緩衝）
遞迴深度0（迭代）純迴圈
Constant factor低一次 max + 一次加法
03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
接 dp[i−K] 而非 dp[i−2] 這是和「取數字1」唯一的差別。距離 ≥ K ⇒ 上一個被取的 index ≤ i−K ⇒ 接 dp[i−K]。寫成 dp[i−2] 只在 K=2 時對。
i − K 可能 ≤ 0 當 K > i 時 i−K 為負；要回退到 dp[0]=0（等於只取這一個）。用 (i−k>=0)?dp[i−k]:0 守住。
不能用滾動兩變數 取數字1 只看前兩格可滾動；這裡 K 任意 ⇒ 至少要留 dp[i−K]。直接開整條 dp[] 最穩（也可 O(K) 環狀緩衝）。
K = 1（無限制） 取 a[i] 接 dp[i−1]，skip 也是 dp[i−1] ⇒ dp[i]=dp[i−1]+a[i] = 全取總和。轉移自動涵蓋，不必特判。
多筆要重置嗎？ 每筆都從 i=1 重填 dp[1..n]、且只讀到 dp[n]，不必清空（不會用到上一筆的殘值）。dp[0] 每筆設 0 即可。
總和型別 aᵢ ≤ 1000、N ≤ 1000 ⇒ 和最大 10⁶，int 綽綽有餘；仍用 long long 保險、習慣。
04

細節走讀// Section Walkthrough

① 狀態定義（1-indexed）

// state
dp[i] = 只看 a[1..i]、任兩個被取的距離 >= K 時的最大和
dp[0] = 0  // 空前綴，當 base
答案 = dp[n]

② 轉移（兩個選擇取大）

// transition
skip = dp[i-1]              // 不取 a[i] → 沿用上一格
take = dp[i-K] + a[i]       // 取 a[i] → 上一個被取的在 <= i-K
dp[i] = max(skip, take)
// i-K <= 0 時 take 接 dp[0]=0 → 等於「只取 a[i] 一個」

「距離 ≥ K」⇒ 若取了 a[i]，前一個被取的 index j 滿足 i − j ≥ K，即 j ≤ i − K， 其最佳和已濃縮在 dp[i−K]（dp 為前綴最大、單調不減）。

③ 與「取數字1」的對應

// equivalence
取數字1（不相鄰）= 距離 >= 2  → dp[i] = max(dp[i-1], dp[i-2] + a[i])
取數字2（距離 >= K）          → dp[i] = max(dp[i-1], dp[i-K] + a[i])
把「2」換成「K」即可，其餘完全相同
K=2 退化回取數字1；K=1 變成全取

ONE-LINE SUMMARY
留整條 dp[]：skip=dp[i−1]、take=dp[i−K]+a[i]，dp[i]=max 兩者，O(N)。
取就接 dp[i−K]（不是 dp[i−2]）；i−K≤0 回退 dp[0]=0。K=1 全取、K=2 退回取數字1。
