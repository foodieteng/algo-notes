[ 01 ] Problem

題意拆解

王老先生有一塊 N × N 的正方形土地，其中 N = 2ᴷ（K ≥ 1 的正整數）。 土地被切成 N × N 個邊長為 1 的小方格， 位置以 (i, j) 表示：i 為由上到下第 i 列、j 為由左至右第 j 行（皆從 1 開始）。

王老先生先挑一格 (X, Y) 作為自己的養老用地， 剩下 N² − 1 格要平分給 (N² − 1) / 3 個子孫。 每個子孫剛好分到 3 格相連，且必須排成 L 型（不能是 1 × 3 直線）。 任兩個子孫不得有重疊。

輸入 / 輸出

本題採 互動式評測。請實作以下函式（lib0114.h 已宣告）：

lib0114.h C++ 
void solve(int N, int X, int Y);  // 你要實作這個
void Report(int x1, int y1,
            int x2, int y2,
            int x3, int y3);      // 由 judge 提供，回報一個 L

solve() 內部對每個 L 呼叫一次 Report， 參數為該 L 的三個格子座標。呼叫順序與三格的順序皆不影響判定， 只要分配滿足規則即視為正確。

限制

  • 30% 資料：1 < N ≤ 8
  • 100% 資料：1 < N ≤ 1024，1 ≤ X, Y ≤ N
  • N 保證為 2 的正整數次方，存在正整數 K 使 N = 2ᴷ

範例

judge 呼叫 solve(4, 3, 4)，你的程式回報 5 次 Report：

// JUDGE
solve(4, 3, 4);
// YOUR REPORTS
Report(2,1, 1,2, 1,1); Report(1,3, 1,4, 2,4); Report(3,3, 4,3, 4,4); Report(3,1, 4,1, 4,2); Report(3,2, 2,2, 2,3);
N = 4、特殊格 (3, 4) 共需 (16 − 1) / 3 = 5 個 L-tromino。 上面的分配方案在下面的動畫區可以一步一步看到。
[ 02 ] Concept

演算法概念

觀察一：數量必為整數

總非特殊格數 = N² − 1 = 4ᴷ − 1。 因為 4 ≡ 1 (mod 3)， 所以 4ᴷ ≡ 1 (mod 3)，即 4ᴷ − 1 ≡ 0 (mod 3)。 L 數量 (N² − 1) / 3 一定是整數， 這暗示分配方案有解 — 接下來用 構造法 直接寫出來。

觀察二：分治

將 N × N 板切成 4 個 N/2 × N/2 象限 TL、TR、BL、BR。 特殊格 (X, Y) 必然落在恰好一個象限裡。 關鍵步驟：在棋盤正中央放一個 L， 讓 L 的 3 個 cell 分別佔住另外 3 個象限最靠近中心的那一個角。 這樣每個象限都變成「N/2 × N/2 板 + 1 個 special 格」， 就是同樣的子問題 — 遞迴解之。

T(N) = 4 · T(N/2) + O(1)   ⇒   T(N) = Θ(N²)

Base Case

N = 2 時，2×2 板共 4 格，扣掉 special 剩 3 格 — 這 3 格本身就是 L，直接 Report 即可。

遞迴正確性

  • 每個子問題的形狀仍是 2ᴷ × 2ᴷ + 1 個 special。
  • Base case 直接可解。
  • 遞迴步驟「在中央放 1 個 L」一定不會與後續放的 L 重疊，因為這 L 的 3 個 cell 被各自象限「吃掉」當 special。
  • 歸納成立 ⇒ 演算法正確。

複雜度

Time O(N²) Space O(log N)

時間：每呼叫一次 Report 是 O(1)，總共 (N²−1)/3 次 ⇒ Θ(N²)。 空間：遞迴深度 log₂ N = K。對 N = 1024 來說只有 10 層，棧很淺。

[ 04 ] Clean Code

Clean Code

p114.cpp C++ 
#include <iostream>
#include "lib0114.h"
using namespace std;

// 在以 (r, c) 為左上角、邊長為 n 的子方格內鋪 L-tromino，
// special cell 位於 (sx, sy)。
void tile(int r, int c, int n, int sx, int sy) {
    if (n == 2) {
        // 4 格中除 special 外的 3 格本身就是一個 L
        int xs[3], ys[3], k = 0;
        for (int dr = 0; dr < 2; ++dr) {
            for (int dc = 0; dc < 2; ++dc) {
                int x = r + dr, y = c + dc;
                if (x == sx && y == sy) continue;
                xs[k] = x; ys[k] = y; ++k;
            }
        }
        Report(xs[0], ys[0], xs[1], ys[1], xs[2], ys[2]);
        return;
    }

    int h  = n / 2;
    int mr = r + h - 1;     // 上半最後一列
    int mc = c + h - 1;     // 左半最後一行

    // 4 個象限「最靠近中心」的格子座標
    int qx[4] = { mr, mr,     mr + 1, mr + 1 };
    int qy[4] = { mc, mc + 1, mc,     mc + 1 };

    // special 落在哪個象限？  0=TL, 1=TR, 2=BL, 3=BR
    int sq = (sx > mr) * 2 + (sy > mc);

    // 在中央放 L：3 cell = 不含 special 的 3 個象限的最角落
    int rx[3], ry[3], k = 0;
    for (int q = 0; q < 4; ++q) {
        if (q == sq) continue;
        rx[k] = qx[q]; ry[k] = qy[q]; ++k;
    }
    Report(rx[0], ry[0], rx[1], ry[1], rx[2], ry[2]);

    // 對 4 象限分別遞迴；非 special 象限以「剛放下的角落格」當新 special
    int sr[4] = { r,     r,     r + h, r + h };
    int sc[4] = { c,     c + h, c,     c + h };
    for (int q = 0; q < 4; ++q) {
        int nx = (q == sq) ? sx : qx[q];
        int ny = (q == sq) ? sy : qy[q];
        tile(sr[q], sc[q], h, nx, ny);
    }
}

void solve(int N, int X, int Y) {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    tile(1, 1, N, X, Y);
}
[ 05 ] Walkthrough

Code 走讀

核心是 tile(r, c, n, sx, sy)：對一個左上角 (r,c)、邊長 n、缺一格 (sx,sy) 的子板鋪 L。

if (n == 2) { /* 4 格扣掉 special 後直接 Report */ }
Base case。 2×2 板只有 4 格，扣掉 special 後剩 3 格必然構成 L（除非 special 不在這 2×2 內，但遞迴保證它在）。 直接掃 4 格，跳過 special，剩 3 格送進 Report。
int mr = r + h - 1; int mc = c + h - 1; int qx[4] = { mr, mr, mr+1, mr+1 }; int qy[4] = { mc, mc + 1, mc, mc+1 };
中央 4 角座標。 每個象限「最靠近中心」的格子，依 TL/TR/BL/BR 順序排： (mr,mc)、(mr,mc+1)、(mr+1,mc)、(mr+1,mc+1)。 這 4 格就是放置中央 L 的候選。
int sq = (sx > mr) * 2 + (sy > mc);
判斷 special 在哪個象限。 把 2 個布林值編成 0~3：
· TL = 0：sx ≤ mr 且 sy ≤ mc
· TR = 1：sx ≤ mr 且 sy > mc
· BL = 2：sx > mr 且 sy ≤ mc
· BR = 3：sx > mr 且 sy > mc
for (int q = 0; q < 4; ++q) { if (q == sq) continue; rx[k] = qx[q]; ry[k] = qy[q]; ++k; } Report(rx[0], ry[0], rx[1], ry[1], rx[2], ry[2]);
放中央 L。 收集「不含 special」3 個象限的最角落格，組成 L 直接 Report。 注意這 3 格之後在各自象限遞迴時會被當成 special — 它們已經被「占用」掉。
for (int q = 0; q < 4; ++q) { int nx = (q == sq) ? sx : qx[q]; int ny = (q == sq) ? sy : qy[q]; tile(sr[q], sc[q], h, nx, ny); }
遞迴 4 個象限。 special 落在哪個象限，那象限就用原 special遞迴； 其他 3 個象限則用剛放下的角落格當 new special。 這保證所有 4 個子問題都是「方板 + 1 special」的同形式。
// Common Pitfalls
  • 座標 1-indexed：題目給 (X, Y) 從 1 開始，tile 的初始呼叫要傳 tile(1, 1, N, X, Y)。
  • mr = r + h − 1：是上半的最後一列，不是中線；下半從 mr + 1 開始。寫成 r + h/2 是錯的（因為這層的 n 已經是 h 的 2 倍）。
  • Base case 不需要再判斷 sq — n=2 時直接 4 格枚舉就好。
  • 本題不需 cin/cout I/O，ios::sync_with_stdio 純粹是習慣性放著（若有額外讀檔再用）。
[ 06 ] Variants

延伸 / 變形題

變形差異解法概念
非 2ᴷ 邊長的 deficient board 例如 N×N 的 N 不是 2 的次方 不一定可解；需檢查 (N²−1) ≡ 0 (mod 3) 的必要條件並做 case analysis
輸出全部 L 的編號 / 顏色 要區分哪格屬於哪個 L tile 中加一個 id 計數器，Report 改成把同一 L 標同色寫進二維陣列
非互動版本（直接輸出） 沒有 Report 函式，要 cout 輸出全部 L 把 Report 換成 printf，記得 ios::sync_with_stdio(false) 加速
分治經典題 — 逆序數對 同章 W06 / P115 Merge Sort 過程順便算跨段逆序，T(N) = 2T(N/2) + O(N) = O(N log N)
