重點回顧// Concept Recap

關鍵點內容
核心對應 「加入城市 k」= Floyd 的「第 k 層 pivot（允許用 k 當中繼）」。
Floyd 不變量 第 k 層後 dist[i][j] = 只用編號 ≤ k 的點當中繼的最短距離。
慘字度 當前所有點對最短路的 max；有 ∞ ⇒ -1。
步驟 2 不可省 先用舊城市修好 k 的行/列，否則漏算進出 k 的多跳捷徑。
−1 語意 輸入 −1 = 沒有直接道路 ⇒ 轉成 INF。
複雜度 每層 O(k²)，加總 O(N³)，與一次完整 Floyd 同量級。
02

觀念題// Concept Questions

Q1 核心對應

為什麼「加入城市 k」可以對應到 Floyd 最外層 for k 的「第 k 層」？兩者的意義怎麼吻合？

Q2 關鍵步驟

程式裡「先用之前的城市修正 dist[i][k]、dist[k][j]」那一步如果省掉，會發生什麼？哪個樣本值會錯？

Q3 不變量

Floyd 第 k 層跑完，dist[i][j] 的精確含義是什麼？為什麼「經過 k 一次」就涵蓋所有新最短路？

Q4 −1 處理

輸入的 -1 代表什麼？如果直接把它當成距離 −1 拿去 min，會出什麼錯？

Q5 複雜度

為什麼總複雜度是 O(N³) 而不是 O(N⁴)？跟「每加一點重跑整次 Floyd」差在哪？

範例 Trace · 跑一次 code 看會輸出什麼// Worked Trace

用官方樣本（N=5）。下面是每加入一城後的 dist 矩陣（只列已加入的 {1..k}）。

// INPUT5
 0  1 -1  2  3
 3  0 -1  4  2
-1 -1  0  9  8
 1  3  8  0  9
 5  4  3  8  0
// 加入城市 1
dist = [ 0 ]
慘字度 = max = 0
// 加入城市 2
dist = [ 0  1 ]
       [ 3  0 ]
慘字度 = max(1, 3) = 3
// 加入城市 3
dist = [ 0    1   INF ]
       [ 3    0   INF ]
       [ INF INF   0  ]
3 進不去、也出不來（c[*][3]、c[3][1..2] 全 -1）⇒ 有 INF
慘字度 = -1  （大慘字規劃）
// 加入城市 4
dist = [  0   1  10   2 ]
       [  3   0  12   4 ]
       [ 10  11   0   9 ]
       [  1   2   8   0 ]
4 把 3 接通了（4↔3 有邊）。最大是 dist[2][3] = 12
慘字度 = 12
// 加入城市 5
dist = [  0   1   6   2   3 ]
       [  3   0   5   4   2 ]
       [ 10  11   0   9   8 ]
       [  1   2   7   0   4 ]
       [  5   4   3   7   0 ]
5 當中繼把 dist[2][3] 從 12 壓到 5（2→5→3 = 2+3）
新的最大變成 dist[3][2] = 11
慘字度 = 11
// OUTPUT0 3 -1 12 11
觀察：加入城市 5 後，它當中繼幫 2→3 抄了捷徑（12 → 5），整體最大值反而下降 （12 → 11）。慘字度不是單調遞增的 —— 新城市可能讓既有路徑變短、規劃變好。

應用題// Quick Calculations

App 1 手算

承上，為什麼加入城市 5 後 dist[2][3] 變成 5？寫出那條最短路徑。

App 2 單調性

慘字度序列會不會一定遞增？用本範例 12 → 11 說明，並解釋為什麼。

App 3 變形

若改成「移除」城市（從 N 倒著刪），還能用增量 Floyd 嗎？該怎麼想？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

A1 · 核心對應

Floyd 最外層 for k 的第 k 圈，意思是「現在允許用編號 k 的點當中繼， 重新鬆弛所有 dist[i][j]」。題目「加入城市 k」恰好就是「讓 k 變成可用的中繼點」。

兩者語意完全一致 ⇒ 把 Floyd 的三層迴圈攤開，按 k=1..N 一層一層跑， 每跑完一層就是「加完城市 k 後」的全點對最短路，順手掃 max 即得慘字度。

A2 · 關鍵步驟

會少算進出 k 的多跳捷徑。只填 k 的直接邊、馬上以 k 當中繼鬆弛， 等於假設「到 k / 從 k」只能走一條直接道路，忽略「i → 某舊城市 m → k」這種更短的走法。 

// 省掉步驟 2 的後果
樣本最後一項：正解 11，少了步驟 2 會算成 12
（因為某些 dist[i][k] 沒被舊城市 m 修短）

所以一定要先用 m ∈ {1..k−1} 修好 dist[i][k]、dist[k][j]，再以 k 鬆弛全對。

A3 · 不變量

第 k 層跑完後，dist[i][j] = 「i 到 j、只允許用編號 ≤ k 的點當中繼」的最短距離。

新增 k 後的任何最短路，若用到 k，可拆成 i ⇝ k 與 k ⇝ j 兩段。 只需經過 k 一次：邊權非負 ⇒ 經過 k 兩次代表繞了一個含 k 的環，不會更短。 所以 min(dist[i][j], dist[i][k] + dist[k][j]) 就完整涵蓋所有「用 k」的新最短路。

A4 · −1 處理

輸入 -1 代表 i 到 j 沒有直接道路（不是距離 −1）。要轉成 INF 表示「目前不可達」。

若直接把 −1 當成距離拿去 min，會得到負的「最短路」、甚至製造負權環， 整個 Floyd 崩潰、答案全錯。正確做法：讀入時 c = (x==-1 ? INF : x)。

A5 · 複雜度

增量版只把 Floyd 的三層迴圈攤開一次：外層 k 跑 N 次，每層做 O(k²) 的鬆弛 + 掃 max， 加總 Σ k² = O(N³)。

「每加一點重跑整次 Floyd」則是每個前綴都重跑一次 O(k³)， 加總 Σ k³ = O(N⁴)。N=500 時 N⁴ = 6×10¹⁰ 會 TLE，N³ = 1.25×10⁸ 則秒過。 差別就在「複用上一層的結果」而非從頭重算。

A · App 1

加入城市 5 前，dist[2][3] = 12（要繞 2→4→3 = 4+8）。加入 5 後，5 當中繼提供捷徑： 

// 2 → 5 → 3
dist[2][5] = 2  (2→5 直接邊)
dist[5][3] = 3  (5→3 直接邊)
dist[2][3] = min(12, 2 + 3) = 5

路徑是 2 → 5 → 3，長度 5。

A · App 2

不一定遞增。本範例第 4 項 12、第 5 項 11，就降下來了。

原因：新城市除了「自己跟別人的距離」會被加進來（可能拉高 max），也能當中繼把既有點對的最短路變短。 當「縮短既有最大那條」的效果勝過「新增的點對」時，慘字度就下降。所以序列可升可降，不能假設單調。

A · App 3

可以，但要倒著想。刪除城市無法直接「反做」一層 Floyd（中繼的影響已散佈到整個矩陣，難以撤銷）。

標準技巧是離線、把刪除反轉成加入：把「依序刪 N, N−1, …」倒過來看成「依序加 1, 2, …」， 正好就是本題的增量 Floyd，再把答案反序輸出即可。「刪除 → 反轉成加入」是這類問題的通用套路。
