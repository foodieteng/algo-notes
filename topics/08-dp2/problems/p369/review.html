01

重點回顧// Concept Recap

關鍵點內容
問題本質 數樓層序列：每步 |y−x| < |B−x|，求長度 K 的序列數。
換軸 跨不過 B ⇒ 只留 S 那一側，狀態改成到 B 的距離 a=|x−B| ∈ [1,L]。
狀態 f[a] = 搭到目前、停在距離 a 的方法數。初值 f[a0]=1。
轉移 nf[a'] = suf[amin] − f[a']，amin=⌈(a'+1)/2⌉（收一段後綴）。
加速 每個 a' 收一段後綴 ⇒ 後綴和，每趟 O(L)。
細節 扣 f[a'] 自身、減法補 MOD、答案 Σ f[a]、K=0/S=B 特判。
複雜度 O(L·K) ≤ O(N·K)；MOD 1e9+7（質數，但只用加減）。
02

觀念題// Concept Questions

Q1 換軸

為什麼可以只保留 S 那一側、把狀態從「樓層」換成「到 B 的距離 a」？另一側發生了什麼？

Q2 門檻推導

從距離 a 出發，合法落點 a' 的範圍是什麼？反過來，目標 a' 能收到哪些來源 a？門檻 amin 怎麼來的？

Q3 為何後綴和

為什麼這版用「後綴和」而不是差分？兩者各 O(L)/趟，差別在哪？

Q4 扣自身

為什麼算完 suf[amin] 還要 − f[a']？不扣會多算什麼？

Q5 答案位置

為什麼答案是 Σ f[a] 而不是某一個 f[?]？K=0 與 S=B 兩個特判分別回什麼？

03

範例 Trace · N=8, S=2, B=5, K=2// Worked Trace

S=2 在 B=5 下側 ⇒ 只留下側 L=B−1=4 層，距離軸 a=1..4（a=1 最靠 B、a=4 最遠）。 起點距離 a0=B−S=3。amin=⌈(a'+1)/2⌉：a'=1→1, a'=2→2, a'=3→2, a'=4→3。

// INPUT8 2 5 2     // N=8, S=2, B=5, K=2
// init (s=0)
距離 a :  1    2    3    4         （a=|樓層−5|；a=3 ⟺ 樓層 2）
f[a]   :  0    0    1    0         ← 起點 a0=3 有 1 種
// 第 1 趟：suf = 後綴和 of [0,0,1,0]
suf[4]=0  suf[3]=1  suf[2]=1  suf[1]=1
nf[1] = suf[amin=1] − f[1] = 1 − 0 = 1
nf[2] = suf[amin=2] − f[2] = 1 − 0 = 1
nf[3] = suf[amin=2] − f[3] = 1 − 1 = 0   ← 唯一來源是自己，扣掉 → 0
nf[4] = suf[amin=3] − f[4] = 1 − 0 = 1
f[a]   :  1    1    0    1
// 第 2 趟：suf = 後綴和 of [1,1,0,1]
suf[4]=1  suf[3]=1  suf[2]=2  suf[1]=3
nf[1] = suf[1] − f[1] = 3 − 1 = 2
nf[2] = suf[2] − f[2] = 2 − 1 = 1
nf[3] = suf[2] − f[3] = 2 − 0 = 2
nf[4] = suf[3] − f[4] = 1 − 1 = 0   ← 唯一來源是自己，扣掉 → 0
f[a]   :  2    1    2    0
// OUTPUTΣ f[a] = 2 + 1 + 2 + 0 = 5
觀察：第 2 趟的 a'=4（離 B 最遠的那格）門檻 amin=3，後綴只剩 {3,4}， 而其中能貢獻的只有它自己 ⇒ 扣掉後變 0 — 離 B 越遠、要從更遠處跳來，來源越稀少。最終 5 種搭法，和差分版完全一致。
04

應用題// Quick Calculations

App 1 為何 0

範例2 N=7, S=1, B=2, K=2 答案是 0。用距離軸解釋：L、a0 各是多少？為什麼第一趟就卡住？

App 2 整數門檻

為什麼程式裡 amin=(a'+2)/2（整數除法）剛好等於 ⌈(a'+1)/2⌉？寫成 (a'+1)/2 會錯在哪？

App 3 兩版對照

差分版（樓層軸、區間加）和這版（距離軸、後綴和）哪裡本質相同、哪裡不同？為什麼距離版常數更小？

Q1 · 換軸

條件 |y−x| < |B−x| 保證每趟搭乘距離 < 到 B 的距離 ⇒ 永遠跳不到也跳不過 B。 所以從 S 出發後，你會永遠待在 B 的同一側，另一側的樓層完全到不了，根本不用建。

把那一側的樓層用「到 B 的距離 a=|x−B|」重新編號（a=1 緊鄰 B、a=L 最遠），距離成了唯一有意義的狀態， 陣列長度從 N 縮成單側 L。

Q2 · 門檻推導

同側兩點距離 a, a'，搭乘距離 |y−x| = |a−a'|，到 B 距離 |B−x| = a。 合法 |a−a'| < a ⟺ 0 < a' < 2a ⟺ 從 a 能到 a' ∈ [1, 2a−1]。

反問「目標 a' 收得到哪些來源 a」：需 a' ≤ 2a−1 ⟺ a ≥ (a'+1)/2 ⟺ a ≥ ⌈(a'+1)/2⌉ = amin。所以 a' 收下的是 f[] 的後綴 [amin, L]。

Q3 · 為何後綴和

兩版本質相同：都把「每步逐對轉移 O(L²)」壓成 O(L)。差別在查詢形狀：

  • 差分版（樓層軸）：每個來源 x 把值灑到一段區間 [x−d+1, x+d−1] ⇒ 用差分做區間加，最後前綴和還原。
  • 後綴和版（距離軸）：換軸後每個目標 a' 收的是一段後綴 [amin, L] ⇒ 直接後綴和，連差分都不必。

距離版只掃單側、陣列短、運算單純（一次後綴和 + 一輪相減），常數更小也更好寫。

Q4 · 扣自身

後綴 [amin, L] 一定包含 a'=a 自己（因為合法範圍 a'∈[1,2a−1]，當 a=a' 也成立）， 但那代表「停在原地、距離不變」，而電梯必須移動（y≠x）。

所以 nf[a'] = suf[amin] − f[a'] 把這份「停原地」的貢獻扣掉。不扣的話每趟都會灌進假搭法。

Q5 · 答案位置

題目只要「連續搭 K 趟」，沒規定終點 ⇒ 停在任何距離都算一種，答案 Σ_a f[a]。

兩個特判：K=0 ⇒ 沒搭電梯，序列只有起點本身，答案 1； S=B ⇒ 起點就在禁止層（題目其實保證 S≠B，但防呆），答案 0。

App 1 · 為何 0

S=1 < B=2 ⇒ 下側 L=B−1=1，a0=B−S=1。整個距離軸只有一格 a=1。

從 a=1 出發，合法目標 a'∈[1, 2·1−1]=[1,1]＝只有自己；扣掉停原地後沒有任何合法落點 ⇒ 第一趟就搭不出去 ⇒ 0 種。直覺：你緊貼禁止層（距離 1），任何移動都會碰到或越過它。

App 2 · 整數門檻

要的是 ⌈(a'+1)/2⌉。對整數除法（向下取整）有恆等式 ⌈p/2⌉ = ⌊(p+1)/2⌋，代入 p=a'+1 得 ⌈(a'+1)/2⌉ = ⌊(a'+2)/2⌋ = (a'+2)/2（整數除）。

若寫成 (a'+1)/2（=⌊(a'+1)/2⌋），對偶數 a' 會少 1（門檻太低）， 把「a 還不夠大、跳不到 a'」的來源也算進去 ⇒ 多算、答案偏大。例如 a'=2 應為 amin=2，寫錯會變 1。

App 3 · 兩版對照

本質相同：都是計數 DP，逐趟把方法數從「能跳過來的來源」聚到每個目標，並用前綴/後綴技巧把整趟降到 O(L)； 兩版在所有測資答案一致（範例 5）。

不同：差分版留在樓層軸、長度 N、合法集合是區間（要差分 + 前綴和兩步）； 距離版換軸後長度只剩單側 L、合法集合是後綴（一次後綴和即可）。少了另一側、少了差分這層間接， 距離版常數更小、程式更短，是這題更乾淨的寫法。
