好的序列

01

題目摘要// Problem Brief

QUESTION

給定正整數 N，輸出一個 1..N 的排列 a[1..N]，使得 不存在任意三個下標 i < j < k 滿足 2·a[j] = a[i] + a[k]（即 a[i], a[j], a[k] 在原順序下構成等差數列，簡稱 3-AP）。 存在多種合法答案時輸出任一即可。

下面用 n = 3 的所有排列做暖身（共 6 種）— 只有「全升」與「全降」是 3-AP，其他 4 種都合法：

排列 是否 3-AP? 收錄?
(1, 2, 3)是 — (1,2,3) 等差 d = +1✗
(1, 3, 2)無 ✓
(2, 1, 3)無 ✓
(2, 3, 1)無 ✓
(3, 1, 2)無 ✓
(3, 2, 1)是 — (3,2,1) 等差 d = −1✗
n = 3 共 6 排列、4 個合法。本題只要輸出其中一個。
02

為什麼要「構造」 + 切一半// Why Constructive Split

A暴力指數爆炸
枚舉所有 N! 種排列再驗證，N = 10 就 3,628,800 個，N = 15 就 10¹² 級別 — 完全不可行。
BDP 狀態爆炸
若用「已選哪幾個」當狀態，O(2ᴺ · N)，N > 24 即超量。本題沒有「最佳解」可被 DP 累積。
C直接「造」
設計一條保證合法的構造規則 —「奇偶切」+ 遞迴。O(N log N) 直接生出答案，不需要驗證。
03

核心策略 — 奇偶切// Odd-Even Split

把全部數字按奇偶分兩組：所有奇數放前段、所有偶數放後段。 每一段內部再遞迴解一次相同問題即可。下面以 N = 8 為例：

1 5 3 7 2 6 4 8
奇 (1,3,5,7) 偶 (2,4,6,8)

奇數段內部由 good(⌈N/2⌉) 遞迴決定順序、偶數段由 good(⌊N/2⌋) 遞迴決定。 上面是 N=8 的最終結果（與下面 Section 06 的 trace 推導一致）。

04

為什麼這樣切就一定 3-AP free?// Proof Sketch

假設我們找到一組逆例 (i, j, k)，使得 2·a[j] = a[i] + a[k]。 觀察右式 a[i] + a[k] 必為偶數（因為左式是 2 倍）， 所以 a[i] 與 a[k] 同奇偶。分以下三種情況：

假設 ① a[i], a[k] 都是奇數 ⇒ 兩者都在前段。 因為前段在後段之前，位置上 i < k 都在前段，所以 j（介於兩者）也必在前段 ⇒ a[j] 也是奇數。整組 3-AP 都落在前段，回到「對 ⌈N/2⌉ 個奇數做同一問題」— 由歸納假設不存在。
假設 ② a[i], a[k] 都是偶數 ⇒ 兩者都在後段。 同理 j 也在後段 ⇒ a[j] 也是偶數。整組落在後段，由歸納假設不存在。
假設 ③ a[i], a[k] 一奇一偶 — 違反「同奇偶」的必要條件，不可能發生。
奇偶律 · PARITY
奇 + 奇 = 偶  ·  奇 + 偶 = 奇  ·  偶 + 偶 = 偶
3-AP 中間元素是兩端的平均 ⇒ 兩端同奇偶 ⇒ 跨段不可能。

遞迴結構 · 公式// Recurrence

f(n) = [ 2 · f(⌈n/2⌉) − 1 ]  ∥  [ 2 · f(⌊n/2⌋) ]
∥ 為串接；2x − 1 把長度 ⌈n/2⌉ 的排列映成 奇數段；2x 映成 偶數段

Base case：f(1) = [1]。當輸入縮到單一元素時直接收尾。

T(n) = 2 · T(n/2) + Θ(n)  ⇒  T(n) = Θ(n log n)

08

一句洞察// One-Sentence Insight

奇偶切的真正力量：3-AP 的中項是兩端的平均 — 而平均要存在於整數空間， 兩端必須同奇偶。把奇偶物理分離後，任何跨段的 3-AP 中項要不在奇段、要不在偶段， 無論如何都會違反「中項位置在兩端之間」的位置條件，所以跨段一定零個 3-AP。 只剩同段內部的 3-AP — 而那是規模一半的同一個問題，遞迴解掉。

程式碼參考// C++ reference

// good(n): 回傳長度 n、不含 3-AP 的 1..n 排列
#include <iostream>
#include <vector>
using namespace std;

vector<int> good(int n) {
    if (n == 1) return {1};    // base case

    int half = (n + 1) / 2;

    vector<int> left  = good(half);     // 奇數段 (⌈n/2⌉ 個)
    vector<int> right = good(n - half);  // 偶數段 (⌊n/2⌋ 個)

    vector<int> res;
    res.reserve(n);
    for (int x : left)  res.push_back(2 * x - 1);  // 映成奇數
    for (int x : right) res.push_back(2 * x);      // 映成偶數
    return res;
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);

    int n; cin >> n;
    auto seq = good(n);
    for (int i = 0; i < (int)seq.size(); ++i)
        cout << seq[i] << (i + 1 == (int)seq.size() ? '\n' : ' ');
}

複習 Q&A// Quick Review

問題 解答 / 重點
half 為何用 ⌈n/2⌉ 而非 ⌊n/2⌋？ 奇數段需要 ⌈n/2⌉ 個元素（1, 3, 5, …），偶數段才是 ⌊n/2⌋ 個。例如 n=5 時奇數有 3 個 (1,3,5)、偶數有 2 個 (2,4)。
跨段 3-AP 為什麼一定不存在？ 3-AP 兩端必同奇偶，但同奇偶的元素已被「物理分離」到同一段；跨段就違反兩端同奇偶條件。
能不能反過來偶數在前？ 可以。奇偶順序不影響正確性，整段交換對應一個鏡射版本仍合法。題目只要求一個合法答案。
時間 / 空間複雜度？ T(n) = 2·T(n/2) + Θ(n) = O(n log n) 時間（Master Theorem case 2）；遞迴深度 O(log n)、總空間 O(n)（最後一份排列）。
ONE-LINE SUMMARY
奇數放奇數，偶數放偶數，每段內遞迴 — 自動避開所有 3-AP。
f(n) = [ 2·f(⌈n/2⌉) − 1 ] ∥ [ 2·f(⌊n/2⌋) ]  ·  O(N log N)
