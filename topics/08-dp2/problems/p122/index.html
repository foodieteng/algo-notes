// 折半枚舉 Meet in the Middle · 子集和湊出 K

PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

有 N 件邪惡物品，第 i 件的邪惡指數為 A[i]（每件恰一個）。你想選出一個子集， 使被選物品的邪惡指數總和恰好等於目標邪惡度 K。題目保證有解，請回報任一組合法選法。

互動式：實作 void solve(int N, long long K, long long A[])， 用 Report(id)（1 ≤ id ≤ N）回報選中的物品，全部選完後呼叫 Report(-1) 結束。 回報順序不影響判定。

範圍：1 ≤ N ≤ 40，1 ≤ K ≤ 10¹⁸，ΣA[i] ≤ 10¹⁸。 白話：這就是子集和（subset-sum）= K 的存在 + 構造。N 最大 40，是「折半枚舉」的招牌尺寸。

02

為什麼用折半枚舉// Why Meet in the Middle

A 枚舉所有子集 O(2ᴺ)
2⁴⁰ ≈ 1.1×10¹² 個子集，逐一檢查 ⇒ 完全跑不完。
× REJECT · TLE
B 背包 DP（值當容量） O(N·K)
K 高達 10¹⁸，無法開 dp[K] 的表，也無法逐格轉移 ⇒ 記憶體 + 時間都爆。
△ REJECT · MLE
BOTH FAIL · ADOPT
C 折半枚舉 Meet in the Middle O(2^(N/2)·N)
把 N 件切兩半（各 ≤20）。左半枚舉 2²⁰ 個子集和並排序；右半每個子集和 s，二分搜左半是否有 K−s。2²⁰≈10⁶，可行。
● ADOPT · STABLE

核心觀察：任一合法選法的物品，必定「一部分在左半、一部分在右半」。 固定右半選法（和為 s）後，左半只要湊出 K − s 即可 ⇒ 兩半各自枚舉 2^(N/2)，再用排序 + 二分搜把兩半拼起來。 指數從 N 砍成 N/2，是 subset-sum 在 N≤40 的標準解。

03

核心策略 · 三步驟// Strategy

  1. 切兩半 · 左半 = 物品 [0, N/2)，右半 = [N/2, N)，各至多 20 件。
  2. 左半建表 · 枚舉左半 2^(N/2) 個子集，記下 (子集和 s, 選法 bitmask)，依 s 排序。
  3. 右半配對 · 枚舉右半每個子集（和 s'），在左半表中二分搜 K − s'。找到 ⇒ 兩邊 bitmask 合起來就是答案，逐位 Report。
存在子集和 = K　⟺　∃ 右半和 s'　使得　K − s' 出現在左半的子集和集合中
左半排序後二分搜每個 K−s' 是 O(log)；總 O(2^(N/2)·N)（建表帶 N 的列舉成本）。

05

為什麼「折半 + 二分搜」就把指數砍一半// Why It Works

// MEET IN THE MIDDLE
  1. 任何一組合法選法 S，把它拆成「落在左半的部分 S_L」與「落在右半的部分 S_R」，兩者互斥且聯集 = S。
  2. 於是 sum(S_L) + sum(S_R) = K。固定 S_R（和為 s'）後，左半只需湊出 K − s'。
  3. 把左半所有子集和先算好、排序成一張查找表（2^(N/2) 筆）；右半枚舉每個 s' 時，只要在表中二分搜 K − s' 是否存在。
  4. 命中就還原兩半的 bitmask、合併輸出。任一合法解都會在某次右半枚舉被找到 ⇒ 不漏解。
  5. 成本：兩半各 2^(N/2) 個子集（2²⁰≈10⁶），加上排序 O(2^(N/2)·N/2) 與每次二分搜 O(N/2) ⇒ 指數從 N 變 N/2。

關鍵是「把問題切成兩個獨立可枚舉的一半，再用一張排序好的表把兩半 O(log) 拼起來」。 這個「列一半、查一半」的套路，就是 Meet in the Middle。

複雜度// Complexity

建表 O(2^(N/2)·N/2) + 排序 O(2^(N/2)·N/2) + 配對 O(2^(N/2)·log) = O(2^(N/2)·N)
N=40 ⇒ 2²⁰≈10⁶，乘上 N 仍是 ~10⁷～10⁸ 級，1 秒內可解；樸素 O(2⁴⁰)≈10¹² 會 TLE
項目數量備註
TimeO(2^(N/2)·N)折半把指數 N 砍成 N/2
SpaceO(2^(N/2))左半 (和, mask) 查找表

Constant factor中排序 + 二分搜常數

值與 K 都到 10¹⁸ ⇒ 全程用 long long；和不會溢位（ΣA ≤ 10¹⁸ < 9.2×10¹⁸）。
