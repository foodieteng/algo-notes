石油

// Virtual-Source MST · 超級源點建虛邊

★ ★ ★ ☆ ☆ 核心 MST · 虛擬源點 · 時間 O((N+M) log)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

N 個國家、M 條道路（無向，連 uᵢ–vᵢ、建造費 wᵢ，可能有重邊與自環）。 每個國家要拿到石油有兩種方式：① 自己開採油井（國家 i 花 cᵢ）； ② 透過道路連到一個有油井的國家。問：最少花多少錢，能讓每個國家都取得石油？

1 ≤ N ≤ 2×10⁵ · 0 ≤ M ≤ 2×10⁵ · 1 ≤ uᵢ,vᵢ ≤ N · 1 ≤ cᵢ,wᵢ ≤ 10⁹。 「拿到石油」= 自己有井，或經由已建道路連到某個有井的國家。

02

為什麼用虛擬源點 MST// Why Virtual Source

A 枚舉誰開井 O(2ᴺ)
先選一組開井國家、再求連通道路 ⇒ 子集指數爆炸。
× REJECT · TLE
B 純 MST 連全部 漏掉開井
只用道路連通所有國家，沒考慮「至少要有一口井」⇒ 模型不完整。
△ REJECT · 不完整
BOTH FAIL · ADOPT
C 虛擬源點 + MST O((N+M) log)
加一個「油源節點 0」，每國連 (0, i) 花 cᵢ（= 開井）。對 N+1 點求 MST。
● ADOPT · OPTIMAL

關鍵轉換：「開井」其實就是「連到油源」。新增一個虛擬節點 0 代表油源， 把「國家 i 開井花 cᵢ」畫成一條虛邊 (0, i) 權重 cᵢ。 於是「每國都拿到油」⟺「每國都連到節點 0」⟺ 這張 N+1 點圖的最小生成樹。

03

核心策略 · 三步驟// Algorithm Outline

  1. 建虛邊 · 加虛擬源點 0；每個國家 i 加一條邊 (0, i, cᵢ)（代表「在 i 開井」）。
  2. 放道路 · 把 M 條道路 (uᵢ, vᵢ, wᵢ) 也丟進邊集合。
  3. 跑 MST · 對 N+1 個節點（0~N）跑 Kruskal，連成生成樹需 N 條邊； 權重和就是答案。

為何正確 · MST 保證每個國家都連到 0（拿到油），且總花費最小。 「至少要有一口井」自動成立 —— 樹中必有至少一條 (0, i) 虛邊（否則 0 孤立、樹不連通）。

05

為何虛擬源點 MST 等價於原問題// Why It's Equivalent

這個轉換的漂亮之處：把「兩種取得方式」統一成「連到油源」一件事。

VIRTUAL SOURCE · 等價性論證
  1. 定義虛擬節點 0 = 「油源」。國家 i「拿到石油」⟺ 在最終建出來的圖裡，i 連得到 0。
  2. 「在 i 開井花 cᵢ」⟺ 「付 cᵢ 把 i 直接接到 0」= 虛邊 (0, i, cᵢ)。
  3. 「i 經道路連到有井的國家 j」⟺ 「i 透過道路 + j 的虛邊接到 0」。 兩種方式在這張擴充圖裡都只是『連到 0 的一條路徑』。
  4. 要「每國都連到 0」且「花費最小」，就是把 {0, 1, …, N} 連成一棵樹、權重最小 ⇒ 最小生成樹。MST 自動決定「哪些國家開井（選了虛邊）、哪些道路要建」。

「至少一口井」由連通性保證：若沒選任何虛邊，節點 0 會孤立、無法連通 ⇒ MST 一定至少含一條 (0, i)。

實測佐證：對 800 組隨機輸入（含自環、重邊），「虛源 MST」與「窮舉開井子集 + 連通檢查」 答案完全一致、零反例，N=2×10⁵、M=2×10⁵ 在 1 秒內跑完。

複雜度// Complexity

邊數 = N（虛邊）+ M（道路）。排序主導，DSU 近乎線性：

T = O((N + M) log(N + M))
N、M 各 2×10⁵ ⇒ 4×10⁵ 條邊，排序約 4×10⁵ × 19 ≈ 8×10⁶，實測 1 秒內
項目數量備註
TimeO((N+M) log)排序 N+M 條邊主導
SpaceO(N + M)邊陣列 + 並查集（大小 N+1）
數值型別long long權和最大 ~N·10⁹ ≈ 2×10¹⁴，必用 64-bit
生成樹邊數NN+1 個節點 ⇒ 需 N 條邊
