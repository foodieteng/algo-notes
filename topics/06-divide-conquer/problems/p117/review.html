重點回顧// Concept Recap

關鍵點內容
2×2 block 切割 A(N) = [[TL, TR], [BL, BR]]，每個子塊大小 K × K = N/2 × N/2。
TL = BR 兩個子塊都填 A(K) 原值（不加 shift）— 值域 {1..2K−1}。
BL = A(K) + shift 整塊平移 shift = 2K−1 = N−1，值域 {2K..2N−2}。
TR 對角 = 2N−1，其餘 = A(K) + shift 對角的 2N−1 是把最大值灑進每個 row/col 的關鍵；其餘格與 BL 同樣 +shift。
正確性 歸納：A(K) 滿足性質 ⇒ A(N) 也滿足。row i ∪ col i 拆成 A(K) 自己 + shifted 版本 + 對角 2N−1 = {1..2N−1}。
複雜度 T(N) = 2 T(N/2) + Θ(N²) = Θ(N²)。N² 次寫入 + N² 次 Report。
輸出順序 row-major：先 row 1 從左到右，再 row 2，... 直到 row N。

觀念題// Concept Questions

Q1 遞迴前提

為何 N 必須是 2 的次方？若 N = 6 或 N = 12，這個分治構造還能用嗎？

Q3 歸納證明

給定 A(K) 滿足性質，請寫出 A(N = 2K) 的 row i ∪ col i = {1..2N−1} 的歸納骨架（針對 i ∈ [1..K] 的情況）。

Q4 值域重複

TL 與 BR 都是 A(K) — 那矩陣會不會出現大量重複值？題目允許嗎？

Q5 變形題

實際上 7 出現在 (1,3) 和 (2,4)（TR 對角）。BR 那一塊沒有 7，但 row 3、row 4 的 7 來自 col 3、col 4 中的 TR 對角格 (1,3) 與 (2,4) — 透過 col 的聯集補上。

App 3 改題思辨

若把題目條件改成「row i ∪ col i = {1..N}」（值域縮到 N，不是 2N−1），這個構造還能用嗎？這個改題還有解嗎？跟原題比起來是哪一類問題？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、A 對應 A。

A1 · 遞迴前提

構造的核心是 2×2 block 切割：把 A(N) 拆成 4 個大小相同的 K × K 子塊。要每一層都能對稱地切，就要 N 一路除 2 都還是整數 ⇒ N = 2^K。

若 N = 6，K = 3，2K = 6 一切都對稱；但下一層要切 A(3) 就切不開了（3 不是 2 的次方）。N = 12 同樣會在 A(3) 卡死。

所以本構造只適用於 N = 2^K。對任意 N 是否存在 Bad Array 是另一個問題（其實有 — 但需要不同的構造），題目刻意限制 N = 2^K 讓分治成立。

A2 · 對角線特殊處理

每個 row 和每個 col 都需要 2N−1。對角線 (i, K+i) 剛好是「row i 與 col K+i」的共同位置 — 同一格同時替兩條線塞入 2N−1。

  • row i（i ∈ [1..K]）會看到 (i, K+i) = 2N−1 ✓
  • col K+i 也會看到 (i, K+i) = 2N−1 ✓
  • 下半 row（i ∈ [K+1..N]）：令 i' = i − K，row i 看的是 BR row i' 與 BL row i' — 都沒 2N−1。但 col i = col K + i'，其中位置 (i', K+i') = (i', i) = 2N−1（TR 對角）⇒ col 補進來 ✓

如果把 2N−1 放到 TR 的非對角位置（譬如反對角 (i, K+K+1−i)）：

  • 那一格只服務 row i 和 col (K+K+1−i)，不是 col (K+i)。
  • row 還是有 2N−1，但 col 的覆蓋會錯位 — 對角的 i ↔ K+i 對應斷掉。
  • 必須再額外補別的格才能覆蓋所有 col，TR 的結構就不再是「對角 + 其餘 +shift」這麼乾淨。

對角線是唯一能讓 row 與 col 配對覆蓋 2N−1 的位置。

A3 · 歸納證明

歸納假設：對 A(K)，所有 i ∈ [1..K] 都滿足 row_K i ∪ col_K i = {1..2K−1}。

對 A(N = 2K) 的 i ∈ [1..K]：

  • row i = TL[i][*] ∪ TR[i][*]
    • TL[i][*] = A(K)[i][*]，貢獻 row_K i 的集合 ⊆ {1..2K−1}
    • TR[i][*]：對角格 (i, K+i) = 2N−1；其餘格 j ≠ i 是 A(K)[i][j] + shift，貢獻 row_K i \ {A(K)[i][i]} 加 shift
  • col i = TL[*][i] ∪ BL[*][i]
    • TL[*][i] = A(K)[*][i]，貢獻 col_K i
    • BL[*][i] = A(K)[*][i] + shift，貢獻 col_K i 加 shift（包含 A(K)[i][i] + shift 那一格）
// UNION
A(K) 部分：row_K i ∪ col_K i = {1..2K−1}    (歸納假設)
shifted 部分：(row_K i ∪ col_K i) + shift = {2K..2N−2}
   注意 BL 補回了 TR 缺的「A(K)[i][i] + shift」
   ⇒ shifted 部分完整覆蓋 {2K..2N−2}
對角部分：{2N−1}    (來自 TR 對角)

合起來：{1..2K−1} ∪ {2K..2N−2} ∪ {2N−1} = {1..2N−1}  ✓

i ∈ [K+1..N] 的情況對稱（用 i' = i − K 改套到下半）。Q.E.D.

A4 · 值域重複

會出現大量重複值。觀察 A(4)： 

// VALUE COUNTS in A(4)
value 1: 出現 4 次  (TL[1][1], TL[2][2], BR[1][1], BR[2][2])
value 3: 出現 2 次  (TL[1][2], BR[1][2])
value 4: 出現 2 次  (BL[1][1], BL[2][2])
...
value 7: 出現 2 次  (TR 對角 = (1,3), (2,4))

但題目允許重複。題目要求的只是「每個 row ∪ col 的集合是 {1..2N−1}」，集合會自動去重 — 同樣的值出現多次不影響集合內容。

實際上「重複」是構造能成功的必要條件：N × N = N² 個格，只有 2N−1 種值 ⇒ 平均每個值出現 N²/(2N−1) ≈ N/2 次。沒有重複根本塞不滿矩陣。

A5 · 變形題

若 BR 改成 A(K) + shift，構造會壞掉。具體看下半的 i ∈ [K+1..N]：

  • row i = BL[i'][*] ∪ BR[i'][*]，兩塊都 +shift ⇒ 整個 row 的值域 ⊆ {2K..2N−2}，少了 {1..2K−1}。
  • col i = TR[*][i'] ∪ BR[*][i']。TR 對角貢獻 2N−1，TR 其餘 +shift，BR 也 +shift ⇒ col 值域 ⊆ {2K..2N−1}，同樣少了 {1..2K−1}。
  • row ∪ col ⊆ {2K..2N−1}，無法覆蓋 {1..2K−1} ⇒ 違反 row i ∪ col i = {1..2N−1}。

BR 必須與 TL 同值，不能平移。直覺是：BR 是下半 row 與右半 col 「拿回低值（{1..2K−1}）」的唯一來源；BL 平移、TR 平移（含對角），三塊都跟「高值」站隊，BR 必須留守低值。

A · App 1

N = 8, K = 4, shift = 2K − 1 = 7, 2N − 1 = 15。

A(8)[1][5]：col 5 = K + 1 = 5 ⇒ 落在 TR 子塊，TR 內的 local index 是 (i=1, j=5−K=1)。

i == j（都是 1）⇒ TR 對角 ⇒ 值 = 2N − 1 = 15。 

// A(8)[1][5] = 15
原始位置 (1, 5)
→ TR 子塊 local (1, 1)
→ i == j 對角 ⇒ 2N - 1 = 15

不需要展開 64 個格，直接看「落在哪個子塊」與「是不是對角」就能 O(log N) 查任意格的值。

A · App 2

N = 1024： 

// N = 1024
Report 次數 = N² = 1024² = 1,048,576 ≈ 10⁶

建構寫入：每格寫一次 = N² = 10⁶
（TL 與 BR 由更深層遞迴寫；BL 與 TR 由當層的雙重 for 寫；每格只被某層寫一次。）

兩者同階：Θ(N²) = 約 10⁶。

瓶頸不是建構也不是輸出，是「兩者加起來 = 2 × 10⁶」 — 對現代 CPU 仍然輕鬆秒過。

A · App 3

若值域縮到 {1..N}，要求 row i ∪ col i = {1..N}：

  • row i 有 N 格、col i 有 N 格、扣掉重疊格 (i, i)，共 2N − 1 個位置。要塞滿 {1..N}（N 種值），必然有 2N−1 − N = N−1 格是重複。
  • 更強的解：Latin square（拉丁方陣） — 每 row 都是 {1..N} 的排列、每 col 也是。對任何 N ≥ 1 都存在（譬如 A[i][j] = ((i + j − 2) mod N) + 1）。
  • 本構造完全不能用 — 我們依賴 {1..2K−1} ∪ {2K..2N−2} ∪ {2N−1} 的拼接，那是「為 2N−1 量身打造的容量規劃」，縮到 N 後拼接邏輯整個垮掉。

所以 {1..N} 版的問題是另一類問題（Latin square / quasigroup），跟本題的 {1..2N−1} Bad Array 屬於不同分類。 本題的「2N−1」是構造能用 D&C 拼接的唯一容量點。
