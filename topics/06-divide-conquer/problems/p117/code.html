01

完整實作// Full C++

互動式題目：實作 solve(N)，由 judge 提供 Report(int)。不需 main()。 整個解只有一個雙層迴圈 — 不存陣列、邊算邊輸出。

p117.cpp C++17 
#include <bits/stdc++.h>
#include "lib0117.h"
using namespace std;

// 0-indexed。對角 = N；i<j 填 i^j，i>j 填 2N-(i^j)，互補對和為 2N。
void solve(int N) {
    for (int i = 0; i < N; ++i) {
        for (int j = 0; j < N; ++j) {
            int val;
            if (i == j)      val = N;                 // 對角線中心
            else if (i < j)  val = i ^ j;             // 互補對較小的一半
            else             val = 2 * N - (i ^ j);   // 較大的一半，和為 2N
            Report(val);
        }
    }
}

雙層迴圈正好走 row-major（先 row 0 從左到右、再 row 1…），就是 judge 期待的 Report 順序。 答案不唯一，這份輸出跟題目範例長得不一樣沒關係 — 只要每個十字合法就會被接受。

02

複雜度分析// Complexity Derivation

把一次迴圈內層的工作量拆開：

步驟內容單格成本
判斷象限i == j / i < j 兩次比較O(1)
算值i ^ j 一次 XOR（+ 一次減法）O(1)
輸出Report(val)O(1)

總成本：

N² 格 × O(1) = O(N²) 時間，O(1) 空間
沒有遞迴、沒有陣列。每格直接由 (i, j) 算出值，零中間狀態。 時間下限就是 N²（輸出量），已達最佳。
項目數量備註
TimeO(N²)每格一次 XOR + 比較 + Report
SpaceO(1)不存陣列，邊算邊 Report（遞迴版要 ~4.8 MB 矩陣，這版省掉）
Recursion Depth—無遞迴
Report CallsN²N = 1024 ⇒ 約 10⁶ 次

子任務對照： N ≤ 8 ⇒ 64 次 Report； N ≤ 64 ⇒ 4096； N ≤ 1024 滿分 ⇒ 約 10⁶ — 全部遠在時限內，瓶頸不是時間而是「想到 XOR 構造」。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
必須 #include "lib0117.h" judge 提供的標頭 — Report 的宣告在裡面。沒 include 會編譯失敗：'Report' was not declared in this scope。和 P116 同樣模式。
N 必須是 2 的次方 XOR 給出完美配對的前提是 i XOR j 在 [0, N) 上封閉 — 只有 N = 2^K 才成立。N 不是 2 次方時 i^j 可能 ≥ N，配對就壞了。題目保證 N = 2^K，不需特判。
0-indexed，對角放 N 不是 2N−1 這版 i, j 從 0 起跳。對角中心是 N（值域正中），不是遞迴版的 2N−1。互補對是 {k, 2N−k}，2N−1 只是其中一組（k=1）的大半。
方向必須用 i < j / i > j 分 (i,j) 與 (j,i) 必須一個拿 i^j、一個拿 2N−(i^j)，這樣相加才 = 2N。若兩格都填 i^j，互補對的「大半」永遠不出現 ⇒ 十字缺一半的數。
Report 順序必須 row-major judge 期待 (0,0), (0,1), ..., (0,N−1), (1,0), ..., (N−1,N−1) 的順序。for i for j Report 就是這個順序。
不用 ios::sync_with_stdio 互動題只透過 Report 與 judge 溝通，沒有 cin/cout ⇒ 加 sync_with_stdio 也沒用，刪掉更乾淨。和 P116 同樣模式。
不需要全域矩陣 這版邊算邊 Report，O(1) 空間。遞迴版要 int M[1100][1100] ≈ 4.8 MB；XOR 閉式解整個省掉，是它相對遞迴版的最大優勢。
04

細節走讀// Section Walkthrough

① 三種象限

// THREE CASES per (i, j)
i == j  →  val = N                 # 對角中心，所有十字共用
i < j   →  val = i ^ j             # 互補對較小的一半
i > j   →  val = 2N - (i ^ j)      # 較大的一半

關鍵不變量：a[i][j] + a[j][i] = (i^j) + (2N - (i^j)) = 2N

② 為什麼固定 i 時 i^j 掃過 1..N−1

// BIJECTION (N = 4, i = 0)
j:      0    1    2    3
i^j:    0    1    2    3     ← 跳過 j=i 的 0，其餘 {1,2,3} 各一次

→ 對角 (i==i) 那格不算十字外的配對，恰好對應 i^j = 0 被排除。
→ 所以 row 0 的非對角格貢獻互補對編號 {1, 2, 3} 各一次。

③ 互補對 — 同色兩格分居對角兩側

// COMPLEMENT PAIR (N = 4, k = i^j)
(i, j) with i < j  →  i ^ j        (小半)
(j, i) with j > i  →  2N - (i ^ j) (大半)

例 (0,1): i<j ⇒ 0^1 = 1
   (1,0): i>j ⇒ 8 - 1 = 7
   → {1, 7} 是和為 8 的互補對 ✓

④ 收集 index 0 的十字（N = 4）

// CROSS of index 0
row 0 = a[0][0..3] = {4, 1, 2, 3}   (對角 4 + i<j 的 0^1,0^2,0^3)
col 0 = a[0..3][0] = {4, 7, 6, 5}   (對角 4 + i>j 的 8-1,8-2,8-3)

聯集 = {1,2,3,4,5,6,7} = {1..2N-1}  ✓

⑤ REPORT ROW-MAJOR

// REPORT ROW-MAJOR (N = 4)
for i in [0, N): for j in [0, N): Report(a[i][j])

A(4) =
  4 1 2 3
  7 4 3 2
  6 5 4 1
  5 6 7 4

輸出順序：4,1,2,3, 7,4,3,2, 6,5,4,1, 5,6,7,4
