重點回顧// Concept Recap

關鍵點內容
問題本質 每個數字 ×2 / 刪 / 維持，求最長「非遞減且每項 ≥ M」的子序列 = 變體 LIS。
候選值 每個 aᵢ 攜帶 {aᵢ, 2aᵢ}，只有 ≥ M 的可用。
狀態 dp[i][c] = 以「第 i 個數用候選 c」結尾的最長非遞減鏈。
轉移 dp[i][c] = 1 + max{dp[j][cc] : j<i, v[j][cc] ≤ v[i][c]}。
細節 非遞減用 ≤；候選 ≥ M；long long；答案可能 0。
複雜度 O(N²)；BIT 升級 O(N log N)（吃滿 N=10⁵）。
02

觀念題// Concept Questions

Q1 建模

「×2 / 刪除 / 維持」這三種操作，如何轉成「每個數字最多兩個候選值」的 LIS 模型？「刪除」對應到什麼？

Q2 為何不貪心

為什麼每個數字不能「一律取較小的 aᵢ」或「一律取較大的 2aᵢ」？舉例說明為何兩種都要保留、用 DP 試。

Q3 非遞減

題目要「非嚴格遞增（非遞減）」，轉移時比較該用 ≤ 還是 <？寫錯會怎樣？

Q4 下界 M

M 這個下界在哪裡用到？什麼情況下答案會是 0？

Q5 複雜度

樸素 DP 為什麼是 O(N²)？要吃滿 N=10⁵ 該怎麼加速？加速後是多少？

範例 Trace · N=3, M=1, a=[2, 1, 3]// Worked Trace

每個數字兩個候選值（皆 ≥ M=1，全部可用）：a₁=2→{2,4}、a₂=1→{1,2}、a₃=3→{3,6}。逐個算 dp[i][c]：

// INPUT3 1        // N=3, M=1
2 1 3      // a = [2, 1, 3]
// i=0 (a=2)，候選 {2, 4}
dp[0][0] (值2) = 1     // 前面沒東西，自己一個
dp[0][1] (值4) = 1
// i=1 (a=1)，候選 {1, 2}
dp[1][0] (值1) = 1     // 前面值都 > 1（2,4），接不上 → 只有自己
dp[1][1] (值2) = 2     // 可接 dp[0][0]=值2 (2 ≤ 2) → 1+1 = 2  ★用兩倍才接得上
// i=2 (a=3)，候選 {3, 6}
dp[2][0] (值3) = 3     // 接 dp[1][1]=值2 (2 ≤ 3)，鏈長 2+1 = 3
dp[2][1] (值6) = 3     // 同樣接 dp[1][1] → 3
// OUTPUTmax dp = 3      // 鏈 [2(a₁維持), 2(a₂×2), 3(a₃維持)] = [2, 2, 3]
觀察：a₂=1 若維持原樣（值 1）接不到前面的 2；改成 ×2=2 才湊出 [2, 2, 3]。 這正是「兩個候選值都要保留、用 DP 試」的精髓 —— 取較大值有時反而讓鏈更長。最終答案 3。
04

應用題// Quick Calculations

App 1 為何 0

範例2 N=3, M=42, a=[7,8,9] 答案是 0。用候選值解釋：每個數字最大能到多少？為什麼全不可用？

App 2 BIT 細節

用 BIT 加速時，為什麼要把一個數字的兩個候選值「先都 query、再都 update」？邊查邊改會出什麼錯？

App 3 變體

若操作改成「可乘 1、2 或 3 倍」（三個候選值），DP 要怎麼改？複雜度變化？

Q1 · 建模

「刪除」= 這個數字不進子序列（不選）。「維持」= 實際值 aᵢ；「變兩倍」= 實際值 2aᵢ。 所以每個數字最多兩個可選的實際值 {aᵢ, 2aᵢ}。

目標「非遞減且最長」就是保留原順序、值不下降的最長子序列（非嚴格 LIS）， 只是每個位置多了「選哪個候選值」的選擇 ⇒ 狀態加一維 c ∈ {0,1}。

Q2 · 為何不貪心

取較小值（aᵢ）比較好接後面，但有時必須取較大值（2aᵢ）才接得上前面較大的鏈； 反之亦然。哪個更優取決於前後文，無法局部決定 ⇒ 兩種都得試，這正是 DP。

範例就是反例：a₂=1 取原值 1 接不到前面的 2，取 2×1=2 才接得上，最終鏈才能到 3。 若貪心一律取小值就會漏掉這條更長的鏈。

Q3 · 非遞減

非嚴格遞增（非遞減）允許相等 ⇒ 轉移條件用 v[j][cc] ≤ v[i][c]。

若誤寫嚴格 <，就變成「嚴格遞增」，會把像範例 [2, 2, 3] 這種含相等的合法鏈排除，答案偏小 → WA。

Q4 · 下界 M

每個候選值必須 ≥ M 才可用：if (v[i][c] < M) dp[i][c]=0; 把不夠大的候選值直接作廢。

當所有數字的所有候選值都 < M（連 2aᵢ 都不夠），就沒有任何可用元素 ⇒ 答案 0。

Q5 · 複雜度

樸素 DP：狀態 2N 個，每個要掃前面所有 O(N) 個狀態取 max ⇒ O(N²)。N ≤ 1000 時 10⁶，秒解。

加速：內層其實是「值域上 ≤ v[i][c] 的 dp 前綴最大值」。離散化候選值後用 BIT（Fenwick）存前綴 max， query/update 各 O(log N) ⇒ 總 O(N log N)，可吃滿 N=10⁵。

App 1 · 為何 0

a=[7,8,9]，每個數字最大候選值是 2aᵢ：14、16、18，全部 < M=42。

沒有任何候選值能達到 M ⇒ 每個 dp[i][c] 都被作廢（=0），子序列一個都湊不出 ⇒ 答案 0。 直覺：再怎麼乘兩倍都太小，整個序列無法滿足「每項 ≥ 42」。

App 2 · BIT 細節

BIT 存「以某個值結尾的最大鏈長」。處理第 i 個數時，它的兩個候選值若「邊 query 邊 update」， 小候選值先 update 進 BIT 後，大候選值的 query 會把同一個 i 的小候選值也算進去。

那等於把同一個數字串進鏈裡兩次（用了兩個候選值），是非法的。 所以要先把兩個候選值都 query 完，再一起 update，確保 query 看到的只有 j < i 的狀態。

App 3 · 變體

三個候選值 {aᵢ, 2aᵢ, 3aᵢ}：狀態維度 c ∈ {0,1,2}，dp[i][c]、轉移內層也對三個候選取 max。

樸素仍是 O(N² · K)（K=候選數，這裡 3，當常數）；BIT 版 O(N · K · log N)。 一般化：每個數字 K 個候選值，模型不變，只是把 c 的範圍從 2 擴到 K。
