// 變體 LIS · 每個數字有兩個候選值（×1 或 ×2）

★ ★ ★ ★ ☆ 核心 兩候選值最長非遞減鏈 · 時間 O(N log N)（tails）
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

給長度 N 的正整數序列 a₁, a₂, …, a_N。對每個數字你可以三選一： 變成原本的兩倍（×2）、從序列中移除、或維持原樣。

你希望操作後的序列是一個每個數字都不小於 M 的非遞減序列（非嚴格遞增 ⇒ 允許相等）。 問這樣的序列最長可以多長？

範圍：1 ≤ N ≤ 10⁵（≥99% 測資 N ≤ 1000）、M ≤ 10⁹、aᵢ ≤ 10⁹，共 T ≤ 1000 筆測資。 白話：每個 aᵢ 提供兩個候選值 {aᵢ, 2aᵢ}（且須 ≥ M），保留順序挑一條最長非遞減子序列。是 LIS 的變體。

02

為什麼是「兩候選值的非遞減 LIS」// Why a Variant LIS

A 枚舉每個數字 ×1/×2/刪 O(3ᴺ)
每個位置三選一，N 個位置 ⇒ 3ᴺ 種組合，N=1000 完全爆。
× REJECT · TLE
B 忽略「兩倍」，只做普通 LIS —
沒考慮每個數可 ×2 ⇒ 漏掉用兩倍才接得上的鏈，答案偏小、會 WA。
△ REJECT · WRONG
BOTH FAIL · ADOPT
C 兩候選值的最長非遞減鏈（tails + upper_bound） O(N log N)
每個 aᵢ 攜帶兩個候選值 {aᵢ, 2aᵢ}（須 ≥ M），求最長非遞減子序列。概念上 dp[i][c] 接前面值 ≤ 它的鏈（O(N²)）；實作用 LIS 的 tails 技巧 + upper_bound 降到 O(N log N)，才吃得滿 N=10⁵。
● ADOPT · STABLE

關鍵轉換：「刪除」= 不選；「維持/兩倍」= 這個數字的實際值取 aᵢ 或 2aᵢ。 於是問題變成「每個元素有最多兩個可選值，求保留順序的最長非遞減子序列」——標準 LIS 加一層「候選值」維度。

03

核心策略 · 三步驟// Strategy

  1. 列候選值 · 第 i 個數的候選值 v[i][0]=aᵢ、v[i][1]=2aᵢ，只有 ≥ M 的才可用。
  2. 定義狀態 · dp[i][c] = 以「第 i 個數、用候選值 v[i][c]」結尾的最長非遞減鏈長度。
  3. 轉移 · dp[i][c] = 1 + max{ dp[j][c'] : j<i, v[j][c'] ≤ v[i][c] }，答案取所有 dp 的最大值。
dp[i][c] = 1 + maxj<i, c'∈{0,1} { dp[j][c'] : v[j][c'] ≤ v[i][c] }　（v[i][c] ≥ M 才有效）
答案 = max dp[i][c]（沒有任何候選值 ≥ M ⇒ 答案 0）。非遞減 ⇒ 條件是 ≤（允許相等）。

06

為什麼只要兩個候選值、為什麼貪心不行// Why Two Candidates

// 操作 ⇒ 候選值
  1. 「移除」⇒ 這個數字不在子序列裡（不選）。
  2. 「維持」⇒ 實際值 = aᵢ；「變兩倍」⇒ 實際值 = 2aᵢ。每個數字最多兩種實際值，不會更多。
  3. 必須 實際值 ≥ M 才合法 ⇒ 候選值先用 M 過濾；若 2aᵢ < M 則這個數字完全不能用。
  4. 目標「非遞減且最長」⇒ 保留原順序、值不下降的最長子序列 = LIS（非嚴格）。每個位置多了「選哪個候選值」的選擇 ⇒ 狀態加一維 c∈{0,1}。
  5. 為何不能貪心取小值？ 取 aᵢ（較小）雖然好接後面，但有時必須取 2aᵢ 才接得上前面較大的鏈。兩種都要試 ⇒ DP 枚舉 c，不能貪心固定。

範例就是證據：a=[2,1,3]，a₂=1 若維持原樣（值 1）接不到前面的 2；改成 ×2=2 才能接上 a₁=2，湊出 [2, 2, 3] 長度 3。 這正是「每個數字保留兩個候選值、用 DP 試」的必要性。

07

複雜度// Complexity

概念 O(N²)（dp[i][c] 掃前面）⇒ 實作用 tails + upper_bound 降到 O(N log N)
99% 測資 N ≤ 1000 連 O(N²) 都秒解；要吃滿 N=10⁵ 就用 tails 版（每數字兩候選各一次二分搜），見 Code 頁。
項目數量備註
TimeO(N log N)tails + upper_bound；概念版 O(N²)
SpaceO(N)tails 陣列
遞迴深度0（迭代）由左往右掃一遍
Constant factor低每數字 2 次二分搜（兩候選）

值到 2×10⁹ > int 上限 ⇒ 候選值與比較用 long long；T 筆測資每筆獨立清空 dp。

08

一句洞察// One-Sentence Insight

「×2 / 刪 / 維持」⇒ 每個數字攜帶兩個候選值 {aᵢ, 2aᵢ}（須 ≥ M）。
問題化為「保留順序的最長非遞減子序列」——
用 LIS 的 tails + upper_bound（大候選先放）一次掃過，O(N log N) 才能 AC。
