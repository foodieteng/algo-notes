哪裡有卦，哪裡就有源

// Bipartite Check · 二分圖判定（染色法）

★ ★ ★ ☆ ☆ 核心 BFS 二著色 · 找奇環 · 時間 O(n + m)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

n 個成員（編號 0 ~ n−1），給 m 對「卦源」。 假設每對卦源必由不同性別的兩人組成（人人異性戀）。 問：這份調查結果有沒有可能合理？可能輸出 NORMAL.，否則 RAINBOW.

把人當點、卦源當邊。「每條邊兩端不同性別」⇔ 用兩種顏色塗點， 每條邊兩端顏色不同 ⇔ 這張圖是不是二分圖（bipartite）。

02

為什麼用 BFS 染色// Why 2-Coloring

A 枚舉每人性別 O(2ⁿ)
n ≤ 10⁵ ⇒ 2¹⁰⁰⁰⁰⁰ 種指派，宇宙熱寂前跑不完
× REJECT · TLE
B 隨便塗再驗 不一定收斂
沒有「沿邊傳遞」的紀律，反覆塗改可能無限迴圈
△ REJECT · UNSTABLE
BOTH FAIL · ADOPT
C BFS 二著色 O(n + m)
任挑起點塗色 0 → 沿邊把鄰居塗成反色 → 撞到「同色相鄰」就是奇環 ⇒ RAINBOW
● ADOPT · STABLE

等價說法：圖是二分圖 ⇔ 沒有奇數長度的環。三角形（長度 3 的環）就是最短的奇環 — A–B、B–C、C–A 三對卦源，三人不可能兩兩異性，必定矛盾。

03

核心策略 · 三步驟// Algorithm Outline

建鄰接表後，對每個還沒著色的點跑一次 BFS（圖可能不連通）：

  1. 起點塗 0 · 推進 queue，color[s] = 0
  2. 沿邊傳遞 · 取出 u，每個鄰居 v：沒塗過就塗成 color[u] ^ 1 推進 queue
  3. 檢查衝突 · 若 v 已塗過且 color[v] == color[u] ⇒ 奇環 ⇒ 整題 RAINBOW.

所有點都塗完、從沒撞到衝突 ⇒ NORMAL.。多筆測資 T ≤ 10，每筆獨立重置。

05

為何「同色相鄰」就等於有奇環// Why It Works

BFS 著色其實在記每個點的層數奇偶：color = 距起點距離 mod 2。 撞到衝突代表找到一條「同奇偶」卻又相鄰的邊 —— 那必然圍出一個奇環。

關鍵觀察 · 衝突邊 = 奇環的最後一塊拼圖
  1. BFS 把每點塗成 dist(起點, 點) mod 2：層 0 塗 0、層 1 塗 1、層 2 塗 0……。
  2. 正常的樹邊都連「相鄰層」⇒ 兩端奇偶相反 ⇒ 顏色一定不同。
  3. 若某條邊 u–v 兩端同色 ⇒ dist(s,u) 與 dist(s,v) 同奇偶。
  4. 從 s 到 u 的路徑（長 du）＋ 邊 u–v（長 1）＋ v 回 s 的路徑（長 dv） 繞一圈，長度 du + 1 + dv。du 與 dv 同奇偶 ⇒ du + dv 偶 ⇒ 全長奇數。
  5. ⇒ 存在奇環 ⇒ 無法二著色 ⇒ 不可能人人異性 ⇒ RAINBOW.。

反過來，若整張圖塗到底都沒衝突，那這個著色本身就是一組合法性別指派 ⇒ 必定 NORMAL.。 所以 BFS 一遍就同時「找到反例」或「給出一組可行解」。

06

複雜度// Complexity

建鄰接表 O(n + m)；BFS 每個點進出 queue 各一次、每條邊看兩次（無向）：

T = O(n + m)
每個點走訪一次 + 每條邊掃兩次 — 與圖大小成線性
項目數量備註
TimeO(n + m)n ≤ 10⁵、m ≤ 2 × 10⁵，單筆即線性掃完
SpaceO(n + m)鄰接表 + color[] + queue
BFS 廣度O(n)queue 最壞裝全部點，用陣列當佇列即可
Constant factor低純整數、無遞迴爆 stack 風險（用 BFS）
