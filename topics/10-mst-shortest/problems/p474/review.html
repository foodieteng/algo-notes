重點回顧// Concept Recap

關鍵點內容
核心轉換 「開井」⟺「連到油源」。加虛擬節點 0 當油源。
虛邊 國家 i 開井費 cᵢ ⇒ 虛邊 (0, i, cᵢ)。
道路 M 條 (u, v, w) 照常放進邊集合（重邊、自環無妨）。
解 對 N+1 個節點求 MST，權和即答案；需 N 條邊。
至少一口井 由連通性保證：MST 必含至少一條 (0, i) 虛邊。
數值 權和達 2×10¹⁴ ⇒ total 用 long long。

觀念題// Concept Questions

Q1 核心轉換

為什麼「開井」可以畫成「連到一個虛擬源點 0 的邊」？這個轉換把兩種取油方式統一成什麼？

Q2 至少一口井

題目要求「每國都能取得石油」，但沒明說至少要開幾口井。為什麼 MST 會自動保證至少一口井？

Q3 邊數

為什麼生成樹要取 N 條邊而不是 N−1 條？並查集要開多大？

Q4 自環重邊

道路允許 u = v（自環）與同一對多條道路。需要特別處理嗎？Kruskal 怎麼自然應付？

Q5 資料型別

哪個變數最容易忘了開 long long？給一個會溢位的數量級。

範例 Trace · 跑一次 code 看會輸出什麼// Worked Trace

用小例子：N=3，開井費 c=[5,4,6]，道路 1–2(1)、2–3(2)。

// INPUT3 2
5 4 6
1 2 1
2 3 2

STEP 1 · 建邊（虛邊 + 道路），加虛源 0：

邊權重類型
1—21道路
2—32道路
0—24虛邊（在 2 開井）
0—15虛邊（在 1 開井）
0—36虛邊（在 3 開井）

STEP 2 · Kruskal（取 N=3 條邊）：

// EDGE 1 · (1—2, w=1)
1, 2 不同塊 → union {1,2}
total = 1, used = 1
// EDGE 2 · (2—3, w=2)
{1,2} 與 3 不同塊 → union {1,2,3}
total = 3, used = 2
// EDGE 3 · (0—2, w=4)
源點 0 與 {1,2,3} 不同塊 → union {0,1,2,3}
total = 7, used = 3 = N ⇒ break ✓
（選 0—2 = 在國家 2 開井；全部國家都連到油源了）
// (0—1, 0—3 不會處理：已連滿)
若處理：0 與該國已同塊 ⇒ 成環 ⇒ 跳過
// OUTPUT7
觀察：最佳解是「只在國家 2 開一口井（4），再用兩條便宜道路（1+2）把 1、3 接上」= 4+1+2 = 7。比「每國各開井」（5+4+6=15）省很多。MST 自動權衡開井 vs 修路，挑最划算的組合。

應用題// Quick Calculations

App 1 手算

樣本 1：N=1, M=0, c=[1]。手算答案。若 M=0 但 N=3, c=[5,4,6] 又是多少？

App 2 極端

若所有道路都超貴（w 遠大於任何 cᵢ），最佳解會長什麼樣？答案是多少？

App 3 變形

若改成「最多只能開 1 口井」，這個虛源 MST 還對嗎？該怎麼調整思路？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

A1 · 核心轉換

把「油源」具象成一個虛擬節點 0。「在國家 i 開井花 cᵢ」=「付 cᵢ 把 i 直接接到油源 0」= 虛邊 (0, i, cᵢ)。

這樣兩種取油方式統一成同一件事：連到節點 0 ——「自己開井」是直接連 0、 「經道路連到有井的國家」是間接連 0。要每國都連到 0 且花費最小 ⇒ 最小生成樹。

A2 · 至少一口井

MST 必須讓所有節點連通，包含虛擬源點 0。若一條虛邊 (0, i) 都沒選， 節點 0 就會孤立，整張圖不連通 ⇒ 不是生成樹。

所以 MST 一定至少含一條 (0, i) 虛邊 = 至少開一口井。這個約束免寫程式、由連通性自動成立。

A3 · 邊數

節點是 0, 1, …, N 共 N+1 個。生成樹連通 K 個節點需 K−1 條邊 ⇒ N+1 個節點需 N 條邊。

並查集要開 N+1（索引 0~N），並初始化 par[0..N]。 若沿用「N−1 條」或忘了節點 0，會少連一塊或越界。

A4 · 自環重邊

不必特別處理。 自環 u=v：find(u)==find(v) 永遠成立 ⇒ 直接被跳過。

同一對的多條道路：第一條（最便宜的，因已排序）會 union，其餘變成同塊 ⇒ 後面的自動跳過。 Kruskal 的「同塊就跳過」機制天生吃得下重邊與自環。

A5 · 資料型別

最容易忘的是累加權和 total（與排序鍵 w）。 

// 溢位量級
最多 N 條邊 ≈ 2×10⁵
每條 c 或 w 最大 10⁹
權和最大 ≈ 2×10⁵ × 10⁹ = 2×10¹⁴
int 上限 ≈ 2.1×10⁹  ⇒  必爆 ❌
long long 上限 ≈ 9.2×10¹⁸  ⇒  OK ✓

單一 c/w ≤ 10⁹ 雖塞得進 int，但累加會爆 ⇒ total 一定要 ll。

A · App 1

樣本 1：N=1、沒有道路 ⇒ 唯一辦法是在國家 1 開井 ⇒ 答案 = c₁ = 1 （MST 只有一條虛邊 0—1）。

N=3, M=0, c=[5,4,6]：沒有道路 ⇒ 每國只能各自開井 ⇒ 三條虛邊全選 ⇒ 5+4+6 = 15。

A · App 2

若所有道路都比任何 cᵢ 貴，Kruskal 會優先把所有虛邊（開井）排在道路前面選 ⇒ 每個國家各自開井、不修任何路。

答案就是 所有 cᵢ 的總和（Σcᵢ）。直覺：路太貴就不如各挖各的井。MST 自動做出這個判斷。

A · App 3

「最多只能開 1 口井」就不再是單純 MST了。此時所有國家必須靠唯一一口井 + 道路連通， 相當於：枚舉「井開在哪個國家 k」，其餘國家必須全靠道路連到 k。

  • 固定唯一井在 k ⇒ 需要「用道路把全部 N 國連成一棵樹」(道路 MST) + cₖ。
  • 道路 MST 與 k 無關（只要全國用道路連通），所以答案 = 道路連通全國的 MST 權和 + min cₖ （前提是道路本身能連通全國，否則無解）。

關鍵差異：原題「開幾口井都行」⇒ 虛源 MST 一次解決；「限 1 口井」⇒ 退化成「道路全連 + 挑最便宜的井」。
