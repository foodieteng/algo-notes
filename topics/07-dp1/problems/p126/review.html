重點回顧// Concept Recap

關鍵點內容
問題本質 3 色塗一列、綠藍不相鄰（含頭尾）⇒ 計數 DP。
狀態 dp[i][色] = 前 i 個、第 i 個塗該色的方法數。合法只看上一格。
轉移 R←{R,G,B}、G←{R,G}、B←{R,B}（綠藍互斥）。
頭尾 固定首色，末位排除與首色形成 {G,B}；或「線性總數 − 頭尾{G,B}」。
多筆 一次預處理 ans[1..maxN]，每筆查表 O(1)。
模數 對 1000007 取餘；減法後 +MOD 再取餘。

範例 Trace · 跑一次 DP 看 N=3 怎麼得 15// Worked Trace

用線性 per-color DP（r,g,b = 末位塗 R/G/B 的方法數），最後扣頭尾：

// i = 1 (base)
r=1  g=1  b=1     // 第 1 個 coin 三色各一種
total = 3
// i = 2
r = r+g+b = 1+1+1 = 3
g = r+g   = 1+1   = 2     // G 不接前面的 B
b = r+b   = 1+1   = 2     // B 不接前面的 G
total = 3+2+2 = 7   ← N=2 沒有頭尾{G,B}（會被相鄰擋掉），答案就是 7 ✓
// i = 3
r = 3+2+2 = 7
g = 3+2   = 5
b = 3+2   = 5
linear total = 7+5+5 = 17
// 頭尾修正
17 種裡，第1個與第3個恰為「一藍一綠」的有：
  G · ? · B   →  中間只能 R（G-?-B，?≠相鄰違規）：G R B  (1 種)
  B · ? · G   →  中間只能 R：B R G               (1 種)
共 2 種違反頭尾規則 ⇒ 扣掉
// OUTPUT17 − 2 = 15
觀察：線性 DP 已經把「綠藍相鄰」擋好（所以 G _ B 中間不能再放 G 或 B）， 但頭尾不相鄰，線性 DP 管不到 ⇒ 必須額外扣掉 G R B / B R G 這 2 種。 這就是「環形限制」要特別處理的地方。
04

應用題// Quick Calculations

App 1 無頭尾限制

若拿掉「頭尾不一藍一綠」只剩相鄰限制，N=3 答案是多少？通式長怎樣？

App 2 真環形

若改成「環狀排列」（第 N 個與第 1 個真的相鄰、所有相鄰都禁綠藍），DP 要怎麼調整？

App 3 k 色推廣

若有 k 種顏色、某些顏色對不能相鄰，這套 DP 怎麼推廣？複雜度？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · 狀態設計

因為「下一個能塗什麼」取決於當前末位的顏色（綠後面不能接藍…）。 若只記方法總數，無法判斷加新 coin 時哪些顏色合法 ⇒ 轉移無從寫起。 把末位顏色加入狀態，才能正確分流轉移。

Q2 · 轉移正確性

g[i] 是「第 i 個塗 G」的方法數。它的前一個 coin 不能是 B（綠藍不相鄰）， 所以只能接在末位是 R 或 G 的序列後面 ⇒ g[i] = r[i-1] + g[i-1]，不含 b[i-1]。 這條式子正是「G 不接 B」這個限制的直接編碼。

Q3 · 頭尾限制

只做線性 DP，N=3 得 17；正解 15，多算 2。 多出來的正是頭尾一藍一綠的兩種：G R B 與 B R G （它們相鄰都合法，但頭尾違規）。所以要扣掉。

Q4 · 多筆策略

T、N 都可達 10⁵，每筆各跑 O(N) ⇒ 最壞 T·N = 10¹⁰，TLE。

關鍵：不同測資只是不同的 N，答案只跟 N 有關。 所以一次預處理 ans[1..10⁵]（O(maxN)），每筆查表 O(1) ⇒ 總 O(maxN + T)。

Q5 · 模運算

3a(n−1) − a(n−2) − a(n−3) 先各自取餘後相減，結果可能是負數 （例如被減項較大）⇒ C++ 的 % 對負數會回傳負餘數。寫 (x % MOD + MOD) % MOD 校正回 [0, MOD)。

模數 1000007 = 29 × 34483不是質數。但本題全程只有加減乘、沒有除法 / 逆元， 合成數不影響正確性。（若要算組合數除法就得小心。）

App 1 · 無頭尾限制

只剩相鄰限制 ⇒ 就是線性總數 L(N)。N=3 為 17。

通式：L(N) = r+g+b，由 r'=r+g+b, g'=r+g, b'=r+b 遞推。 因對稱 g=b，可化簡為 L(N) = L(N−1) + 2·g(N−1) 一類二階遞迴 （數列 3,7,17,41,99,… 滿足 L(N)=2L(N−1)+L(N−2)，即 Pell 型）。

App 2 · 真環形

環狀（所有相鄰、含 N–1 接回 0 都禁綠藍）的標準做法：列舉第一個 coin 的顏色 c₀， 對每個 c₀ 跑線性 DP，最後只累加「末位與 c₀ 也合法相鄰」的方法數。

  • 固定 c₀ ⇒ 末位限制 = 「不能與 c₀ 綠藍相鄰」。
  • 3 個 c₀ 各跑一次 O(N)，加總即環形答案。

本題的「頭尾不一藍一綠」就是這個技巧的特例（只禁 {G,B} 這一對端點）。

App 3 · k 色推廣

狀態 dp[i][c]（c 是 k 種顏色之一）；轉移 dp[i][c] = Σ_{c' 可接 c} dp[i-1][c']。

用「相鄰相容矩陣」A[c'][c] 表示哪些對可相鄰，DP 就是反覆乘這個 k×k 矩陣 ⇒ 單次 O(k²)，N 步共 O(N·k²)；若 N 巨大可用矩陣快速冪 O(k³ log N)。 環形版同樣用「固定首色」處理。
