合成円円

// 區間 DP · 相鄰合併最小成本（石子合併）

★ ★ ★ ☆ ☆ 核心 區間 DP（matrix-chain / 石子合併） · 時間 O(N³)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

N 隻円円排成一列，各有「糟糕值」a₁, …, a_N。只有相鄰兩隻能合成； 每次合成前，一隻円円要吃掉「等於自己糟糕值」的高棕櫚，所以一次合成的高棕櫚成本 = 被合的兩隻糟糕值之和，合成後新円円的糟糕值 = 兩者之和。

把全部 N 隻合成成一隻，求最小總高棕櫚。 多筆 T ≤ 10，1 ≤ N ≤ 100，1 ≤ aᵢ ≤ 1000。

這是經典的「石子合併 / matrix-chain」：最終糟糕值固定 = 總和（與順序無關）， 但每次合成付的成本會疊加，合成順序決定總成本。範例 [1,2,3] → 9： 先合 1+2（付 3）得 3，再合 3+3（付 6）⇒ 3+6 = 9（比先合 2+3 的 11 省）。

02

為什麼用區間 DP// Why Interval DP

A 枚舉所有合成順序 O(卡塔蘭)
N−1 次合成的二元樹形狀數 = 卡塔蘭數，指數級爆炸 ⇒ N=100 完全不可能。
× REJECT · TLE
B 貪心合最小相鄰對 O(N²)
「每次合當前最小的相鄰兩隻」會選錯：不能任意重排、相鄰限制下貪心非最佳（≠ Huffman）。
△ REJECT · WRONG
BOTH FAIL · ADOPT
C 區間 DP O(N³)
合成 [i..j] 的最後一步一定切在某個 k：左半 [i..k]、右半 [k+1..j] 先各自合好，再合兩者。枚舉 k 取最小。
● ADOPT · STABLE

為什麼貪心錯？ 因為「相鄰才能合」⇒ 不像 Huffman 可任意配對。 最小相鄰對未必該先合（合了會改變後續相鄰結構）。DP 對每個區間枚舉所有切點，才不漏最佳。

03

狀態與轉移// State & Transition

dp[i][j] = 把 a[i..j] 合成成「一隻」所需的最小總高棕櫚。

dp[i][j] = mini≤k<j( dp[i][k] + dp[k+1][j] ) + sum(i..j)
切點 k：左半 [i..k]、右半 [k+1..j] 各自合好（子問題）；最後把兩隻合起來那一步要付 sum(i..j)（兩隻糟糕值之和 = 整段總和）
  1. base · dp[i][i] = 0（單一隻不必合成）。
  2. 關鍵觀察 · 不論 k 切哪裡，最後一步合 [i..j] 都要付 sum(i..j) ⇒ 這項是常數，只需最小化兩個子區間之和。
  3. 填表順序 · 按區間長度 len 由小到大填（先短後長），保證用到的子區間已算好。
  4. 答案 · dp[1][N]。

05

為什麼這個轉移是對的// Correctness

// 最後一步切點 + 成本與順序的關係
  1. 把 [i..j] 合成成一隻的整個過程，最後一定有「最後一次合成」：它把某個 [i..k] 已合成的一隻、與 [k+1..j] 已合成的一隻合在一起。
  2. 因為只能合相鄰、且結果是連續區段，這個 k 必落在 i ≤ k < j。枚舉所有 k 就涵蓋所有可能的最後一步。
  3. 最後一步的成本 = 兩隻糟糕值之和 = sum(i..j)（合成不改變總糟糕值，整段加起來就是 sum）。與 k 無關。
  4. 左右兩半的合成彼此獨立、且只動到自己區段 ⇒ 子問題最佳和能拼出大區間最佳（最優子結構）。
  5. 所以 dp[i][j] = sum(i..j) + min_k( dp[i][k] + dp[k+1][j] )。

換個角度：每個原始 aₓ 對總成本的貢獻 = aₓ ×（它在合成樹中的深度）。 合成順序就是在決定這棵二元合成樹的形狀，DP 等於在所有合法（相鄰）樹形中找加權路徑和最小者。

N ≤ 100 ⇒ ~10⁶ 次、T ≤ 10 ⇒ ~10⁷，秒過；用前綴和讓 sum(i..j) 是 O(1)

一句洞察// One-Sentence Insight

合成 [i..j] 的最後一步一定切在某個 k，且那一步固定付 sum(i..j)。
所以只要枚舉切點、最小化左右子區間成本之和 —
