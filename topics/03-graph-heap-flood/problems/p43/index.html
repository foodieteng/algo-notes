喵喵抓老鼠

// BFS 最短路 · 迷宮多源終點

★ ★ ☆ ☆ ☆ 核心 BFS Shortest Path（無權圖） · 時間 O(N × M)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

給 N × M 迷宮（N, M ≤ 100），四周圍牆。 # 不能走、. 可走、@ 是老鼠（可能多隻）、K 是貓（只有一隻）。 貓只能上下左右移動，問離貓最近的老鼠要走幾步？走不到任何老鼠輸出 = ="。多筆 T ≤ 31。

迷宮每一步成本都是 1（無權圖），求單源最短路到「任一終點」。 無權最短路 = BFS，不需要 Dijkstra。

02

為什麼用 BFS// Why BFS

A DFS 找路 不保證最短
DFS 先衝到底，第一次碰到 @ 的路徑不一定最短；要找最短得遍歷所有路徑 ⇒ 指數爆炸。
× REJECT · WRONG
B Dijkstra O(NM log NM)
正確，但每步權重都是 1，用 priority queue 是殺雞用牛刀，多一個 log。
△ REJECT · OVERKILL
BOTH FAIL · ADOPT
C BFS O(N × M)
從 K 出發一圈一圈擴散，第一個被取出的 @ 必為最近；每格只入列一次。
● ADOPT · STABLE

核心保證：BFS 按距離分層擴散 — dist 0、dist 1、dist 2…逐層取出。 所以最先被取出的老鼠，它的 dist 就是全場最小，當下即可停止。這正是「無權圖最短路」的標準解。

03

核心策略 · 三步驟// Algorithm Outline

維護 dist[N][M]（初始 −1 = 未訪），把貓 K 當起點：

  1. 起點入列 · 找到 K，dist[K] = 0，推進 queue。
  2. 逐格擴散 · 取出隊頭，看四方向；可走且未訪的鄰居 dist = 當前 + 1 並入列。
  3. 碰到老鼠就停 · 取出的格子若是 @，它的 dist 即答案。queue 空了還沒碰到 ⇒ 走不到 ⇒ = ="。

關鍵不變量：queue 裡的 dist 單調不減（最多差 1）⇒ 第一個取出的 @ 一定最近。

05

為什麼第一個取出的 @ 必為最近// Correctness

// LEMMA · BFS queue 的 dist 單調不減
  1. BFS 用普通 queue（FIFO）。每次擴散，新入列的格子 dist = 父 dist + 1。
  2. 因為父節點是按 dist 由小到大取出的，子節點的 dist 也跟著由小到大入列 ⇒ queue 內 dist 永遠單調不減，且最多差 1。
  3. 所以「取出順序」= 「dist 由小到大」⇒ BFS 是逐層擴散：先把所有 dist=k 取完才開始 dist=k+1。
  4. ⇒ 第一個被取出的 @，它的 dist 不可能比任何還沒取出的 @ 大 ⇒ 它就是最近的，可立即回傳。
  5. 每格 dist 一旦設定就不再更新（已訪即略過）⇒ 每格恰好入列一次 ⇒ 總工作 O(N×M)。

對照 DFS：DFS 用 stack（LIFO），會「一路衝到底」⇒ 第一次碰到 @ 的路長未必最短。 所以最短路一定要 BFS（FIFO）。

06

複雜度// Complexity

每格水路最多入列一次、出列一次，出列後看固定 4 個方向 ⇒ 每格 O(1)。

T(N, M) = O(N × M)  每筆測資
無權圖 BFS = 點數 + 邊數 = O(V + E)，網格上 E ≈ 4V ⇒ O(N × M)
項目數量備註

一句洞察// One-Sentence Insight

無權圖最短路 = BFS。
