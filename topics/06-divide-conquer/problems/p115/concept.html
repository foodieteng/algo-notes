P115 · CONCEPT DIVIDE & CONQUER

演算法概念

觀察一 · 暴力為何不夠

直接枚舉所有下標對 (i, j) 是 O(N²)： N = 10⁶ 時要算 ≈ 5 × 10¹¹ 對，連 30% 分數段（N ≤ 1000）以上都會 TLE。 所以需要在不枚舉所有對的情況下，依舊把貢獻全部計入。

觀察二 · Merge Sort 自帶逆序對

Merge sort 把陣列切兩半、各自遞迴排序、再合併。 合併（merge）兩個已排序的子陣列時， 「右半元素被取出」這個動作就對應到一群逆序對：

假設目前左半的游標在 i（指向左半剩餘元素中的最小者）， 右半的游標在 j。若 a[i] > a[j]，要取右邊的 a[j]。 因為左半已排序，所以左半剩下的所有元素 a[i], a[i+1], …, a[mid] 都 ≥ a[i] > a[j]。 這些元素在原序列裡都排在 a[j] 之前（左半在右半之前）， 所以每一個都跟 a[j] 形成一個逆序對。

本次 merge step 的貢獻 = (a[i] + a[i+1] + … + a[mid]) + a[j] × (mid − i + 1)

也就是「左半剩餘元素之和」加上「a[j] 乘上左半剩餘個數」。

// Key Insight

排序與計貢獻可以共用同一次掃描。merge 本來就會逐一比較兩半元素 — 我們只是在「取右邊」的時候多加一筆 O(1) 計算， 完全不會讓 merge 變慢。

觀察三 · 維護左半剩餘和

為了在 O(1) 取得「左半剩餘和」， merge 開始前先算一次左半總和 remaining， 每次取左半元素 a[i] 出去時就 remaining -= a[i]。 這樣每次需要貢獻時直接讀 remaining 即可。

複雜度

Time O(N log N) Space O(N)

T(N) = 2 · T(N/2) + O(N)   ⇒   T(N) = Θ(N log N)

Master Theorem case 2：a = 2, b = 2, f(N) = O(N) = O(N^{log_b a})， 所以 T(N) = Θ(N log N)。空間是 merge 需要的 O(N) 暫存陣列。

為什麼一定要 mod

值最多 10⁹、N 最多 10⁶。最壞情況（全降序）：

  • 逆序對數量 ≈ N(N − 1)/2 ≈ 5 × 10¹¹
  • 每對的和 ≤ 2 × 10⁹
  • 未 mod 的總和上界 ≈ 10²¹

long long 最大值約 9.2 × 10¹⁸，會溢位。 所以累加時就必須 mod 10000019。 注意 a[j] × count 這項，10⁹ × 10⁶ = 10¹⁵ 仍在 long long 內， 可以乘完再 mod。

// 觀念提示

mod 不影響「是否為逆序對」的判斷 — 排序比較永遠用原始值。 mod 只作用在「累加進 inv_sum」與「remaining 的維護」這兩個地方。 千萬不要把陣列裡的值 mod 掉後拿去 sort，那會破壞順序。
