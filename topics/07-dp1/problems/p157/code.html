01

完整實作 · 容斥枚舉排除人數// Full C++

先用巴斯卡三角建好 comb[n][k]（全測資共用，只用加法所以對任何模數都安全）。 每筆測資枚舉排除人數 j，算 comb[M][j] · ∏ comb[M−j][aᵢ]，交錯加減。

p157.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const long long MOD = 1000007;   // 注意：不是質數！29 × 34483
const int MAXN = 105;

long long comb[MAXN][MAXN];      // comb[n][k] = C(n,k) mod MOD
int cnt[MAXN];                  // cnt[i] = 第 i 類高棕櫚的數量 aᵢ

void buildPascal() {            // 巴斯卡三角：只用加法，任何模數都合法
    for (int n = 0; n < MAXN; n++) {
        comb[n][0] = 1;
        for (int k = 1; k <= n; k++)
            comb[n][k] = (comb[n - 1][k - 1] + comb[n - 1][k]) % MOD;
    }
}

void solve() {
    int N, M;
    cin >> N >> M;
    for (int i = 0; i < N; i++) cin >> cnt[i];

    long long ans = 0;
    for (int excluded = 0; excluded <= M; excluded++) {   // 排除 j 位鄰居
        int remain = M - excluded;                       // 剩 M−j 位可分
        long long term = comb[M][excluded];            // 選哪 j 位被排除：C(M,j)
        for (int i = 0; i < N; i++) {
            if (cnt[i] > remain) { term = 0; break; }   // 剩的不夠選 aᵢ 位 ⇒ 此項=0
            term = term * comb[remain][cnt[i]] % MOD;     // 第 i 類選 aᵢ 位：C(M−j, aᵢ)
        }
        if (excluded & 1) ans = (ans - term + MOD) % MOD;   // 奇數項：減（補 MOD 防負）
        else             ans = (ans + term) % MOD;           // 偶數項：加
    }

    cout << ans << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    buildPascal();
    int T;
    cin >> T;
    while (T--) solve();

    return 0;
}

為什麼 break 後就能歸零？ 一旦某類的 aᵢ > remain，C(remain, aᵢ)=0， 乘進去整項必為 0；提早 break 省掉剩餘乘法，也避免讀到表外。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
模數不是質數 1000007 = 29 × 34483。不能用「乘逆元」算 C(n,k)（逆元可能不存在）。必用巴斯卡三角加法遞推。
減法產生負數 奇數項要減 ⇒ (ans − term) 可能為負；務必 (ans − term + MOD) % MOD。
aᵢ > M−j 時的項 剩的鄰居不夠選 aᵢ 位 ⇒ C(M−j, aᵢ)=0，整項歸零。靠 comb 表回 0 自動處理，或提早 break。
j=0 也要算 j=0 那一項就是「全集」∏ C(M, aᵢ)，是容斥的起點，不能漏。
乘法溢位 每步 % MOD 後值 < 10⁶，相乘 < 10¹²，用 long long 安全；別忘了乘完立刻取模。
每筆重置 comb 表全程共用不必重建；但 ans 每筆要從 0 起算（這裡 ans 是函式區域變數，天然乾淨）。
04

細節走讀// Section Walkthrough

① 把題目翻成組合

// model
第 i 類 aᵢ 個、同類視為相同、每人最多拿 1 個同類
  ⇒ 第 i 類 = 從 M 位鄰居挑 aᵢ 位來給 = C(M, aᵢ)
各類獨立 ⇒ 不管「至少一個」時，總分法 = ∏ C(M, aᵢ)

② 容斥扣掉「有人沒拿到」

// inclusion–exclusion
A_t = 「鄰居 t 完全沒拿到」的分法
要的是 不落入任何 A_t 的分法數
答案 = Σ_{j=0..M} (−1)^j · C(M,j) · ∏_i C(M−j, aᵢ)
       └ 排除 j 位（C(M,j) 選法）剩 M−j 位自由分 ┘

③ 為何巴斯卡三角而非逆元

// non-prime modulus
MOD = 1000007 = 29 × 34483  (合數)
逆元法 C = n! · (k!)^{-1} · ((n−k)!)^{-1} 需要逆元
  但與 MOD 不互質的數沒有逆元 → 失效
巴斯卡 C[n][k]=C[n−1][k−1]+C[n−1][k] 只用加法 → 永遠正確
