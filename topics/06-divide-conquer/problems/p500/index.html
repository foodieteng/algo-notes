好的連續子序列

// Good Contiguous Subsequences · 分治 + max/min 重新框架

PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

給定一個 {1, …, N} 的排列 a[1..N]。對 (L, R) 配對來說， 若 a[L], a[L+1], …, a[R] 排序後是連續整數（b[i+1] − b[i] = 1），就稱它為 good。 問有幾組 (L, R) 滿足條件？

限制：T ≤ 5 × 10⁵ 組測資、每組 N ≤ 5 × 10⁵、 所有 N 加總 S ≤ 5 × 10⁵。子任務分數：S ≤ 5000 (20%)、S ≤ 10⁵ (60%)、無限制 (20%)。

關鍵觀察（題目原本的提示）：因為值兩兩相異， a[L..R] 是 good ⟺ max(a[L..R]) − min(a[L..R]) = R − L。 把「排序後連續」這個結構性問題壓成一個 O(1) 可查的等式，分治才有施力點。

03

核心策略 · 重新框架 + 4-case// Algorithm Outline

  1. 重新框架 · sorted-consecutive ⟺ max − min = R − L（因為 permutation ⇒ 值兩兩不同）
  2. 切半遞迴 · mid = (lo + hi) / 2，遞迴算 solve(lo, mid) + solve(mid+1, hi)
  3. 跨界 4-case · 對 L ≤ mid < R，依 max / min 各自來自左還是右半，分 4 種 case 各自 O(N) 計數

Base case · lo == hi 直接回傳 1（單一元素自動是 good）。

04

Base case 動畫 · 分治 on N=3// Animation A

// 先搞懂：動畫在數什麼

什麼算 good？一段連續區間 a[L..R]，把裡面的值拿出來排序， 如果剛好是一串連續整數、中間不缺號，就叫 good。單一元素一定是 good。

怎麼 O(1) 判定？因為 a 是排列、值兩兩不同，可以不用排序，直接查一個等式：

a[L..R] 是 good  ⟺  max − min = R − L
區間有 R−L+1 個相異值；最大減最小至少是 R−L， 剛好等於 R−L 才代表沒缺號（填滿 [min, max]）。

用動畫的 [2, 1, 3] 試三個區間：

// good 判定 on [2, 1, 3]  (1-indexed)
[2]      L=1,R=1: max=2 min=2 → max-min=0, R-L=0 → 0=0 ✓ good
[2,1]    L=1,R=2: max=2 min=1 → max-min=1, R-L=1 → 1=1 ✓ good
[2,1,3]  L=1,R=3: max=3 min=1 → max-min=2, R-L=2 → 2=2 ✓ good
[1,3]    L=2,R=3: max=3 min=1 → max-min=2, R-L=1 → 2≠1 ✗ 不是（缺 2）

動畫在數什麼？數「這個區間裡有幾段 good 子區間」。 分治把答案拆成三塊相加：

  1. LEFT — 完全落在左半的 good 段數（遞迴算）
  2. RIGHT — 完全落在右半的 good 段數（遞迴算）
  3. CROSS — 跨過中線的 good 段數（L ≤ mid < R，4-case 計數）
  4. TOTAL — 三者相加 = 這個區間的答案

了解定義後，先用小範例 [2, 1, 3] 走過分治骨架：SPLIT → LEFT → RIGHT → CROSS → TOTAL。 這是同一個演算法在最小有趣輸入下的展開 — 不是 brute，就是 D&C 自己。

06

正確性 · 為什麼 max − min = R − L 等價於 good// Lemma

// LEMMA · 連續整數集等價判定（permutation 前提）
  1. 令 S = {a[L], …, a[R]}，|S| = R − L + 1。 因為 a 是排列，元素兩兩相異 ⇒ |S| = R − L + 1 個不同值。
  2. 任何 R − L + 1 個不同整數，其 max − min ≥ (R − L + 1) − 1 = R − L（pigeonhole 風格的下界）。
  3. 等號成立 ⟺ S 恰好填滿區間 [min, max] 的所有整數 ⟺ 排序後 b[i+1] − b[i] = 1 ⟺ good。
  4. 結論：max(a[L..R]) − min(a[L..R]) = R − L ⟺ a[L..R] 是 good。 這把「排序後是否連續」化成 O(1) 可查的等式。

⇒ 分治拆完後，跨界子問題的「good 判定」變成「maxL/minL/maxR/minR 在四種組合下的等式」， 每個 case 可用 two-pointer + count[] 在 O(N) 內掃完。

複雜度分析// Complexity

每層遞迴 N 個位置，4 個 case 各掃 O(N)，maxL/minL/maxR/minR 預計算也是 O(N)：

T(N) = 2 · T(N/2) + O(N)  ⇒  Θ(N log N)
Master Theorem case 2：a = 2, b = 2, f(N) = O(N) = O(Nlogb a)
項目數量備註
Time (單組)O(N log N)N = 5 × 10⁵ ⇒ 約 10⁷ 次運算

一句洞察// One-Sentence Insight

permutation ⇒ distinct ⇒ max − min ≥ size − 1，等號成立才是 good。
