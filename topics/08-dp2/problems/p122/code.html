01

完整實作 · 折半 + 二分搜// Full C++

互動題：只需實作 solve，Report(id) 由評分程式提供。左半枚舉 2^(N/2) 個子集和 存成 (和, mask) 並排序；右半每個子集和 s 去左半二分搜 K − s，命中就把兩半的 bitmask 逐位 Report。

p122.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

void Report(int id);   // 評分程式提供：回報選中的物品 id

void solve(int N, long long K, long long A[]) {
    int half = N / 2;            // 左半 [0, half)，右半 [half, N)
    int rest = N - half;

    // 左半：枚舉 2^half 個子集，記錄 (子集和, bitmask)
    int leftCnt = 1 << half;
    vector<pair<long long, int>> leftSum(leftCnt);
    for (int mask = 0; mask < leftCnt; mask++) {
        long long s = 0;
        for (int b = 0; b < half; b++)
            if (mask >> b & 1) s += A[b];
        leftSum[mask] = {s, mask};
    }
    sort(leftSum.begin(), leftSum.end());   // 依和排序，之後二分搜

    // 右半：枚舉 2^rest 個子集，對每個找左半是否有 K - s
    int rightCnt = 1 << rest;
    for (int mask = 0; mask < rightCnt; mask++) {
        long long s = 0;
        for (int b = 0; b < rest; b++)
            if (mask >> b & 1) s += A[half + b];
        long long need = K - s;
        if (need < 0) continue;             // 右半已超過 K，跳過

        // 在排序後的左半子集和裡二分搜 need
        int lo = 0, hi = leftCnt - 1, found = -1;
        while (lo <= hi) {
            int mid = (lo + hi) / 2;
            if (leftSum[mid].first == need) { found = mid; break; }
            else if (leftSum[mid].first < need) lo = mid + 1;
            else hi = mid - 1;
        }
        if (found == -1) continue;          // 左半湊不出 need，換下一個右半子集

        // 找到：回報左半 + 右半選中的物品（1-indexed）
        int leftMask = leftSum[found].second;
        for (int b = 0; b < half; b++)
            if (leftMask >> b & 1) Report(b + 1);
        for (int b = 0; b < rest; b++)
            if (mask >> b & 1) Report(half + b + 1);
        Report(-1);
        return;
    }
    Report(-1);   // 題目保證有解；保險用
}

為什麼存 bitmask？ 二分搜只比「和」，但回報要知道具體選了哪幾件。 所以 leftSum 同時記 (和, mask)，命中後用 mask 的位元還原左半選法，右半的 mask 當下就有 ⇒ 兩半合併輸出。

複雜度分析// Complexity Derivation

步驟內容成本
左半建表2^half 個子集，各掃 half 位O(2^(N/2)·N)
排序對 2^half 筆 (和,mask) 排序O(2^(N/2)·N)
右半配對2^rest 個子集，各二分搜 O(N)O(2^(N/2)·N)
還原回報命中一次，逐位 ReportO(N)
總計 = O(2^(N/2) · N)
N=40 ⇒ 2²⁰≈10⁶，× N ≈ 4×10⁷，秒內可解；樸素 O(2⁴⁰)≈1.1×10¹² 直接 TLE
項目數量備註
TimeO(2^(N/2)·N)折半把指數砍成一半
SpaceO(2^(N/2))左半 (和,mask) 表，2²⁰ 筆
遞迴深度0（迭代）位元枚舉，無遞迴
Constant factor中排序 + 二分搜常數
03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
全程 long long K、A[i]、子集和都到 10¹⁸ ⇒ 一律 long long。ΣA ≤ 10¹⁸ < 9.2×10¹⁸，和不溢位。
1-indexed 回報 陣列 0-indexed，但 Report(id) 要 1..N ⇒ 回報時都 +1。左半 b+1、右半 half+b+1。
need < 0 要跳過 右半和 s 已超過 K 時 need=K−s<0，左半子集和非負，不可能命中 ⇒ continue 省一次二分搜。
記得存 mask 只存「和」無法還原選法。leftSum 要存 (和, mask)，命中後用 mask 還原左半。
找到就 return 題目只要任一組解 ⇒ 第一次命中就回報並 return，別繼續枚舉（也避免重複 Report）。
空子集 mask=0 枚舉含 mask=0（空集、和 0）是對的：例如 K 全靠另一半湊出時，這半就選空集。別跳過 0。
04

細節走讀// Section Walkthrough

① 切兩半

// split
half = N/2,  rest = N - half          // 例 N=3 → half=1, rest=2
左半 = 物品索引 [0, half)             // {A[0]}
右半 = 物品索引 [half, N)             // {A[1], A[2]}

② 左半建表 + 排序

// build left table
對 mask = 0 .. 2^half − 1：
  s = Σ A[b]  (b 在 mask 中)
  leftSum[mask] = (s, mask)           // 同時記和與選法
sort(leftSum)                          // 依 s 排序，準備二分搜

③ 右半配對（二分搜 K − s）

// pair right half
對 mask = 0 .. 2^rest − 1：
  s = Σ A[half+b]  (b 在 mask 中)
  need = K − s;  if (need < 0) continue
  在 leftSum 二分搜 first == need
  命中 → 回報左半 mask 的位 + 右半 mask 的位（皆 +1），Report(-1)，return

④ 為什麼不漏解

// completeness
任一合法解 S：S_L = S∩左半，S_R = S∩右半
當右半枚舉到 mask = S_R 時，need = K − sum(S_R) = sum(S_L)
而 sum(S_L) 一定在左半表中（S_L 是左半的某個子集）⇒ 必被找到。
