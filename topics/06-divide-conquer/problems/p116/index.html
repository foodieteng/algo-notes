01

題目簡介// Brief

QUESTION

給 N × M 防禦力矩陣（1 < N ≤ M ≤ 10⁶）。對每一列找重要守護點 — 該列唯一的最大值欄位。 題目保證：每列的最大值欄位嚴格大於上一列的最大值欄位（i.e. row-peak 欄位嚴格遞增）。

互動式：實作 solve(N, M)，judge 提供 int GetVal(row, col) 與 void Report(col)。 GetVal 呼叫上限 10⁸。

暴力 N · M = 10¹² 完全爆 — 必須利用「列峰嚴格遞增」這條結構性條件，把搜尋範圍逐列縮小。

03

核心策略 · 分治 on rows// Algorithm Outline

場景比喻。把矩陣想成一片戰場，每一列駐紮一隊太陽軍團，每隊都有一位隊長（= 該列的最大值）。題目保證：越下面的隊伍，隊長站得越靠右。所以把所有隊長的位置由上到下連起來，會看到一條從左上往右下走的階梯。

暴力為什麼爆？沒有任何結構假設，每列都要從第 1 格掃到第 M 格才能挖出隊長 — N · M 次 GetVal。N = M = 10⁶ ⇒ 10¹² 次，遠超 10⁸ 上限。

關鍵剪枝。「隊長階梯」這條約束，給了我們一個免費資訊：

只要先在正中間那列（mid）把隊長挖出來、記它的欄位為 m，那麼：
① 所有上半列的隊長一定全在 m 左邊（嚴格 <）
② 所有下半列的隊長一定全在 m 右邊（嚴格 >）
⇒ 上半之後只需搜 cols [colLo, m−1]、下半只需搜 cols [m+1, colHi] — 範圍立刻變窄，且兩段欄位互不重疊。

怎麼遞迴？上下半各自就是「N/2 列、更窄欄位範圍」的同形子問題 — 對它們繼續同一招：挑中間列、掃出隊長、切上下半。每往下一層列數對半切，最多切 log₂ N 層就到底。

整理成 4 步驟（當前處理 rowLo..rowHi 列、欄位範圍 colLo..colHi）：

  1. 挑中間列：mid = (rowLo + rowHi) / 2。
  2. 掃出 mid 列的隊長：在 cols [colLo, colHi] 內逐格比大小，找到峰欄 m，記下 answer[mid] = m。
  3. 遞迴上半：dnc(rowLo, mid−1, colLo, m−1) — 上半隊長必 < m，所以欄位上界縮到 m−1。
  4. 遞迴下半：dnc(mid+1, rowHi, m+1, colHi) — 下半隊長必 > m，所以欄位下界縮到 m+1。

Base case：當 rowLo > rowHi 或 colLo > colHi 就 return（沒有列要處理、或欄位範圍空了）。
Report 時機：遞迴順序是「中間 → 上半 → 下半」，不是 row 1..N，所以結果先存進 answer[]，等整個遞迴跑完，再依 row 順序 Report(answer[1..N])。

06

為什麼這樣切是對的// Correctness

// CORRECTNESS · 階梯結構保證範圍切得乾淨
  1. 掃完中間列得到隊長位置 m（這列的答案就鎖定了）。
  2. 上半列（i < mid）的隊長一定在 m 左邊 ⇒ 上半之後只能在 cols [colLo, m−1] 找。m 本身已經被中間列搶走，不會再是上半的答案。
  3. 下半列（i > mid）的隊長一定在 m 右邊 ⇒ 下半之後只能在 cols [m+1, colHi] 找。
  4. 每次遞迴把「列數對半切」（mid 切上下兩半），所以最多切 O(log N) 層就見底。
  5. 邊界情況：若 m == colLo，上半的欄位範圍變成空 — dnc 入口的 colLo > colHi 守備會自動 return，不會有問題。m == colHi 同理（下半變空）。

換句話說：「列峰嚴格遞增」就像一條從左上往右下走的階梯 — 知道中間那階站在哪，就立刻把整個搜尋空間切成「上半（只能在階梯左邊）」與「下半（只能在階梯右邊）」兩塊互不重疊的小區。

07

複雜度// Complexity

這題的真正限制是 GetVal 呼叫次數 ≤ 10⁸（不是時間）。所以我們直接數每次呼叫 GetVal 的次數，看會不會超預算。

觀察一：整支遞迴最多切 log₂ N 層就到底（每往下一層列數對半切）。N = 10⁶ ⇒ 約 20 層。

觀察二（關鍵）：把同一層所有子問題的 GetVal 加起來，總和不超過 M + (該層子問題數)。為什麼？

想像把同一層所有子問題的「欄位搜尋區間」並排畫出來：
[colLo₁, colHi₁]，[colLo₂, colHi₂]，[colLo₃, colHi₃]...

因為上下半切割是 [..., m−1] 和 [m+1, ...]，相鄰兩個子問題完全不重疊（中間還少一格 m）。

把它們攤平在「欄位 1 到 M」這條線上，總長度加起來 ≤ M。所以該層所有 GetVal 加起來 ≤ M。

結論：每層 ≤ M 次 GetVal × log₂ N 層 = M · log₂ N 次。加上掃過每列至少 1 次的成本（共 N 次），總 GetVal ≤ (N + M) · log₂ N。

GetVal 次數 ≤ (N + M) · log₂ N
N = M = 10⁶ ⇒ 2 × 10⁶ × 20 = 4 × 10⁷ 次 ⇒ 遠在 10⁸ 上限內 ●
項目數量白話備註
TimeO((N + M) log N)同上面 GetVal 推導，每次 GetVal 是 O(1)
GetVal 次數≤ (N + M) log₂ NN = M = 10⁶ ⇒ ≈ 4 × 10⁷，安全
SpaceO(N)answer[1..N] 存每列峰欄；遞迴堆疊 O(log N)
遞迴深度O(log N)N = 10⁶ ⇒ 約 20 層

數字直覺：如果不用分治、每列都掃整排 M 格 ⇒ N × M = 10¹² 次（暴力 1 兆），絕對爆。分治版本壓到 4 千萬次，差距 5 個數量級。

08

一句洞察// One-Sentence Insight

「列峰嚴格遞增」這條結構性條件，把每層遞迴的欄位總長壓在 O(M)。
所以 D&C on rows 不是「砍列數」，而是用列峰位置把欄位範圍切開 — 砍的是欄位，省的是 GetVal。
