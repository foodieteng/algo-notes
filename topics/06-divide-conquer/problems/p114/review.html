重點回顧// Concept Recap

關鍵點內容
數學基礎 4 ≡ 1 (mod 3) ⇒ 4ᴷ − 1 ≡ 0 (mod 3)，L 數量 (N² − 1) / 3 必為整數。
分治架構 切 4 個 N/2 × N/2 象限；blocked 必落在恰好 1 個象限。
中央 L 的作用 在「不含 blocked」3 個象限角落各種 1 cell — 成為各象限的新 blocked，子問題完全同形。
Base case n = 2 時，4 格扣 blocked 剩 3 格必為 L，直接 Report。
時間複雜度 T(N) = 4·T(N/2) + O(1) ⇒ Θ(N²)（Master Theorem case 1）。
空間複雜度 遞迴深度 O(log N) = K；N = 1024 只有 10 層。

範例 Trace · solve(4, 1, 1) 完整 5 個 Report// Worked Trace

用 N = 4, X = 1, Y = 1（blocked 在左上角）跑一遍 — 共會輸出 (4² − 1) / 3 = 5 個 Report。

// CALL 1 · tile(1, 1, 4, 1, 1)
n = 4 > 2 → 不是 base case
h  = 2
mx = 1 + 2 − 1 = 2
my = 1 + 2 − 1 = 2

bq = (1 > 2) * 2 + (1 > 2) = 0 * 2 + 0 = 0   → TL

4 個中心角：
  q=0 (TL): (2, 2)
  q=1 (TR): (2, 3)
  q=2 (BL): (3, 2)
  q=3 (BR): (3, 3)

中央 L = q ∈ {1, 2, 3} 的角：(2,3), (3,2), (3,3)
▸ Report(2, 3,  3, 2,  3, 3)

遞迴 4 個象限：
  TL (q=0)  tile(1, 1, 2, 1, 1)   ← 用原 blocked
  TR (q=1)  tile(1, 3, 2, 2, 3)   ← 用剛種的 (2,3)
  BL (q=2)  tile(3, 1, 2, 3, 2)   ← 用剛種的 (3,2)
  BR (q=3)  tile(3, 3, 2, 3, 3)   ← 用剛種的 (3,3)
// CALL 2 · tile(1, 1, 2, 1, 1)  →  base case
n = 2 → 命中 base
掃 (1,1), (1,2), (2,1), (2,2)，跳過 blocked (1,1)
▸ Report(1, 2,  2, 1,  2, 2)
// CALL 3 · tile(1, 3, 2, 2, 3)  →  base case
掃 (1,3), (1,4), (2,3), (2,4)，跳過 blocked (2,3)
▸ Report(1, 3,  1, 4,  2, 4)
// CALL 4 · tile(3, 1, 2, 3, 2)  →  base case
掃 (3,1), (3,2), (4,1), (4,2)，跳過 blocked (3,2)
▸ Report(3, 1,  4, 1,  4, 2)
// CALL 5 · tile(3, 3, 2, 3, 3)  →  base case
掃 (3,3), (3,4), (4,3), (4,4)，跳過 blocked (3,3)
▸ Report(3, 4,  4, 3,  4, 4)
// OUTPUT  ─  共 5 次 Report
Report(2, 3,  3, 2,  3, 3)
Report(1, 2,  2, 1,  2, 2)
Report(1, 3,  1, 4,  2, 4)
Report(3, 1,  4, 1,  4, 2)
Report(3, 4,  4, 3,  4, 4)
觀察：DFS 走訪下，中央 L 永遠最先 Report（因為它在遞迴前印出）； 接著是 TL → TR → BL → BR 順序的子板。每個子板在自己呼叫裡也是先 Report 中央 L 再遞迴更深層 — 直到 n = 2 base case。

應用題// Quick Calculations

A1 數量

N = 8、16、1024 時，各要放幾個 L？

A2 遞迴深度

N = 1024 時，tile() 的最大遞迴深度是？

A3 象限定位

給 N = 8, X = 5, Y = 7，第 1 層 tile 中 bq 等於多少？blocked 落在哪個象限？中央 L 的 3 個 cell 是？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、A 對應 A。

A1 · 數學

4 ≡ 1 (mod 3)，所以 4ᴷ ≡ 1ᴷ ≡ 1 (mod 3)， 因此 4ᴷ − 1 ≡ 0 (mod 3)，即 (N² − 1) / 3 必為整數。

這保證 L 數量是「乾淨的整數」，分配方案有可能存在 — 接下來用構造法直接寫出來。

A2 · 分治

程式用 bq 把 blocked 所在象限編號出來， 放中央 L 時用 if (q == bq) continue; 主動跳過它。 因此中央 L 的 3 個 cell 必然落在不含 blocked 的 3 個象限— 與原 blocked 所在象限完全不交集。

A3 · 構造

放完中央 L 後：

  • 含原 blocked 的象限：仍只有 1 個 blocked（原本的 (X, Y)）。
  • 其他 3 個象限：被中央 L 在最角落各種了 1 個 cell — 那一格就成為該象限的「新 blocked」。

每個子象限都是「N/2 × N/2 板 + 恰好 1 個 blocked」，與原問題同形。 ⇒ 可以安全遞迴。

A4 · Base case

2×2 中任意 3 個格子不可能共線，因為：

  • 同一列只有 2 格（c 從 1 到 2）。
  • 同一行只有 2 格。

因此扣 1 格剩 3 格時，這 3 格必然包含至少一個轉折 — 就是 L 型。

A5 · 複雜度

Master Theorem 形式：T(N) = a·T(N/b) + f(N)，這裡 a = 4, b = 2, f(N) = O(1) = O(N⁰)。

計算 log_b a = log₂ 4 = 2。 因為 f(N) = O(N⁰) 且 0 < 2，進入 case 1 ⇒ T(N) = Θ(N^{log_b a}) = Θ(N²)。

直覺：總共要放 (N² − 1) / 3 = Θ(N²) 個 L，每個 L 是 O(1) 工作 ⇒ Θ(N²)。

A · App 1

套公式 (N² − 1) / 3：

  • N = 8    ⇒ (64 − 1) / 3 = 21 個 L
  • N = 16   ⇒ (256 − 1) / 3 = 85 個 L
  • N = 1024 ⇒ (1,048,576 − 1) / 3 = 349,525 個 L

實作上一次 solve 就是這麼多次 Report 呼叫。

A · App 2

遞迴深度 = log₂ N。 N = 1024 = 2¹⁰，所以最深 10 層（1024 → 512 → 256 → ... → 4 → 2）。 遠遠在預設 stack 大小可承受範圍內，不需要改 stack 或改 iterative。

A · App 3

N = 8, h = 4, mx = 1 + 4 − 1 = 4, my = 1 + 4 − 1 = 4：

// step 1
bq = (5 > 4) * 2 + (7 > 4)
   = 1 * 2 + 1
   = 3                       → BR 象限

4 個中心角：
  q=0 (TL): (4, 4)
  q=1 (TR): (4, 5)
  q=2 (BL): (5, 4)
  q=3 (BR): (5, 5)   ← skip (q == bq)

中央 L = q ∈ {0, 1, 2} 的角：(4,4), (4,5), (5,4)
▸ Report(4, 4,  4, 5,  5, 4)

blocked (5, 7) 位於 BR (右下) 象限。
