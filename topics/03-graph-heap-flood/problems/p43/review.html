重點回顧// Concept Recap

關鍵點內容
問題本質 無權圖（每步成本 1）的單源最短路到任一終點。
選 BFS FIFO queue 逐層擴散；DFS 不保證最短，Dijkstra 是 overkill。
dist 單調不減 queue 內 dist 最多差 1 ⇒ 取出順序 = 距離由小到大。
第一個取出的 @ 它的 dist 即全場最小 ⇒ 立刻回傳，判斷放在 pop。
入列即定 dist dist != -1 兼當 visited，避免同格重複入列。
走不到 queue 空仍未碰 @ ⇒ 印 = ="。

範例 Trace · 跑一次 code 看會輸出什麼// Worked Trace

用動畫那張 5×7 迷宮（座標 (row, col) 0-indexed）。貓 K 在 (1,1)，老鼠 @ 在 (1,4) 與 (3,3)：

// INPUT (這一筆)5 7
#######
#K..@.#
#.##..#
#..@..#
#######

BFS 從 K(1,1) 出發，dist[1][1]=0 入列。以下每行是「pop 一格 → 推鄰居」：

// pop (1,1) dist 0
鄰居可走且未訪：(1,2) (2,1)
→ dist 設 1，push
queue: [ (1,2)d1, (2,1)d1 ]
// pop (1,2) dist 1
上(0,2)# 下(2,2)# 左(1,1)已訪 右(1,3)未訪
→ push (1,3) dist 2
queue: [ (2,1)d1, (1,3)d2 ]
// pop (2,1) dist 1
上(1,1)已訪 下(3,1)未訪 左(2,0)# 右(2,2)#
→ push (3,1) dist 2
queue: [ (1,3)d2, (3,1)d2 ]
// pop (1,3) dist 2
上(0,3)# 下(2,3)# 左(1,2)已訪 右(1,4)='@' 未訪
→ push (1,4) dist 3
queue: [ (3,1)d2, (1,4)d3 ]
// pop (3,1) dist 2
上(2,1)已訪 下(4,1)# 左(3,0)# 右(3,2)未訪
→ push (3,2) dist 3
queue: [ (1,4)d3, (3,2)d3 ]
// pop (1,4) dist 3  ← grid 是 '@'！
ans = dist[1][4] = 3
break（不再處理另一隻 @(3,3)，它更遠）
// OUTPUT3
觀察：另一隻老鼠 (3,3) 距離是 4 （經 (3,1)→(3,2)→(3,3)）。但 BFS 在 dist 3 那層就先取出 (1,4) 並 break， 根本沒去碰更遠的那隻 — 這就是「第一個 pop 的 @ 必最近」省下的工。

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · BFS vs DFS

BFS 用 FIFO queue，逐層擴散：先把 dist=k 的格全處理完，才進 dist=k+1。 所以第一次碰到終點時走的步數一定最少。

DFS 用 stack（或遞迴），會「一條路衝到底」再回溯。第一次碰到 @ 的路徑長度 取決於它選的方向順序，未必最短；要保證最短得遍歷所有路徑 ⇒ 指數級。 所以無權最短路一律用 BFS。

Q2 · 終點判斷時機

放在出列（pop）。因為「pop 順序 = dist 由小到大」，pop 到的第一個 @ 保證最近。 

// 若放在 push 就回傳 → 可能錯
擴散方向的關係，一個 dist 較大的 @ 可能比 dist 較小的 @ 「先被 push」
push 順序 ≠ 距離排序
∴ 必須等 pop 時才確認最近

（本題終點是 @ 格；起點 K 不是終點，所以 pop K 時不會誤判。）

Q3 · dist 與 visited

用 dist[i][j] == -1 代表「未訪」。一格被入列的當下就把 dist 設成非 −1 的值， 等於同時標記了 visited。之後任何鄰居再看到它都因 dist != -1 而略過。

這樣夠，是因為 BFS 中一格第一次被設定的 dist 就是它的最短距離， 永遠不需要更新 ⇒ 不必區分「訪過但還能更短」的狀態，一個 dist 陣列就身兼兩職。

Q4 · 複雜度

每格恰好 push 一次（入列即定 dist，之後不再入列）。pop 後看固定 4 個方向 ⇒ 每格 O(1)。 所有格加總 ≤ N×M ⇒ 整段 BFS 是 O(N × M)。

會誤以為 O((NM)²) 通常是把「每格都重新掃整張圖」當成本；但有了 dist/visited， 每格只處理一次，沒有重複。

Q5 · 多筆測資

必清 dist（每筆讀圖時設回 −1）。grid 整張重讀會覆蓋、不必特別清； qx/qy 靠每筆開頭 head=tail=0 歸零。

若漏清 dist：上一筆設過的格在這一筆被當「已訪」⇒ BFS 擴散不出去 ⇒ 可能誤判走不到 ⇒ 答案錯（偏大或印 = ="）。

App 1 · 多源 BFS

把所有貓在 BFS 開始前同時入列、各自 dist = 0。 之後照常擴散，dist[i][j] 自然就是「離最近的貓多遠」。 

// multi-source BFS
for 每個貓 (cx,cy):
    dist[cx][cy] = 0
    push (cx,cy)
// 接著同一個 while 迴圈擴散，無需改動

複雜度仍是 O(N×M)，因為每格還是只入列一次。

App 2 · 無解判定

#K..#...@#（3×10，中間列）：貓在 (1,1)，往右走到 (1,3) 後 (1,4)='#' 擋住；上下兩列整排 # 繞不過去 ⇒ 到不了 (1,8) 的老鼠。

BFS queue 把左側能走的幾格擴散完就空了，ans 仍是 −1。 負責輸出的是這行： 

// 無解if (ans == -1) cout << "= =\"" << '\n';
App 3 · 輸出陷阱

要印出的 4 個字元是：等號、空白、等號、雙引號 → = ="。 雙引號在 C++ 字串裡要跳脫成 \"： 

// 正確寫法cout << "= =\"" << '\n';
// 等價：cout << "= =" << '"' << '\n';

漏掉跳脫（寫成 "= =""）會被編譯器當成兩個相鄰字串拼接，少印一個雙引號。
