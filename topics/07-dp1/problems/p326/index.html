PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

大魔王生命值上限 M、每秒回復 R（不超過 M），沒有攻擊力。實實有 N 種咒術， 每種只能施放一次：第 i 種當大魔王生命值 ≤ pᵢ% × M 時才可施放，施放後每秒固定造成 cᵢ 點傷害。

每秒流程（從第 0 秒起）：① 已施放的咒術一起扣血 → ② 大魔王回血 R（封頂 M）→ ③ 若生命值 ≤ 0 即被打敗 → ④ 實實可選擇施放至多一種咒術。 求最快第幾秒能打敗，或輸出 −1。

1 ≤ N,M,R ≤ 10⁵，0 ≤ cᵢ ≤ 10⁵，0 ≤ pᵢ ≤ 100。範例：M=100,R=10，咒術(100,11)、(90,9) ⇒ 答案 19。

兩個關鍵：咒術傷害是持續的（一旦施放每秒都扣 cᵢ，所以越早放越好）；低 pᵢ 的咒術被鎖住， 要等生命值掉到門檻才能放。所以策略是「能放就放、放最強的」，靠強咒把血壓低去解鎖更多咒術。

02

為什麼用貪心模擬// Why Greedy Simulation

A 枚舉施放順序 / 時間 O(N! · …)
「每種何時放」的組合是階乘級；N=10⁵ 完全不可能列舉。
× REJECT · TLE
B 每秒重掃 N 個咒術找最強 O(N · 答案)
答案可達 ~M=10⁵ 秒，每秒掃 N=10⁵ ⇒ 10¹⁰ 次，會 TLE。邏輯對但太慢。
△ REJECT · TLE
BOTH FAIL · ADOPT
C 貪心逐秒模擬 + 堆加速 O((M+N) log N)
咒術按門檻排序；血量下降時把新解鎖的丟進最大堆（以 c 為鍵），每秒彈出最強來放。早停判 −1。
● ADOPT · STABLE

為什麼貪心對？ 傷害是持續累加、且只增不減：多放一個咒術、或更早放，絕不會讓打敗時間變晚（無後效、單調）。 所以「每秒在可放的咒術裡放 c 最大的」就是最佳；不需 DP。

03

核心策略 · 三步驟// Strategy

  1. 排序 · 咒術按解鎖門檻 pᵢ·M 由大到小（pᵢ 大 ⇒ 血量還高時就能放）。
  2. 逐秒模擬 · 每秒：扣血(dps) → 回血(封頂 M) → 判死 → 把所有「血量已達門檻」的咒術推進最大堆，彈出 c 最大者施放，dps 累加。
  3. 判 −1 · 若某秒沒有可放的咒術且 dps ≤ R，血量不會再下降 ⇒ 永遠打不死，輸出 −1。
每秒：hp ← min(M, hp − dps + R)；可放時 dps += max c（解鎖的咒術裡）
有限答案 ≤ M + N（dps>R 後每秒至少掉 1，最多 M 秒死；放咒至多 N 次）⇒ 可當作迴圈上界 / 早停依據

05

為什麼「能放就放最強」最佳// Correctness

// GREEDY EXCHANGE · 早放、放強都不吃虧
  1. 傷害只增不減：咒術一旦施放，往後每秒都扣 cᵢ。所以任何一個咒術，越早施放，到任何時刻累積的傷害都 ≥ 晚放。
  2. 把某次施放提前不會更糟：若最佳解在第 t 秒才放某可用咒術，把它提前到「最早可放的那秒」，每一秒的生命值都 ≤ 原解 ⇒ 打敗時間不會變晚（交換論證）。
  3. 同一秒該放最強：那一秒只能放一個，選 c 最大的能讓往後每秒扣最多、血掉最快，解鎖後續咒術也最快 ⇒ 不會比放較弱的差。
  4. 合起來：每秒在已解鎖的咒術裡放 c 最大的，就得到最早打敗時間。
  5. 判 −1：若某秒無咒可放且 dps ≤ R，扣血 ≤ 回血 ⇒ 生命值不再下降，永遠到不了更低門檻、也死不了。

注意門檻條件是「生命值 ≤ pᵢ%·M 才可放」（題目：生命值大於 pᵢ% 時無法施放）。 整數判斷用 hp × 100 ≤ pᵢ × M 最穩，避免百分比除法誤差。
