01

題目簡介// Brief

QUESTION

給 N × M 防禦力矩陣（1 < N ≤ M ≤ 10⁶）。對每一列找重要守護點 — 該列唯一的最大值欄位。 題目保證：每列的最大值欄位嚴格大於上一列的最大值欄位（i.e. row-peak 欄位嚴格遞增）。

互動式：實作 solve(N, M)，judge 提供 int GetVal(row, col) 與 void Report(col)。 GetVal 呼叫上限 10⁸。

暴力 N · M = 10¹² 完全爆 — 必須利用「列峰嚴格遞增」這條結構性條件，把搜尋範圍逐列縮小。

B 每列二分搜尋 O(N log M)
列中沒有單調性，數值是任意排列。binary search 找不到峰 — 結果錯。
△ REJECT · WRONG
BOTH FAIL · ADOPT
C 分治 on rows + 欄位範圍縮減 O((N+M) log N)
取中間列直接掃 → 利用單調性把上下半的欄位範圍各自縮小 → 遞迴。N = M = 10⁶ 約 4 × 10⁷ 次 GetVal。
● ADOPT · FITS BUDGET
03

核心策略 · 分治 on rows// Algorithm Outline

dnc(rowLo, rowHi, colLo, colHi) 做 4 件事：

  1. 挑中間列 · mid = (rowLo + rowHi) / 2
  2. 直接掃 · 在欄位範圍 [colLo, colHi] 內 brute force 找 row mid 的峰欄 m，把 answer[mid] = m
  3. 遞迴上半 · dnc(rowLo, mid−1, colLo, m)（上方列峰必在 m 左側或等於 m）
  4. 遞迴下半 · dnc(mid+1, rowHi, m, colHi)（下方列峰必在 m 右側或等於 m）

遞迴順序不是 row 1..N，所以 Report 要等遞迴結束、依 answer[] 順序輸出。

06

為什麼這樣切是對的// Correctness

// CORRECTNESS · 列峰嚴格遞增 ⇒ 範圍縮減
  1. 令 row mid 的峰欄為 m。
  2. 對任意 i < mid，題目保證 row i 的峰欄 p_i 嚴格小於 row mid 的峰欄 ⇒ p_i ≤ m − 1 < m。 所以上半的搜尋範圍可安全縮為 [colLo, m]（包含端點 m 是為了程式碼一致；實際 p_i ≤ m−1）。
  3. 對任意 i > mid，題目保證 p_i > m ⇒ 下半搜尋範圍縮為 [m, colHi]。
  4. 每次遞迴列數對半切（mid 切上下兩半），所以深度為 O(log N)。

換句話說：「列峰嚴格遞增」就像一條從左上往右下走的階梯 — 知道中間那階在哪，就把搜尋範圍切成「不可能在右邊的上半」與「不可能在左邊的下半」。

07

複雜度// Complexity

每一層遞迴的所有子問題加起來掃過的欄位總長 ≤ M + (該層子問題數) — 因為相鄰子問題的欄位範圍只在端點 m 重疊 1 格。

T(N, M) ≤ 2 · T(N/2, ?) + O(M_layer)
每層欄位總長 = O(M)，共 O(log N) 層 ⇒ Total = O((N + M) log N)
項目數量備註
TimeO((N + M) log N)遞迴每層的欄位總長 ≤ M + 子問題數
SpaceO(N)answer[1..N] 暫存；遞迴堆疊 O(log N)
Recursion DepthO(log N)N = 10⁶ ⇒ 約 20 層
GetVal Calls≤ (N + M) log₂ NN = M = 10⁶ ⇒ ≈ 2×10⁶ × 20 = 4 × 10⁷，遠在 10⁸ 上限內

GetVal 預算才是這題的真正限制 — 不是時間複雜度。10⁸ 的限制意味著就算單列 brute O(M) 也只能用 100 次（N ≤ 100 的子任務），全題必須靠遞迴縮減欄位範圍。

08

一句洞察// One-Sentence Insight

「列峰嚴格遞增」這條結構性條件，把每層遞迴的欄位總長壓在 O(M)。
所以 D&C on rows 不是「砍列數」，而是用列峰位置把欄位範圍切開 — 砍的是欄位，省的是 GetVal。
