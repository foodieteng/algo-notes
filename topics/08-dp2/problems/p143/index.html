高棕櫚農場

// 0/1 背包 · 一維滾動陣列（容量倒序）

★ ★ ★ ☆ ☆ 核心 0/1 背包（飽足感=重量、滿足感=價值） · 時間 O(N·M)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

吃到飽農場有 N 個高棕櫚，第 i 個吃掉會得到「飽足感 Aᵢ」與「滿足感 Bᵢ」。 円円的胃容量有限：總飽足感不能超過 M。每個高棕櫚只能吃一次（拿或不拿）。

問在飽足感 ≤ M 的前提下，滿足感總和最大是多少？ 多筆 T ≤ 10，1 ≤ N ≤ 100，1 ≤ M ≤ 10⁶，1 ≤ Aᵢ ≤ 10⁵，1 ≤ Bᵢ ≤ 100。

換個說法就是經典 0/1 背包：背包容量 = 胃容量 M、物品重量 = 飽足感 Aᵢ、物品價值 = 滿足感 Bᵢ。 範例 M=5、物品 (2,4)、(3,3)：兩個都吃飽足 2+3=5 ≤ 5、滿足 4+3=7。

02

為什麼用 0/1 背包 DP// Why Knapsack DP

A 枚舉每個吃不吃 O(2ᴺ)
每個高棕櫚拿/不拿是 2 種，N=100 ⇒ 2¹⁰⁰ 種子集，天荒地老也跑不完。
× REJECT · TLE
B 二維 dp[i][w] O(N·M) 空間
dp[i][w] 表大小 100×10⁶ = 10⁸ 格 ⇒ 記憶體爆掉（~400MB+）。時間對、空間不行。
△ REJECT · MLE
BOTH FAIL · ADOPT
C 一維滾動陣列（容量倒序） O(N·M) 時間 · O(M) 空間
只留一條 dp[w]，逐物品更新；w 由大到小掃，保證每個物品只被拿一次。容量上界取 min(M, ΣAᵢ)。
● ADOPT · STABLE

為什麼一維就夠？ 二維的 dp[i][w] 只依賴上一列 dp[i−1][·]，把列壓掉、原地更新即可。 關鍵是 w 倒序——這樣更新 dp[w] 用到的 dp[w−Aᵢ] 還是「上一輪（沒放第 i 個）」的值，物品才不會被重複拿。

03

狀態與轉移// State & Transition

dp[w] = 在飽足感恰好用 ≤ w 的限制下，能得到的最大滿足感。

dp[w] = max( dp[w] , dp[w − Aᵢ] + Bᵢ )　（w 由大到小，w ≥ Aᵢ）
dp[w] = 不吃第 i 個（沿用上一輪）；dp[w−Aᵢ]+Bᵢ = 吃第 i 個（先騰出 Aᵢ 空間再加 Bᵢ）。取較大者。
  1. 初始 · dp[w] = 0（什麼都沒吃，滿足感 0）。
  2. 容量上界 · cap = min(M, ΣAᵢ)——容量超過所有東西的總重量沒意義，可縮小表。
  3. 逐物品 × 倒序容量 · 外層跑 i，內層 w 從 cap 到 Aᵢ 更新。
  4. 答案 · dp[cap]。

05

為什麼 w 一定要由大到小// Why Descending

// 倒序 = 每個物品只拿一次（0/1 的關鍵）
  1. 一維 dp[w] 在更新第 i 個物品時，dp[w−Aᵢ] 應該代表「還沒考慮第 i 個」的最佳值（即二維的 dp[i−1][w−Aᵢ]）。
  2. 倒序（w 從大到小）：更新 dp[w] 時，dp[w−Aᵢ]（較小的 w）本輪還沒被改到 ⇒ 仍是上一輪的值 ⇒ 第 i 個只被拿一次。✓
  3. 正序（w 從小到大）：更新 dp[w] 前，dp[w−Aᵢ] 本輪已被改過（可能已含第 i 個）⇒ 第 i 個會被重複拿 ⇒ 變成「完全背包」，答案錯。✗
  4. 所以「倒序 = 0/1 背包、正序 = 完全背包」——差一個迴圈方向，意義完全不同。

記法：0/1 背包倒著掃、完全背包順著掃。這題每個高棕櫚只能吃一次，所以倒序。

06

複雜度// Complexity

N 個物品 × 容量 cap = O(N · min(M, ΣAᵢ)) 每筆
N ≤ 100、cap ≤ M ≤ 10⁶ ⇒ ~10⁸；T ≤ 10 但常數極小（一次加法一次比較），實測 < 0.5s
項目數量備註
TimeO(N·M) 每筆兩層迴圈 i / w
SpaceO(M)一維 dp[] 滾動；二維會 MLE
遞迴深度0（迭代）純填表
Constant factor低每格一次 max

一句洞察// One-Sentence Insight

飽足感 = 重量、滿足感 = 價值、胃容量 = 背包 ⇒ 標準 0/1 背包。
