01

重點回顧// Concept Recap

關鍵點內容
建模 人 = 點、卦源 = 無向邊；「人人異性」⇔ 圖可二著色（二分圖）。
判定核心 二分圖 ⇔ 沒有奇環。BFS 沿邊塗反色，撞到同色相鄰即奇環。
不連通處理 對每個 color == −1 的點各跑一次 BFS。
輸出 全塗完無衝突 ⇒ NORMAL.；任一衝突 ⇒ RAINBOW.。
複雜度 O(n + m) 每筆；多筆測資各自重置。
實作細節 用 BFS 陣列佇列（避免遞迴爆 stack）；反色用 color[u] ^ 1。

範例 Trace · 跑樣本第 1 筆// Worked Trace

樣本第 1 筆：n = 3、m = 3，卦源 0–1 · 1–2 · 2–0（三角形）。

// INPUT (第 1 筆)3
3
0 1
1 2
2 0

建好的鄰接表（無向，兩邊都加）：

// adjacency
adj[0] = [1, 2]
adj[1] = [0, 2]
adj[2] = [1, 0]
color  = [-1, -1, -1]

主迴圈：color[0] == −1 ⇒ 呼叫 canColor(0)：

// BFS · 推入起點 0
color[0] = 0
queue = [0]
// pop 0 (color 0)
看 adj[0] = [1, 2]
  v=1: color[1] == -1 → 塗 color[0]^1 = 1，推進 queue
  v=2: color[2] == -1 → 塗 color[0]^1 = 1，推進 queue
queue = [1, 2]   color = [0, 1, 1, ...]
// pop 1 (color 1)
看 adj[1] = [0, 2]
  v=0: color[0]=0 ≠ color[1]=1 → OK（這是來時的樹邊）
  v=2: color[2]=1 == color[1]=1 → 同色相鄰！return false
// OUTPUT (第 1 筆)canColor(0) = false → ok = false → RAINBOW.
觀察：點 1 和點 2 都被塗成顏色 1（因為都離起點 0 距離 1）， 但 1–2 之間本來就有一條卦源邊 — 同色卻相鄰，正是把三角形這個奇環封口的那條邊。 三人不可能兩兩異性 ⇒ RAINBOW。

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

A1 · 建模

每個人是一個點；每對卦源是一條無向邊。「人人異性戀、每對卦源異性」代表： 每個人被指派一種性別（兩種選擇），每條邊兩端必須不同性別。

把兩種性別看成兩種顏色，問題就是「能不能用 2 種顏色塗所有點，使每條邊兩端異色」—— 這正是二分圖（bipartite）的定義。能 ⇒ NORMAL；不能 ⇒ RAINBOW。

A2 · 奇環

沿一個環走，顏色必須 0,1,0,1,… 交替。要走一圈回到起點且顏色一致，環長必須是偶數。 若環長是奇數，繞一圈後顏色會跟起點相反 ⇒ 矛盾 ⇒ 不可二著色。

三角形是長度 3 的環（最短奇環）：A 異於 B、B 異於 C ⇒ A 同 C，但 A–C 又是一對卦源要求異性 ⇒ 衝突 ⇒ RAINBOW。

A3 · 不連通

圖可能有多個連通塊。只從點 0 跑一次 BFS，只會著色點 0 所在那塊，其他塊完全沒檢查。 

// 會被坑的測資
n = 5, m = 3
邊: 0–1   (塊 A：一條邊，OK)
    2–3, 3–4, 4–2  (塊 B：三角形 → 應 RAINBOW)
只從 0 跑 ⇒ 只看到塊 A ⇒ 誤判 NORMAL ❌

所以必須 for i in [0,n): if color[i]==-1: canColor(i)，逐塊檢查。

A4 · DFS vs BFS

DFS 染色邏輯一樣正確，但遞迴深度可達 O(n)。n = 10⁵ 的長鏈 0–1–2–…–99999 會遞迴十萬層 ⇒ stack overflow（RE）。

BFS 用一個顯式的陣列佇列，深度永遠 O(1) 的呼叫堆疊、佇列吃 heap 空間 ⇒ 不會爆。 （想用 DFS 也可以，但要改成顯式 stack 的迭代版。）

A5 · 著色語意

^ 1 是位元 XOR：0 ^ 1 = 1、1 ^ 1 = 0，剛好把顏色翻面。 所以 color[v] = color[u] ^ 1 就是「把 v 塗成 u 的反色」。

不會塗錯，是因為我們只在 color[v] == −1（還沒塗）時才塗； 已經塗過的點不重塗，只拿來檢查是否衝突。每個點的顏色一旦定下就固定，等於記下它到起點距離的奇偶。

A · App 1

4 環是偶環，輸出 NORMAL.。BFS 從 0 塗： 0→0、1→1、3→1、2→0。最後邊 1–2 連的是 color 1 與 color 0，異色 OK； 2–3 連 color 0 與 color 1，也 OK。沒有任何同色相鄰 ⇒ 可二著色。

性別指派：{0,2} 一種性別、{1,3} 另一種，完全自洽。

A · App 2

單筆 O(n + m) = O(10⁵ + 2×10⁵) = 3 × 10⁵；T = 10 筆 ⇒ 約 3 × 10⁶ 次基本運算。

遠在 3 秒時限內（時限 3000ms），不會 TLE。瓶頸反而是 I/O — 記得 ios::sync_with_stdio(false); cin.tie(nullptr); 與用 '\n' 不用 endl。

A · App 3

「最少刪幾條邊使圖變二分圖」是 Maximum Bipartite Subgraph / Edge Bipartization， 在一般圖上是 NP-hard，沒有已知多項式解。

所以不能再用這招簡單解決 —— 本題只問「是否可能」（判定），是 O(n+m) 的簡單版； 一旦改問「最少修改量」就跳進難題範疇。判定與最佳化的難度落差，是這類題的常見陷阱。
