玩電梯

// 計數 DP · 區間轉移用差分陣列加速

★ ★ ★ ★ ☆ 核心 dp[趟數][樓層] + 差分區間加 · 時間 O(N·K)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

n 層樓，你在樓層 a，樓層 b 是禁止進入的研究室。你想連續搭 K 趟超級電梯。 每趟從樓層 x 搭到 y，要求 |y − x| < |b − x|（搭乘距離小於你到 b 的距離 ⇒ 永遠跳不到也跳不過 b）。

第 i+1 趟的起點 = 第 i 趟的終點（連續鏈接）。兩種搭法視為相同 ⟺ 每一趟所在樓層都一樣。 問有幾種不同搭法，mod 1000000007。2 ≤ n ≤ 2000，a ≠ b，1 ≤ K ≤ 2000。

白話：一個「搭法」就是樓層序列 a = f₀, f₁, …, f_K，每步 |fᵢ − fᵢ₋₁| < |b − fᵢ₋₁|。 這條件保證每趟都跳不到 b（若 y=b 則 |y−x|=|b−x| 不 <），所以自動避開禁止樓層。求序列數。

02

為什麼用計數 DP + 差分// Why DP + Diff Array

A 枚舉所有樓層序列 O(nᴷ)
每趟都有近 n 種落點，K 趟 ⇒ 指數爆炸；n,K=2000 完全不可能。
× REJECT · TLE
B 樸素 DP，每步逐對轉移 O(N²·K)
dp[s][f] 對，但每步對每個 x 掃所有 y ⇒ n²·K = 2000²·2000 ≈ 1.6×10¹⁰，TLE。
△ REJECT · TLE
BOTH FAIL · ADOPT
C DP + 差分陣列做區間加 O(N·K)
關鍵觀察：從 x 出發的合法落點 y 是一段連續區間 [x−d+1, x+d−1]（d=|x−b|）。用差分陣列一次 O(1) 加整段，每步 O(n)。
● ADOPT · STABLE

核心觀察：|y − x| < d（d = |x − b|）⟺ y ∈ [x−d+1, x+d−1]，是以 x 為中心的連續區間（再扣掉 y=x）。 「把一個值加到一段連續區間」正是差分陣列的拿手好戲，把每步從 O(n²) 壓到 O(n)。

03

核心策略 · 三步驟// Strategy

cur[f] = 搭完目前這 s 趟後，停在樓層 f 的方法數。每趟做一次轉移：

  1. 算區間 · 對每個樓層 x，落點範圍是 [max(1, x−d+1), min(n, x+d−1)]，其中 d = |x − b|。
  2. 差分加 · 把 cur[x] 加到該區間：diff[lo] += cur[x]、diff[hi+1] −= cur[x]。
  3. 前綴和 + 扣自身 · 對 diff 取前綴和得 nxt[]；再把每個 nxt[x] −= cur[x]（因為電梯要動，y ≠ x）。
nxt[y] = Σx : |y−x| < |b−x|, y≠x cur[x]　（用差分 O(n) 完成整步）
重複 K 趟；答案 = Σ_f cur[f]（停在任一樓層都算一種搭法）。MOD = 1000000007（質數）。

05

為什麼合法落點是一段連續區間// Why a Contiguous Range

// 條件 |y−x| < |b−x| 展開 = 區間
  1. 令 d = |x − b|（x 到禁止樓層的距離）。條件 |y − x| < d 就是「y 離 x 的距離 < d」。
  2. 展開：−d < y − x < d ⟺ x − d < y < x + d ⟺ y ∈ [x−d+1, x+d−1]（整數）。一段以 x 為中心的連續區間。
  3. 這區間不會碰到 b：b 在距離 d 處，而區間最遠只到 d−1，所以 b 永遠在區間外 ⇒ 自動避開禁止樓層，不必特判。
  4. 區間內含 x 自己（|x−x|=0<d），但電梯要動 ⇒ 最後把 y=x 的貢獻扣掉。
  5. 「把 cur[x] 加到一段連續區間」⇒ 用差分陣列：diff[lo]+=v, diff[hi+1]−=v，全部加完再前綴和，每步 O(n)。

沒有差分的話，每個 x 要逐一加到 O(n) 個 y，整步 O(n²)；差分把「區間加」變 O(1)，整步降到 O(n)，K 趟總共 O(N·K)。

06

複雜度// Complexity

K 趟 × 每趟 O(n)（差分 + 前綴和）= O(N·K)
n,K ≤ 2000 ⇒ 4×10⁶，瞬殺；樸素 O(N²·K)=1.6×10¹⁰ 會 TLE
項目數量備註
TimeO(N·K)差分讓區間轉移 O(n)/趟
SpaceO(N)cur / nxt / diff 三條陣列滾動
遞迴深度0（迭代）逐趟前進
Constant factor低每步兩次線性掃

MOD = 1000000007 是質數，但本題只用加減、沒有除法，所以不涉及逆元；減法記得 +MOD 防負。
