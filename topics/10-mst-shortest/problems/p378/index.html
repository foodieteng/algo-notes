江神與他的小火車

// Two-Sided Dijkstra · 每條候選邊 O(1) 回答

★ ★ ★ ☆ ☆ 核心 最短路 · 正反兩次 Dijkstra · 時間 O(M log N + Q)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

N 個城市、M 條有向軌道（aᵢ → bᵢ，長度 wᵢ）。 江神想各別評估 Q 種新增軌道的方案：第 i 種是加一條 cᵢ → dᵢ、長度固定為 1 的有向軌道。對每一種方案分開回答： 若在原圖加上這條軌道，城市 1 到城市 N 的最短路徑會變成多少？

N ≤ 2×10⁵ · M ≤ 5×10⁵ · Q ≤ 10⁵ · wᵢ ≤ 10⁹。每個 query 獨立 （候選邊只是「假設加上去」，不會累積）。保證原圖本來就存在 1 → N 的路徑。

02

為什麼正反各跑一次最短路// Why Two Dijkstras

A 每 query 重跑最短路 O(Q·M log N)
每次加一條邊重算 1→N。Q=10⁵ × Dijkstra ⇒ 太慢，TLE。
× REJECT · TLE
B 全加進去再跑一次 語意錯
query 是各自獨立，不能把 Q 條候選邊一起塞進圖。
△ REJECT · 誤解題意
BOTH FAIL · ADOPT
C 正 + 反 兩次 Dijkstra O(M log N + Q)
預處理 d1[]（1 到各點）與 dN[]（各點到 N），每個 query O(1)。
● ADOPT · OPTIMAL

關鍵觀察：候選邊 c → d 只可能讓最短路長這樣 —— 「1 ⇝ c」走原圖 → 走候選邊 c→d（長度 1）→「d ⇝ N」走原圖。 把這條新路徑的長度算出來，跟原本的最短路比小即可。

03

核心策略 · 三步驟// Algorithm Outline

  1. 正向 Dijkstra · 從城市 1 在原圖求 d1[v] = 1 到 v 的最短距離。
  2. 反向 Dijkstra · 把每條邊反向建一張反圖，從城市 N 出發求 dN[v] = v 到 N 的最短距離（在反圖上 = N 到 v）。
  3. 每個 query O(1) · 答案 = min(d1[N], d1[c] + 1 + dN[d])。 前項是原最短路，後項是「借候選邊 c→d」的新路徑。

為何只需兩次 · d1[] 一次算好「從起點到任何 c」，dN[] 一次算好 「從任何 d 到終點」，組合起來就能 O(1) 回答任意候選邊 —— 預處理一次、查詢無數次。

05

為何答案 = min(原最短路, d1[c]+1+dN[d])// Correctness

加一條邊後的最短路，要嘛不用這條新邊、要嘛用它。兩種情形取最小就是答案。

CASE 分析 · 用 or 不用候選邊
  1. 不用候選邊 · 最短路就是原圖的 d1[N]，不變。
  2. 用候選邊（恰一次） · 因為候選邊只有一條 c→d，任何用到它的路徑都長 「1 ⇝ c → d ⇝ N」。其中 1⇝c 段最短就是 d1[c]、d⇝N 段最短就是 dN[d]、 中間那條邊長 1 ⇒ 最佳 = d1[c] + 1 + dN[d]。
  3. 會不會用兩次？ 用兩次代表路徑經過 c→d 兩遍 ⇒ 中間繞了一個含 c、d 的環。 邊權非負 ⇒ 繞環不會更短 ⇒ 用一次一定不劣，所以只需考慮「恰用一次」。
  4. 兩種情形取 min ⇒ ans = min(d1[N], d1[c] + 1 + dN[d])。

（dN[d] 怎麼來？把所有邊反向得反圖，從 N 跑 Dijkstra； 反圖上「N 到 d」的距離 = 原圖「d 到 N」的距離。）

邊不可達的處理：若 d1[c] 或 dN[d] 為 ∞（c 到不了 / d 出不去）， 這條候選邊形不成有效路徑 ⇒ 該項視為 ∞、不參與 min。題目保證原圖 1→N 連通，所以 base 一定有限。

複雜度// Complexity

兩次 Dijkstra 各 O(M log N)，預處理完每個 query 只是一次比較 O(1)：

T = O(M log N + Q)
M=5×10⁵、N=2×10⁵、Q=10⁵ ⇒ 約 5×10⁵×18 + 10⁵ ≈ 10⁷，輕鬆通過
