01

題目簡介// Brief

QUESTION

給 N × N 棋盤（N = 2ᴷ，1 < N ≤ 1024）， 其中一格 (X, Y) 是王老先生的養老地不能蓋。 剩下 N² − 1 格要用 L-tromino（3 格 L 型，不可 1×3 直線） 鋪滿，不可重疊。題目互動式，對每塊 L 呼叫一次 Report。

不是求最佳解，是構造一組合法分配 — 寫一段「保證每步合法」的遞迴規則即可。

03

核心策略 · 三步驟// Algorithm Outline

對「左上 (top, left)、邊長 n、blocked 格 (bX, bY)」的子板：

  1. 切 4 象限 · TL / TR / BL / BR，每個 n/2 × n/2。blocked 必落在恰好 1 個象限。
  2. 中央放 L · 在另外 3 個象限「最靠近中心」的角各放 1 cell，組成一個 L → 一次 Report。
  3. 遞迴 4 子板 · 含原 blocked 的象限用原 blocked；其他 3 個用「剛種下的角」當新 blocked。

Base case · n = 2 時 4 格扣 blocked 剩 3 格本身就是 L，直接 Report。

05

Base case · n = 2 四種情況// Base Visual

遞迴最深處只會遇到 n = 2。blocked 在 4 個格之一，剩下 3 格本身就是 L （黑格是 blocked，灰格是該 L 的 3 cell）：

blocked = (1,1)
blocked = (1,2)
blocked = (2,1)
blocked = (2,2)

2×2 中 4 格扣掉任一格後，剩 3 格必為 L 型 — 不可能 1×3 直線， 因為 2×2 內任 3 格不可能共線（同行同列只有 2 格）。

更大的 N 完全沿用同一個遞迴模板：每層放 1 個中央 L， 把問題切成 4 個更小的同形子問題，直到 n = 2。

06

為何中央放 L 一定不重疊// Correctness

// CORRECTNESS · 構造法歸納
  1. 原 blocked 必落在恰好 1 個象限（座標唯一決定）— 稱為「bq 象限」。
  2. 中央 L 的 3 個 cell 各佔其他 3 個象限「最靠近中心」的那一角 — 與 bq 象限無交集。
  3. 放完中央 L 後，bq 象限仍只有 1 個 blocked（原來的 (X, Y)）。
  4. 其他 3 個象限新得 1 個 blocked（剛種下的角落 cell） — 也是恰好 1 個。
  5. 每個子象限 ⇒「n/2 × n/2 板 + 恰好 1 個 blocked」⇒ 與原問題同形 ⇒ 歸納成立。

⇒ 任意層遞迴放下的 L 都不會與其他層的 L 重疊，因為 cell 在哪層被「占用」就在哪層成為下層的 blocked。

07

複雜度分析// Complexity

每層 tile() 做的工：算 bq、列舉 4 個角、Report 一次中央 L、遞迴 4 個子板 — 都是 O(1)。

T(N) = 4 · T(N/2) + O(1)  ⇒  Θ(N²)
Master Theorem case 1：a = 4, b = 2, f(N) = O(1)，log₂ 4 = 2 > 0 ⇒ T(N) = Θ(N²)。
項目數量備註
TimeΘ(N²)總共要放 (N² − 1)/3 個 L，每個 O(1)。
SpaceO(log N)遞迴堆疊深度 = K = log₂ N。
Recursion Depthlog₂ NN = 1024 ⇒ 10 層，stack 很淺。
L 數量(N² − 1) / 3N = 1024 ⇒ 349,525 個 Report。

直覺：要放 Θ(N²) 個 L、每個 O(1)，下限已經是 Θ(N²) — 分治剛好達到這個下限。
