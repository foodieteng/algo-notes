重點回顧// Concept Recap

關鍵點內容
問題本質 持續傷害 vs 回血的競速；低門檻咒術要血量掉夠才解鎖。
貪心 每秒在已解鎖咒術裡放 c 最大者（傷害只增不減，早放放強都不吃虧）。
解鎖機制 咒術依門檻 pᵢ·M 排序，血量下降時用單調指標推進最大堆。
門檻判斷 可放 ⟺ hp × 100 ≤ pᵢ × M（整數，避免浮點誤差）。
判 −1 沒咒可放且 dps ≤ R ⇒ 血不再降，永遠打不死。
複雜度 O((M+N) log N)；有限答案 ≤ M+N。
02

觀念題// Concept Questions

Q1 貪心正確性

為什麼「每秒放當前可用 c 最大的咒術」是最佳？用「傷害只增不減」說明早放、放強為何不吃虧。

Q2 解鎖順序

低 pᵢ 的咒術為何一開始放不了？把咒術按 pᵢ·M 由大到小排，和血量下降有什麼對應關係？

Q3 判 −1

什麼情況輸出 −1？為什麼條件是「沒咒可放且 dps ≤ R」而不是只看 dps ≤ R？

Q4 效率

「每秒重掃 N 個咒術找最強」邏輯也對，為什麼會 TLE？最大堆 + 單調指標如何把它降到 O((M+N) log N)？

Q5 邊界

為什麼用 hp × 100 ≤ pᵢ × M 而非 hp ≤ pᵢ/100 × M？滿血 (hp=M) 時 p=100 的咒術可放嗎？

03

範例 Trace · M=100,R=10,A(100,11),B(90,9)// Worked Trace

每秒：hp ← min(M, hp − dps + R) → 判死 → 解鎖並放最強。咒術 A 門檻 100%（滿血即可放）、B 門檻 90%。

// INPUT2 100 10
100 11      // A: p=100 c=11
90 9        // B: p=90  c=9
// sec 0
hp = min(100, 100 − 0 + 10) = 100   不死
A 解鎖 (hp×100=10000 ≤ 100×100) → 放 A，dps = 11
// sec 1 .. 10  (dps=11 > R=10，每秒淨掉 1 血)
sec1: hp = 100 − 11 + 10 = 99
sec2: hp = 98
 ...
sec10: hp = 90    B 解鎖 (hp×100=9000 ≤ 90×100) → 放 B，dps = 20
// sec 11 .. 19  (dps=20，每秒淨掉 10 血)
sec11: hp = 90 − 20 + 10 = 80
 ...
sec18: hp = 10
sec19: hp = 10 − 20 + 10 = 0   ≤ 0 → 打敗！
// OUTPUT19
觀察：A 先把血從 100 慢慢壓到 90（每秒淨掉 1），解鎖 B 後 dps 翻到 20、每秒淨掉 10， 九秒內把血清空。先用高門檻強咒解鎖低門檻咒正是這題的節奏。
04

應用題// Quick Calculations

App 1 −1 判定

M=100, R=10，只有一個咒術 (p=100, c=10)。答案是多少？為什麼？

App 2 全 p=100

40% 測資所有 pᵢ=100（全部一開始就能放）。這時演算法退化成什麼？怎麼最快？

App 3 上界

為什麼有限答案一定 ≤ M+N？這個界對「設定迴圈上界 / 判 −1」有什麼用？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · 貪心正確性

咒術一旦施放，往後每秒都扣 cᵢ。所以同一個咒術越早放，到任何時刻累積的傷害都 ≥ 晚放 — 把最佳解裡某次施放「提前到最早可放的那秒」，每一秒的血量都不會更高，打敗時間不會變晚（交換論證）。

同一秒只能放一個，挑 c 最大的能讓往後每秒扣最多、血掉最快，連帶最快解鎖後續咒術。 所以「每秒放已解鎖中最強」就是最佳，不需 DP。

Q2 · 解鎖順序

第 i 咒要 hp ≤ pᵢ%·M 才可放。低 pᵢ ⇒ 門檻低，得等血量掉夠才解鎖；滿血時只有高 pᵢ 的能放。

把咒術按門檻 pᵢ·M 由大到小排序後，血量是單調下降的（一旦 dps>R）， 於是用一個只前進的指標 ptr，每當血量降到某咒門檻就把它推進堆 — 每咒一生只解鎖一次。

Q3 · 判 −1

−1 ⟺ 進入一個沒有任何咒可放、且 dps ≤ R 的狀態。此時每秒扣血 ≤ 回血，血量不再下降， 既不會掉到更低門檻去解鎖新咒、也永遠歸不了零。

不能只看 dps ≤ R：因為可能還有咒可放（堆非空），放下去 dps 就會超過 R、血開始降。 必須「無咒可放」且「dps ≤ R」同時成立才是真的卡死。

Q4 · 效率

每秒重掃 N 個咒術找最強是 O(N) / 秒，而秒數可達 ~M=10⁵ ⇒ 總 O(N·M)=10¹⁰，TLE。

改用最大堆：咒術按門檻排序，血量下降時用單調指標把新解鎖的 push 進堆（每咒一次，總 O(N log N)）， 每秒只 top()/pop() 取最強 O(log N)。秒數 ≤ M+N，總 O((M+N) log N)。

Q5 · 邊界

hp ≤ pᵢ/100 × M 牽涉浮點除法，邊界（剛好等於門檻）可能因誤差判錯。改成等價的整數不等式 hp × 100 ≤ pᵢ × M，pᵢ·M ≤ 10⁷、hp·100 ≤ 10⁷，long long 全程精確。

滿血 hp=M、p=100 時：M×100 ≤ 100×M 成立（等號）⇒ 可放。所以 p=100 的咒術第 0 秒就能施放， 和範例 A 在 sec0 被放一致。

App 1 · −1 判定

放了唯一咒術後 dps = 10 = R。第 0 秒 hp=min(100,100−0+10)=100，放咒；之後每秒 hp = hp − 10 + 10 = hp，血量卡在 100 不動。沒有其他咒可放、dps ≤ R ⇒ 答案 −1。

差一點：若 c=11，則每秒淨掉 1，100 秒後死，答案會是有限值（≈100）。dps 必須嚴格大於 R 才殺得死。

App 2 · 全 p=100

所有咒術一開始就解鎖 ⇒ 退化成「按 c 由大到小，每秒放一個」：第 0 秒放最大、第 1 秒放次大… 每秒 dps 單調增加，等到累積 dps>R 後血開始掉，再算何時歸零。

本題的堆在此情形等同「把所有 c 排序後依序取」；單調指標一次把全部 push 進堆即可，邏輯不變。

App 3 · 上界

一旦 dps > R，血量每秒至少淨掉 1，從 ≤ M 出發最多 M 秒歸零；而把 dps 推到 >R 之前， 每一秒要嘛在放咒（至多 N 次）、要嘛已卡死（立即判 −1）。

所以任何有限答案 ≤ M + N。這給了安全的迴圈上界：超過就一定是 −1（雙保險）， 也說明逐秒模擬的總步數是 O(M+N)、不會爆。
