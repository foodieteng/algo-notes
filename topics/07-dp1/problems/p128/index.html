取數字 2

// 線性 DP · 任兩數距離 ≥ K 的最大子集和

★ ★ ☆ ☆ ☆ 核心 一維 DP（House Robber 的 K 推廣） · 時間 O(N)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

給一個正整數陣列 a₁, a₂, …, a_N，要從中取出一些數字， 但任兩個被取的數字 index 距離不小於 K，即同時取 aᵢ 與 aⱼ 須滿足 |i−j| ≥ K， 求取出數字的總和最大值。

K = 1 表示完全沒限制；K = 2 表示兩數不能相鄰（就是「取數字1」）。 多筆 T ≤ 10，1 ≤ K ≤ N ≤ 1000，1 ≤ aᵢ ≤ 1000。

這就是「取數字1 / House Robber」的一般化：原本「不能相鄰」= 距離 ≥ 2， 現在把「2」換成任意 K。範例：[1,2,3,4], K=2 → 6（取 a₂+a₄）、 [1,3,4,2], K=3 → 4（只取 a₃=4，因為任取兩個距離都湊不到 3 還更大）。

02

為什麼用 DP// Why DP

A 枚舉子集 O(2ᴺ)
試每種取法、再檢查任兩個距離 ⇒ N=1000 完全不可能（連 N=40 都爆）。
× REJECT · TLE
B 貪心取大 O(N log N)
每次挑最大的、封鎖左右各 K−1 格 ⇒ 會選錯：拿一個大數會擋掉更好的組合。
△ REJECT · WRONG
BOTH FAIL · ADOPT
C 線性 DP O(N)
每個位置只有「取 / 不取」兩種；取了就往回跳 K 格接 dp[i−K]。一遍掃完。
● ADOPT · STABLE

和「取數字1」完全同構，只差「取這格」要接的不是前兩格、而是前 K 格 — 把 dp[i−2] 換成 dp[i−K] 即可。貪心同樣會被「眼前最大」綁架而選錯。

03

狀態與轉移// State & Transition

1-indexed。dp[i] = 只看 a[1..i] 時的最大合法和。 每個位置兩個選擇：

dp[i] = max( dp[i−1] , dp[i−K] + a[i] )
跳過 a[i] ⇒ 沿用 dp[i−1]；取 a[i] ⇒ 上一個被取的須距離 ≥ K ⇒ 接 dp[i−K] 再加 a[i]
  1. base · dp[0] = 0（空集合）。當 i − K ≤ 0 時取 a[i] 就接 dp[0]=0（等於只取它一個）。
  2. 答案 · dp[N]。
  3. 退化檢查 · K = 2 就是「取數字1」；K = 1 全取（dp[i]=dp[i−1]+a[i]）。

05

為什麼這個轉移是對的// Correctness

// 最優子結構 + 無後效性
  1. 看最後被考慮的 a[i]，最佳解只有兩種可能：沒取它 或 取了它。
  2. 沒取 a[i] ⇒ 問題退化成「a[1..i−1] 的最佳解」= dp[i−1]。
  3. 取了 a[i] ⇒ 上一個被取的 index j 須滿足 i − j ≥ K ⇒ j ≤ i − K， 剩下自由的是 a[1..i−K] ⇒ dp[i−K] + a[i]。
  4. 兩種互斥且涵蓋所有情況 ⇒ 取 max 即 dp[i]（最優子結構）。
  5. dp[i] 只依賴更小的 index（無後效性）⇒ 由小到大一次填完即可。

關鍵：「取了 a[i]」時前一個被取的可以是 j = i−K, i−K−1, … 中任何一個，但這些都已包含在 dp[i−K] 裡 （因為 dp 全為前綴最佳、且 aᵢ ≥ 1 ⇒ dp 單調不減，dp[i−K] 已是 dp[1..i−K] 的最大）。所以只接 dp[i−K] 一項就夠。

06

複雜度// Complexity

每格 O(1) × N 格 = O(N) 每筆
存整條 dp[] ⇒ 空間 O(N)；N ≤ 1000、T ≤ 10 ⇒ 約 10⁴ 次運算，秒過
項目數量備註
TimeO(N)單次掃描，每格一次 max + 一次加法
SpaceO(N)需存 dp[i−K] ⇒ 留整條 dp[]（也可滾動環狀緩衝壓到 O(K)）
遞迴深度0（迭代）純迴圈
Constant factor低一次比較一次加法
07

一句洞察// One-Sentence Insight

就是 House Robber，把「前兩格」換成「前 K 格」。
取就往回跳 K 格接 dp[i−K]，不取就沿用 dp[i−1]，兩者取大 —
dp[i] = max(dp[i−1], dp[i−K] + a[i])，一遍 O(N) 解決。
