重點回顧// Concept Recap

關鍵點內容
問題本質 相鄰合併最小成本 = 經典「石子合併 / matrix-chain」區間 DP。
狀態 dp[i][j] = 把 a[i..j] 合成一隻的最小總高棕櫚。
轉移 dp[i][j] = min_k(dp[i][k]+dp[k+1][j]) + sum(i..j)。
核心觀察 最後一步合 [i..j] 固定付 sum(i..j)，與切點 k 無關。
填表順序 區間長度 len 由小到大（無後效性）。
複雜度 O(N³)；前綴和讓 sum(i..j) O(1)。
02

觀念題// Concept Questions

Q1 轉移推導

解釋 dp[i][j]=min_k(dp[i][k]+dp[k+1][j])+sum(i..j)：k 代表什麼？為什麼要加 sum(i..j)？

Q2 為何貪心錯

「每次合當前最小的相鄰兩隻」是貪心。為什麼這題不能用？和 Huffman 編碼差在哪？

Q3 填表順序

為什麼一定要按「區間長度由小到大」填？直接 for i for j 會出什麼問題？

Q4 前綴和

若每次都用迴圈重算 sum(i..j) 而非前綴和，複雜度變成多少？會超時嗎？

Q5 成本本質

為什麼總成本 = 每個 aₓ 乘上「它在合成樹中的深度」？這對最佳化有什麼啟發？

03

範例 Trace · 填 dp 表 [1,2,3]// Worked Trace

1-indexed，dp[i][i]=0，prefix = [0,1,3,6]。按長度填：

// INPUT3
1 2 3       // a[1]=1 a[2]=2 a[3]=3
// len = 2
dp[1][2]: sum=a1+a2=3, k=1 → dp[1][1]+dp[2][2]+3 = 0+0+3 = 3
dp[2][3]: sum=a2+a3=5, k=2 → dp[2][2]+dp[3][3]+5 = 0+0+5 = 5
// len = 3  (整段 [1..3], sum=6)
k=1: dp[1][1]+dp[2][3]+6 = 0 + 5 + 6 = 11
k=2: dp[1][2]+dp[3][3]+6 = 3 + 0 + 6 = 9   ← 較小
dp[1][3] = 9   (最佳切點 k=2)
// OUTPUTdp[1][3] = 9
觀察：兩種切法都要付最後一步的 sum=6；差別在子問題 — k=2 先合 [1,2]（付 3）比 k=1 先合 [2,3]（付 5）省。 對應題目：先合 1+2 再合 (3)+3 = 3+6 = 9，比先合 2+3 的 5+6=11 更省。

應用題// Quick Calculations

App 1 較大例

對 [1,4,2,5,3]（答案 35），dp[1][5] 的最佳切點 k 是多少？對應怎麼拆？

App 2 環形版

若円円排成環（首尾也相鄰可合），怎麼用同一套 DP 解？複雜度變多少？

App 3 加速

O(N³) 對更大的 N（如 N=2000）會太慢。有什麼經典優化能降到 O(N²)？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · 轉移推導

k 是「最後一次合成」的切點：把 [i..j] 合成一隻的過程，最後一步必是把 左段 [i..k] 已合成的一隻、與右段 [k+1..j] 已合成的一隻合起來。

加 sum(i..j)，因為這最後一步合的兩隻糟糕值之和 = 整段總和（合成不改變總糟糕值）。 左右兩半各自合成的最小成本就是子問題 dp[i][k]、dp[k+1][j]，枚舉 k 取最小即得。

Q2 · 為何貪心錯

Huffman 能把任意兩個節點配對（不限相鄰）⇒ 貪心合最小兩個最佳。 本題只能合相鄰，合了還會改變剩下序列的相鄰結構， 「眼前最小相鄰對」先合未必導向全域最小。

反例可構造：相鄰限制下被迫多次搬動大數，使某些大數深度被拉高。一定要 DP 對每個區間枚舉切點。

Q3 · 填表順序

算 dp[i][j] 依賴更短的子區間 dp[i][k]（長度 k−i+1 < len）與 dp[k+1][j]（長度 j−k < len）。

若直接 for i=1..n for j=i..n 順序填，算 dp[i][j] 時 dp[k+1][j]（同 j、起點較大） 可能還沒算 ⇒ 讀到 0 而出錯。按長度由小到大（或記憶化遞迴）才保證依賴都先就緒。

Q4 · 前綴和

每個 (i, j, k) 三元組都需要 sum(i..j)。若每次用迴圈重算是 O(N)，三層迴圈外再乘一層 ⇒ O(N⁴)。N=100 時 ~10⁸、T≤10 ⇒ 10⁹，有超時風險。

前綴和把 sum(i..j)=prefix[j]−prefix[i−1] 降到 O(1)，總複雜度回到 O(N³) ≈ 10⁶，穩過。

Q5 · 成本本質

把合成過程看成一棵二元樹：葉子是原始 aₓ，每個內部節點是一次合成。 一個 aₓ 每被「往上合一層」就再被計入一次成本 ⇒ 它對總成本的貢獻 = aₓ × 深度。 總成本 = Σ aₓ × depth(x)。

啟發：最佳化就是讓大的 aₓ 深度盡量小（少被重複加）。若無相鄰限制這就是 Huffman； 有相鄰限制則只能在合法樹形中用區間 DP 找最小加權路徑和。

App 1 · 較大例

[1,4,2,5,3]：dp[1][5] = 35，最佳切點 k = 3 ⇒ 左 dp[1][3] = 12（合 [1,4,2]）+ 右 dp[4][5] = 8（合 [5,3]）+ sum(1..5)=15 ⇒ 12+8+15 = 35。 

// 關鍵子表
dp[1][2]=5  dp[2][3]=6  dp[3][4]=7  dp[4][5]=8
dp[1][3]=12 dp[2][4]=17 dp[3][5]=17
dp[1][4]=24 dp[2][5]=28
dp[1][5]=35 (k=3)
App 2 · 環形版

標準技巧：把陣列複製一份接在後面（長度 2N），在這條長為 2N 的序列上跑同一套區間 DP， 答案 = min over 起點 i of dp[i][i+N−1]（每個長度 N 的連續視窗 = 一種「斷開環的方式」）。

複雜度 O((2N)³) = O(N³)（常數變大）。本題未提環形，僅作延伸。

App 3 · 加速

經典「四邊形不等式（Knuth–Yao）優化」：本題滿足區間單調性，最佳切點 opt[i][j] 對 i、j 單調 ⇒ 內層 k 的搜尋範圍可限制在 opt[i][j−1] ≤ k ≤ opt[i+1][j]， 把 O(N³) 降到 O(N²)。

本題 N ≤ 100 不需要；但面對 N 上千的石子合併變體，這是標準加速法。
