01

重點回顧// Concept Recap

關鍵點內容
目標 挑 N−1 條邊連通全圖、權和最小 = MST。
Kruskal 邊權由小到大，貪心加「不成環」的邊。
判環工具 並查集 DSU：find 同根 ⇒ 同塊 ⇒ 成環 ⇒ 跳過。
正確性 cut property：跨任意切口的最小邊必屬某棵 MST。
DSU 優化 路徑壓縮 + 按秩合併 ⇒ 單次近 O(α(N))。
數值 權和達 2×10¹⁴ ⇒ total 用 long long。

被跳過的邊（兩端已同塊）對 MST 有沒有可能有用？為什麼可以直接丟掉？

範例 Trace · 跑一次 code 看會輸出什麼// Worked Trace

用官方樣本：N=3, M=3，邊 (1-2,3)、(2-3,4)、(3-1,5)。

// INPUT3 3
1 2 3
2 3 4
3 1 5

STEP 1 · 排序（已由小到大）：

順序邊權重 w
11—23
22—34
33—15
 
// DSU INIT
par = [_, 1, 2, 3]   ← 每點自成一塊
total = 0; used = 0
// EDGE 1 · (1—2, w=3)
find(1)=1, find(2)=2  不同塊 ⇒ 加入
unite(1,2)  → {1,2} 一塊
total = 3; used = 1
// EDGE 2 · (2—3, w=4)
find(2)=1, find(3)=3  不同塊 ⇒ 加入
unite(2,3)  → {1,2,3} 一塊
total = 7; used = 2 = N-1 ⇒ break ✓
// (EDGE 3 · (3—1, w=5) 不會被處理，已提早結束)
若處理：find(3)=find(1) 同塊 ⇒ 成環 ⇒ 會被跳過
// OUTPUT7
觀察：MST 取了最便宜的兩條邊 3 + 4 = 7，自然避開最貴的 (3-1, w=5)。 就算沒提早 break，這條邊也會因「1、3 已同塊（成環）」而被跳過 —— 貪心 + 判環兩件事就構成整個 Kruskal。

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

A1 · 貪心正確性

由 cut property：對任意把點切兩堆的切口，跨切口的最小邊一定屬於某棵 MST。

Kruskal 每次加的邊，正是「把已連好那塊當一邊、其餘當另一邊」這個切口的最小跨切邊 ⇒ 每步都安全。 用交換論證：若某 MST 不含這條邊，加它成環、環上必有另一條更貴（或等重）的跨切邊，換掉它權和不增 ⇒ 存在含這條邊的 MST。逐步歸納 ⇒ 整體最優。

A2 · 並查集

每次加邊後跑 DFS/BFS 檢查連通是 O(N+M)，整體 O(M·(N+M)) 太慢。 並查集把「判兩點是否同塊」與「合併兩塊」都做到攤銷近 O(α(N))（反阿克曼，幾乎常數）。

整體因此降到由排序主導的 O(M log M)。

A3 · 跳過的邊

被跳過的邊兩端已在同一連通塊 ⇒ 它不是跨切邊，加進去只會在塊內形成環， 不連通任何新的點。

而且因為是「由小到大」掃，現在這條（同塊）邊一定 ≥ 當初連起這兩端的那些邊 ⇒ 換掉也不會更好。 所以直接丟掉，對 MST 毫無損失。

A4 · 資料型別

最容易忘的是累加權和 total。 

// 溢位量級
邊數最多 N-1 ≈ 2×10⁵
每條 w 最大 10⁹
權和最大 ≈ 2×10⁵ × 10⁹ = 2×10¹⁴
int 上限 ≈ 2.1×10⁹  ⇒  必爆 ❌
long long 上限 ≈ 9.2×10¹⁸  ⇒  OK ✓

單一 w ≤ 10⁹ 雖然塞得進 int，但累加會爆 ⇒ total 一定要 ll。

A5 · Prim vs Kruskal

Prim（堆實作 O(M log N)）適合稠密圖或已有鄰接表、且想從一點長出樹； 樸素 Prim O(N²) 在稠密小 N 時甚至更快。

Kruskal 以邊為中心、只需邊表 + 排序 + DSU，對稀疏圖（M 與 N 同量級）最直覺。 本題給的就是邊表、且 M ≤ 2×10⁵ 稀疏 ⇒ Kruskal 寫起來最短最穩。

A · App 1

排序後依序：(1-2,1)(2-3,2)(1-3,3)(3-4,4)(4-5,5)(2-5,6)。

// kruskal trace
(1-2,1): {1,2}        加 → total=1
(2-3,2): {1,2,3}      加 → total=3
(1-3,3): 1,3 已同塊   跳過（環）
(3-4,4): {1,2,3,4}    加 → total=7
(4-5,5): {1,2,3,4,5}  加 → total=12, used=4=N-1 break
(2-5,6): 未處理（若處理會因同塊跳過）

被跳過：(1-3,3)、(2-5,6)。MST 權和 = 12。（與動畫一致。）

A · App 2

相同權重時，誰先被加可能讓選到的邊集合不同（MST 不唯一）， 但權和一定相同 —— 因為所有 MST 權和都等於最小值，這是 MST 的基本性質。

本題只要權和，所以 tie-break 怎麼排都不影響答案。

A · App 3

要最大生成樹只需把排序改成由大到小，其餘（DSU 判環、加滿 N−1）完全不變：

// 最大生成樹
sort(edges.rbegin(), edges.rend());   // 由大到小
// 或自訂 comparator 反向；其餘照舊

cut property 對稱地成立（跨切口的最大邊屬某棵最大生成樹），所以只改排序方向即可。
