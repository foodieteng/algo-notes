01

完整實作 · 差分區間轉移// Full C++

輸入只有一行 n a b k。cur[f] 是當前方法數。每趟用 差分陣列把 cur[x] 加到合法區間， 前綴和還原成 nxt[]，再扣掉 y=x 自身。答案是 Σ cur[f]。

p369.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const long long MOD = 1000000007;
const int MAXN = 2005;

long long cur[MAXN];     // cur[f] = 搭完目前趟數後，停在樓層 f 的方法數
long long nxt[MAXN];
long long diff[MAXN];    // 差分陣列，把「區間加」變 O(1)

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, a, b, k;
    cin >> n >> a >> b >> k;

    for (int f = 0; f <= n + 1; f++) cur[f] = 0;
    cur[a] = 1;

    for (int s = 0; s < k; s++) {
        for (int f = 0; f <= n + 1; f++) diff[f] = 0;

        // 每個樓層 x 把 cur[x] 加到合法落點區間 [x-d+1, x+d-1]，d = |x-b|
        for (int x = 1; x <= n; x++) {
            if (!cur[x]) continue;
            int distB = abs(x - b);           // x 到禁止樓層 b 的距離
            if (distB <= 0) continue;        // x==b 到不了，略過
            int lo = max(1, x - distB + 1);
            int hi = min(n, x + distB - 1);
            diff[lo]     = (diff[lo]     + cur[x]) % MOD;
            diff[hi + 1] = (diff[hi + 1] - cur[x] + MOD) % MOD;
        }

        // 前綴和還原 nxt[]
        long long run = 0;
        for (int y = 1; y <= n; y++) {
            run = (run + diff[y]) % MOD;
            nxt[y] = run;
        }
        // 扣掉 y==x 的自身貢獻（電梯要動，x != y）
        for (int x = 1; x <= n; x++) {
            if (!cur[x]) continue;
            if (abs(x - b) <= 0) continue;
            nxt[x] = (nxt[x] - cur[x] + MOD) % MOD;
        }

        for (int f = 1; f <= n; f++) cur[f] = nxt[f];
    }

    long long ans = 0;
    for (int f = 1; f <= n; f++) ans = (ans + cur[f]) % MOD;
    cout << ans << '\n';
    return 0;
}

為什麼最後要扣 y=x？ 區間 [x−d+1, x+d−1] 含 x 自己（距離 0），但電梯不能停在原地（x ≠ y）， 所以前綴和後把每個 nxt[x] −= cur[x] 修正掉。

複雜度分析// Complexity Derivation

步驟內容成本
每趟差分加每個 x 一次 O(1) 區間加O(N)
每趟前綴和掃一遍還原 nxtO(N)
每趟扣自身掃一遍修正 y=xO(N)
共 K 趟—O(N·K)
最後加總Σ cur[f]O(N)
K 趟 × O(N)/趟 = O(N·K)
2000×2000 = 4×10⁶，遠快於樸素 O(N²·K)=1.6×10¹⁰
項目數量備註
TimeO(N·K)差分把區間轉移 O(N)/趟
SpaceO(N)cur / nxt / diff
遞迴深度0（迭代）逐趟前進
Constant factor低每趟三次線性掃
03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
別忘了扣 y=x 區間含 x 自己，但電梯要動 ⇒ 前綴和後每個 nxt[x] −= cur[x]。漏扣會多算「停原地」的假搭法。
區間 clamp 到 [1,n] lo=max(1,x−d+1)、hi=min(n,x+d−1)。不夾範圍會寫到陣列外。
條件是嚴格 < |y−x| < |b−x| 是嚴格小於，所以 hi = x+d−1（不是 x+d）。寫成 ≤ 會跨過 b、答案錯。
減法補 MOD 差分的 diff[hi+1] −= cur[x] 與扣自身都可能為負 ⇒ (… + MOD) % MOD。
輸入只有一行 沒有測資數 T，直接讀 n a b k 四個數。別套 while(T--)。
答案是 Σ cur[f] 搭完 K 趟後停在任一樓層都算一種搭法，要全部加起來，不是只取某一格。
04

細節走讀// Section Walkthrough

① 狀態定義

// state
cur[f] = 搭完目前 s 趟後，停在樓層 f 的方法數
初值 cur[a] = 1（起點），其餘 0
答案 = Σ_f cur[f]（搭完 K 趟後）

② 一趟的轉移（區間 + 差分）

// one ride
對每個 cur[x] > 0：
  d  = |x - b|
  lo = max(1, x-d+1),  hi = min(n, x+d-1)   // 合法落點區間
  diff[lo] += cur[x];  diff[hi+1] -= cur[x] // 差分區間加
前綴和 diff → nxt[]
每個 nxt[x] -= cur[x]                         // 扣掉 y=x（電梯要動）

③ 為什麼差分能加速

// why diff
樸素：每個 x 逐一加到 O(n) 個 y → 每趟 O(n²)
差分：每個 x 只動兩格 diff[lo], diff[hi+1] → O(1) 區間加
      最後一次前綴和 O(n) 還原 → 每趟 O(n)，K 趟 O(nK)
