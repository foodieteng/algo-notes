重點回顧// Concept Recap

關鍵點內容
問題本質 最大「不相鄰子集和」= House Robber。
狀態 dp[i] = 看到 a[0..i] 的最大和；每格只有取 / 不取。
轉移 dp[i] = max(dp[i−1], dp[i−2] + a[i])。
滾動優化 只用前兩格 ⇒ take / skip 兩變數，空間 O(1)。
正確性 取 / 不取兩種互斥且涵蓋全部 ⇒ 最優子結構。
貪心不行 局部拿最大會封鎖鄰居、漏掉更好組合。

Q2 貪心反例

給一個「每次拿最大、刪鄰居」會選錯的具體陣列，並說明 DP 為何對。

滾動版的 take 為什麼接 skip、skip 為什麼取 max(take,skip)？

Q4 更新順序

為什麼要先用舊值算出 newTake / newSkip 再一起更新？先覆寫 take 會怎樣？

範例 Trace · 跑滾動 DP 看 [1,3,4,2]// Worked Trace

追 take / skip 每一步（初值 take=skip=0）：

// INPUT4
1 3 4 2
// a = 1
newTake = skip + 1 = 0 + 1 = 1
newSkip = max(take, skip) = max(0, 0) = 0
take=1  skip=0
// a = 3
newTake = skip + 3 = 0 + 3 = 3
newSkip = max(take, skip) = max(1, 0) = 1
take=3  skip=1
// a = 4
newTake = skip + 4 = 1 + 4 = 5      // = 取 a[0]=1 與 a[2]=4
newSkip = max(take, skip) = max(3, 1) = 3
take=5  skip=3
// a = 2
newTake = skip + 2 = 3 + 2 = 5      // = 取 a[1]=3 與 a[3]=2
newSkip = max(take, skip) = max(5, 3) = 5
take=5  skip=5
// OUTPUTmax(take, skip) = max(5, 5) = 5
觀察：最後 take=skip=5 — 代表有兩條不同的最佳取法： 取 a[0]+a[2]=1+4=5（最後一格沒取）或取 a[1]+a[3]=3+2=5（最後一格有取）。 DP 不在乎是哪一條，只回傳最大值 5。
04

應用題// Quick Calculations

App 1 環形版

若陣列是環形（a[0] 與 a[N−1] 也算相鄰），怎麼用同一套 DP 解？

App 2 輸出方案

若還要輸出「取了哪些 index」而不只是總和，要怎麼回溯？

App 3 至少間隔 k

若限制改成「任兩個被取的數至少間隔 k 格」，轉移怎麼改？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · 轉移推導

dp[i−1] = 不取 a[i]：問題退化成只看 a[0..i−1]。 dp[i−2] + a[i] = 取 a[i]：因為 a[i−1] 與 a[i] 相鄰不能同取， 取了 a[i] 後最近能自由選的是 a[0..i−2] ⇒ 接 dp[i−2] 再加 a[i]。兩種互斥取 max。

Q2 · 貪心反例

[3, 4, 3, 4, 3]。貪心先拿最大的 4（其一）、刪兩鄰，再拿另一個 4 ⇒ 4+4 = 8。 但最佳是取三個 3（index 0,2,4）= 9。

DP 不會被「眼前最大」綁架：它對每個位置都同時保留「取 / 不取」兩條最佳鏈， 最後才比較，因此能找到 9。

Q3 · 滾動變數

take（取這格）⇒ 上一格必須沒取，所以只能接 skip，得 skip + a。

skip（不取這格）⇒ 上一格取或不取都合法（反正這格沒取、不會相鄰衝突）， 所以取兩者較大 max(take, skip)。

Q4 · 更新順序

因為 newTake 和 newSkip 都依賴舊的 take / skip。若先把 take 覆寫成 newTake， 接著算 newSkip = max(take, skip) 就會用到已被汙染的 take ⇒ 算錯。

所以要先用舊值算出兩個新值（存到暫存），再同時更新；或用一個暫存變數保留舊 take。

Q5 · 正數假設

本題 aᵢ ≥ 1，所以「能取就取不會虧」，take = skip + a 一定 ≥ 不取的情況，邏輯成立。

若允許負數，取一個負數可能變差。此時：

  • 「空集合」要算合法（和 = 0）⇒ 答案 max(take, skip, 0)，或讓 skip 永遠能保留 0。
  • 視題意，可能要允許「取了但寧可不加」⇒ newTake = max(skip, ...) + a 之類，或直接 max(skip + a, skip)。

核心轉移結構不變，只是 base / 是否允許空集合要重新界定。

App 1 · 環形版

環形時 a[0] 與 a[N−1] 相鄰，不能同取。標準技巧：拆成兩個線性問題取最大 —

  • 情況 A：不取 a[0] ⇒ 在 a[1..N−1] 上跑線性 DP。
  • 情況 B：不取 a[N−1] ⇒ 在 a[0..N−2] 上跑線性 DP。

答案 = max(A, B)。這樣保證頭尾不會同時被取。（N=1 特判直接回 a[0]。）

App 2 · 輸出方案

開整條 dp[]（不滾動），再從 dp[N−1] 往回走： 

// backtrack
i = N-1
while i >= 0:
    if i == 0 or dp[i] == dp[i-1]:   // 沒取 a[i]（沿用前一格）
        i -= 1
    else:                            // 取了 a[i] = dp[i-2]+a[i]
        記錄 i; i -= 2

反序記錄的 index 即為取法（之一）。需要 O(N) 空間存 dp。

App 3 · 至少間隔 k

「任兩個被取的至少間隔 k 格」⇒ 取 a[i] 後，下一個最近能取的是 a[i+k]（中間 k−1 個不能取）。 轉移改成： 

// gap >= k
dp[i] = max(dp[i-1], dp[i-k] + a[i])
// 取 a[i] ⇒ 接 dp[i-k]（保證間隔 ≥ k）
// k=2 就退化回本題

仍是 O(N)，只是「取」那項從 dp[i−2] 換成 dp[i−k]。
