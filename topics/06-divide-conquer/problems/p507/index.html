// Closest Pair of Points · 平面分治經典

01

題目簡介// Brief

QUESTION

平面 N ≤ 2 × 10⁵ 個整數座標點（|x|, |y| ≤ 10⁹）。 找最近兩點的距離平方 d²。

用平方比就好 — a < b ⇔ a² < b²，免開根號、整題純 long long。

02

03

核心策略 · 三步驟// Algorithm Outline

先把點按 x 排序，closest(lo, hi) 做 3 件事：

  1. 切半 · mid_x = pts[(lo+hi)/2].x
  2. 遞迴 · d = min(closest 左, closest 右)
  3. 合併 strip · 收 |x − mid_x| < √d → 按 y 排序 → 每點看下 7 個鄰居 → 更新 d

Base case · n ≤ 3 直接 brute（3 對以內）。

06

為何只看寬 2√d 的 strip 就夠// Why the Strip is Enough

回到本層時手上有 d = min(d_L, d_R)（兩半各自最佳的距離平方）。 接下來想找跨界 pair（左右各一點）有沒有打敗 d。但跨界對最壞有 N²/4 種 — 怎麼把候選縮到 O(N)？

關鍵觀察 · 距 mid_x 太遠的點直接出局

設當前最佳距離為 √d（程式裡 d 存的是距離平方）。對左半任一點 p：

  1. 右半任何點 q 都滿足 x_q ≥ mid_x。 ⇒ x_q − x_p ≥ mid_x − x_p。
  2. 所以 p 跟右半任何 q 的水平距離至少 mid_x − x_p。
  3. 若 mid_x − x_p ≥ √d（即 p 離中線 ≥ √d）⇒ 任何 |pq| ≥ √d ⇒ p 不可能改善 d，可以丟掉。
  4. 對右半 q 同理 — 只有 |x_q − mid_x| < √d 的點才有資格。
  5. ⇒ 跨界候選都落在 |x − mid_x| < √d 的寬 2√d 垂直條帶（strip）內。

關鍵 trick：寫成 (x − mid_x)² < d 直接整數比較， 整題不必動 sqrt，long long 走到底。
下一節接著問：strip 內最壞還是 O(N) 點，要怎麼避免每點都跟全 strip 比？

07

為什麼只比 7 個鄰居就夠// Pigeonhole Bound

兩點距離 < d ⇒ y 差也 < d。問：strip 任意 d 高度視窗最多塞幾個點？

Pigeonhole · 鴿籠論證
  1. 對任一點 p，y 排序後想跟 p 「成最近」的別人，y 差必須 < d ⇒ 它們都落在 p 正上方一塊 2d × d 視窗（寬 = strip 寬，高 = d）。
  2. 把這塊 2d × d 切成 4 × 2 = 8 個 (d/2) × (d/2) 小格。
  3. 每小格對角線 = d / √2 ≈ 0.71 d < d。但左半屬於 d_L ≥ d、右半屬於 d_R ≥ d 的範圍 ⇒ 同一小格內塞 2 點 ⇒ 矛盾。
  4. 所以每小格最多 1 點 ⇒ 整個 2d × d 視窗最多 8 點（含 p 自己）。
  5. 扣掉 p ⇒ 別人最多 7 個 ⇒ y 排序後每點只跟「後 7 個」比就夠。

速記：2d × d ÷ (d/2)² = 8 格 → 8 點 → 扣自己 → 7 鄰居。 d 來自上一層遞迴的最佳值，是「不能再小」的保證 → 才能逼出每格 ≤ 1 點。

08

複雜度分析// Complexity (step by step)

三步驟拆解 O(N log² N)：

STEP 1 算「一次 closest() 呼叫」做多少工

事情說明成本
① 切半 + 遞迴左右 呼叫兩個 N/2 的子問題 2 · T(N/2)
② 收集 strip 掃過所有 N 個點，挑 |x − mid_x|² < d 的 O(N)
③ strip 排序 by y 這就是多出來的那個 log！ strip 最壞含 N 點，sort() 要 N log N。 O(N log N)
④ strip 內 7 鄰居比 每點 O(1)，總共 O(N) O(N)
T(N) = 2 · T(N/2) + O(N log N)
遞迴成本 + 一次合併成本（其中最貴的是 strip 排序）

STEP 2 畫遞迴樹，每層算一次總工

每往下切一層，N 變一半，剛好 log₂ N 層：

總複雜度 = O(N log² N)
對 N = 2 × 10⁵ ⇒ 約 2 × 10⁵ × 18² ≈ 6 × 10⁷ 次運算，秒過

STEP 3 能不能更快？

瓶頸：每層 strip 重排序多一個 log。改用 merge sort 維護 y-sorted 副本 → 合併 O(N)：

T(N) = 2 · T(N/2) + O(N) = O(N log N)
程式較複雜；本題 N = 2×10⁵，O(N log² N) 已秒過、不必硬上
09

一句洞察// One-Sentence Insight

價值不在「切半」，而在「合併」。
寬 2d 的 strip 把跨界候選縮到 O(N)，
按 y 排序 + 7 鄰居 再把比對縮到 O(N)。
關鍵是幾何 + pigeonhole，不是遞迴。
