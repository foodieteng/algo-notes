動畫演示

Base Case · 兩個單元素的 merge

Merge sort 的 base case 是「子陣列長度為 1」。 兩個長度為 1 的子陣列合併時，只比較一次。共有 2 種情況：

Case A
2 ≤ 5 → 無逆序對
Case B
5 > 4 → 1 對，貢獻 5+4 = 9
// Base case 為何單純

單元素 vs 單元素只有一次比較，所以最多一個逆序對。 遞迴展開後，每一層 merge 都是「兩個已排序段」對接，貢獻公式 (左半剩餘和) + a[j] × count 都適用。

INITIAL · 原始序列 [5, 4, 2, 1, 3]

Merge 步驟對照

4 個 merge 對應 merge sort 遞迴展開時的 4 個合併節點：

Step左 / 右before → after本步貢獻inv_sum
1[5] | [4][5,4,2,1,3] → [4,5,2,1,3]99
2[4,5] | [2][4,5,2,1,3] → [2,4,5,1,3]1322
3[1] | [3][2,4,5,1,3] → [2,4,5,1,3]022
4[2,4,5] | [1,3][2,4,5,1,3] → [1,2,3,4,5]2951
// 觀察

Step 4 內含 兩次「取右」事件：先比 2 vs 1（取 1），再往後比 4 vs 3（取 3）。 所以雖然是「一個 merge」但會累加多筆貢獻 — 動畫顯示的是 step 完成後的總貢獻 29。 Code 走讀（下一節）會把這個過程拆到迴圈每個 iteration。
