01

重點回顧// Concept Recap

關鍵點內容
問題本質 子集和（subset-sum）：找一組物品使邪惡指數總和 = K，並構造出選法。
為何折半 N≤40 ⇒ 2ᴺ 太大；K≤10¹⁸ ⇒ 不能背包。折半枚舉把指數砍成 N/2。
左半 枚舉 2^(N/2) 個子集，存 (和, mask) 並排序成查找表。
右半 枚舉每個子集和 s，在左半二分搜 K − s。
構造 命中時用兩半的 bitmask 還原選法，逐件 Report(id+1)，最後 Report(-1)。
複雜度 O(2^(N/2)·N) 時間、O(2^(N/2)) 空間；全程 long long。
02

觀念題// Concept Questions

Q1 為何折半

看到哪些線索就該想到折半枚舉？為什麼這題不能用 O(2ᴺ) 暴力，也不能用容量 K 的背包 DP？

Q2 配對原理

把物品切成左右兩半後，為什麼「固定右半子集和 s、在左半找 K−s」就能找出所有合法解？怎麼確定不漏解？

Q3 排序+二分

為什麼左半要排序？排序後配對階段的單次查詢複雜度是多少？不排序會退化成什麼？

Q4 構造解

二分搜只比「和」，為什麼還要在表裡存 mask？回報時左右兩半的 id 分別怎麼算？

Q5 複雜度

整體時間/空間複雜度是多少？N=40 時各階段大概多少次運算？為什麼能在時限內？

03

範例 Trace · solve(3, 17, {6, 8, 9})// Worked Trace

N=3 ⇒ half=1（左半 = 物品1=6）、rest=2（右半 = 物品2=8、物品3=9）。目標 K=17。

// 左半建表（2^1 = 2 個子集），依和排序
mask=0 → 選 {}        和 0
mask=1 → 選 {物品1}   和 6
排序後 leftSum = [ (和0, mask0),  (和6, mask1) ]
// 右半枚舉（2^2 = 4 個子集），二分搜 K − s
mask=0 {}            s=0   need=17−0 =17  二分搜 {0,6} → 沒有
mask=1 {物品2=8}     s=8   need=17−8 = 9  二分搜 {0,6} → 沒有
mask=2 {物品3=9}     s=9   need=17−9 = 8  二分搜 {0,6} → 沒有
mask=3 {物品2,物品3} s=17  need=17−17= 0  二分搜 {0,6} → 命中！（左半 mask0 = 空集）
// 還原 + 回報
左半 mask=0 → 沒選任何物品
右半 mask=3 → 物品2(=half+0+1)、物品3(=half+1+1)
Report(2);  Report(3);  Report(-1);
// OUTPUTReport(2)  Report(3)  Report(-1)     // 8 + 9 = 17 ✓
觀察：這個範例答案完全落在右半（左半選空集）。折半枚舉之所以對，正因為它允許某一半選空集 — mask=0 代表「這半不貢獻」，K 全由另一半湊出。逐一試右半 4 個子集，第 4 個 s=17 時左半只需湊 0（空集即可），命中。
04

應用題// Quick Calculations

App 1 為何 +1

陣列是 0-indexed，但 Report 要 1..N。右半第 b 個被選物品，回報的 id 是多少？為什麼是 half+b+1？

App 2 溢位

為什麼一定要用 long long？子集和最大會到多少？會不會溢位？

App 3 變體

若題目改問「子集和 ≤ K 的最大和」（而非恰好 =K），折半枚舉怎麼改？二分搜改找什麼？

Q1 · 為何折半

線索：N ≤ 40（剛好是 2^(N/2)=2²⁰≈10⁶ 可枚舉的尺寸）+ 子集和問題 + 目標值 K 很大。

  • O(2ᴺ)：2⁴⁰≈1.1×10¹²，逐一檢查跑不完。
  • 背包 DP：狀態是「能否湊出和 v」，v 可達 10¹⁸ ⇒ 開不了 dp[v] 的表，時間 O(N·K) 也爆。

折半枚舉把指數從 N 變 N/2，是 N≤40 子集和的標準解。

Q2 · 配對原理

任一合法解 S 被切成 S_L = S∩左半 與 S_R = S∩右半，互斥且 sum(S_L)+sum(S_R)=K。

當右半枚舉到 mask = S_R 時，need = K − sum(S_R) = sum(S_L)， 而 S_L 是左半的某個子集，其和一定在左半表裡 ⇒ 必被二分搜找到。 因為每個合法解的右半部分都會被枚舉到，所以不漏解。

Q3 · 排序+二分

左半排序後，配對階段對每個右半和 s 只要 O(log 2^(N/2)) = O(N/2) 二分搜 K−s 是否存在。

不排序就得線性掃左半每筆，單次 O(2^(N/2)) ⇒ 總體退化成 O(2^(N/2)·2^(N/2)) = O(2ᴺ)，折半就白做了。 （也可改用 hash set，但排序 + 二分常數穩定、好寫。）

Q4 · 構造解

二分搜只比「和」相不相等，但題目要回報具體選了哪幾件，光知道和不夠。 所以 leftSum 存 (和, mask)，命中後用 mask 的位元還原左半選法。

右半的選法就是當下枚舉的 mask。回報 id（1-indexed）：左半第 b 位 ⇒ Report(b+1)； 右半第 b 位 ⇒ Report(half+b+1)。

Q5 · 複雜度

時間 O(2^(N/2)·N)：建表 2^(N/2) 個子集（各掃 N/2 位）、排序 O(2^(N/2)·N/2)、配對 2^(N/2) 次二分搜（各 O(N/2)）。 空間 O(2^(N/2))（左半表）。

N=40：左半 2²⁰≈10⁶ 筆建表 + 排序、右半 2²⁰ 次二分搜，各乘上 ~20 ⇒ 總約 10⁷～10⁸ 級運算，1 秒內可完成。

App 1 · 為何 +1

右半的物品索引是 half + b（0-indexed，b 是右半內第幾個），對應原物品編號（1-indexed）就是 half + b + 1。

例：範例 half=1，右半 b=0 是物品 1+0+1=2、b=1 是物品 1+1+1=3。所以回報 Report(2)、Report(3)。

App 2 · 溢位

K、A[i] 都可到 10¹⁸，int（上限 ~2.1×10⁹）裝不下 ⇒ 全程 long long。

子集和最大 = ΣA[i] ≤ 10¹⁸，而 long long 上限 ≈ 9.2×10¹⁸，不會溢位。 （這也是題目特意保證 ΣA ≤ 10¹⁸ 的原因。）

App 3 · 變體

改問「子集和 ≤ K 的最大和」：左半一樣建表排序。右半枚舉每個和 s 後，在左半二分搜 「≤ K − s 的最大值」（用 upper_bound 後退一格），取 s + 該值 更新答案最大值。

差別：本題找恰好相等（lower_bound + 比對），變體找不超過的最大（upper_bound − 1）。 兩者都靠「左半排序表 + 二分搜」這個骨架，只換查詢方式。
