重點回顧// Concept Recap

關鍵點內容
分治架構 按 x 切半 → 各自遞迴 → merge 時用 strip 補抓跨界 pair。
Strip 條件 |x − mid_x|² < d — 純整數比平方，避開 sqrt。
7 鄰居上限 strip 按 y 排序後，pigeonhole 保證每點至多 7 個鄰居可能 < d。
Base case n ≤ 3 brute 3 對；遞迴最深處沒有任何巧思。
複雜度 T(N) = 2 T(N/2) + O(N log N) = O(N log² N)。
純整數 全程 long long，無 sqrt，避免浮點誤差。

範例 Trace · 跑一次 code 看會輸出什麼// Worked Trace

用一個小範例跑 code 一步一步看：N = 4 個點

idx0123
pointP1P2P3P4
x1246
y1632

輸入：

// INPUT4
1 1
2 6
4 3
6 2

已按 x 排序好 (本例已遞增)，呼叫 closest(pts, 0, 3)：

// CALL 1 · closest(pts, 0, 3)
n = 4 > 3 → 不是 base case
mid = (0 + 3) / 2 = 1
mid_x = pts[1].x = 2

→ 遞迴左半 closest(pts, 0, 1)
→ 遞迴右半 closest(pts, 2, 3)
// CALL 2 · closest(pts, 0, 1)
n = 2 ≤ 3 → base case (brute)
比較 P1−P2: dist²(P1, P2) = (1−2)² + (1−6)² = 1 + 25 = 26
return 26
// CALL 3 · closest(pts, 2, 3)
n = 2 ≤ 3 → base case (brute)
比較 P3−P4: dist²(P3, P4) = (4−6)² + (3−2)² = 4 + 1 = 5
return 5
// CALL 1 · 合併 strip 階段
d_L = 26
d_R = 5
d   = min(26, 5) = 5

收集 strip (條件 dx² < d = 5):
  P1(1,1): dx² = (1−2)² = 1 < 5 ✓
  P2(2,6): dx² = (2−2)² = 0 < 5 ✓
  P3(4,3): dx² = (4−2)² = 4 < 5 ✓
  P4(6,2): dx² = (6−2)² = 16 ❌
strip = [P1, P2, P3]

按 y 排序 strip:
  P1(y=1), P3(y=3), P2(y=6)

7-neighbor 檢查:
  i=0 (P1):
    j=1 (P3): dy² = (3−1)² = 4 < 5
              dist² = (1−4)² + (1−3)² = 9 + 4 = 13 ≥ 5 (未更新)
    j=2 (P2): dy² = (6−1)² = 25 ≥ 5 → break
  i=1 (P3):
    j=2 (P2): dy² = (6−3)² = 9  ≥ 5 → break
  i=2 (P2): 無後續

d 維持 5
return 5
// OUTPUT5
觀察：本範例右半的 P3-P4 就是最佳 pair，strip 沒有抓到更近的。 不過 strip 仍然檢查了 P1-P3（dist²=13）— 這是必要的代價， 因為若 P1 比 P3 更近的鄰居在另一邊，strip 就會找出來。
04

應用題// Quick Calculations

A1 最壞情況

所有點都集中在中線上（x 值都相同），strip 會多大？演算法還能跑嗎？

A2 遞迴深度

N = 2 × 10⁵ 時 closest 的最大遞迴深度是多少？

A3 浮點座標

如果輸入座標是 double（非整數），整段 code 要做哪些調整？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、A 對應 A。

A1 · strip 寬度

左右遞迴各自保證內部距離 ≥ d。跨界 pair 若要 < d， 則它在 x 軸的差也 < d。所以兩個點都必須在 [mid − d, mid + d] — 寬度恰好 2d。

  • 取 1d：會漏掉兩個端點剛好分別距中線略小於 d 的 pair（兩者距離 < d 但都在 ±d 範圍）。
  • 取 4d：正確性沒問題但 strip 變大、無意義候選變多、效能變差。
A2 · pigeonhole

把 2d × d 矩形切成 4 個 (d/2) × (d/2) 子格。每子格對角線 d / √2 ≈ 0.71 d < d，所以子格內兩點必互相 < d。

但左半已遞迴出 d_L ≥ d，即左半任兩點距離 ≥ d ⇒ 左半 2 個子格各最多 1 個點。 右半同理。整個 2d × d 矩形最多 4 點。 擴成 2d × 2d（兩矩形上下疊）⇒ 最多 8 點（含自己）。

所以對 strip[i] 而言，後續至多 7 個鄰居可能與它構成 < d 的距離 — 檢查這 7 個即可。

A3 · 排序方向

要按 y 排序。因為跨界候選 pair 必滿足 「|Δx| < d 且 |Δy| < d」。 x 軸由 strip 條件保證；y 軸用排序+只看下幾個保證。

若按 x 排序，strip 內 x 値會非常接近（都在 ±d 範圍），無法分隔候選 — 等同 O(N²) 暴力比較整個 strip。

A4 · 資料型別

座標 ≤ 10⁹，Δx 可達 2 × 10⁹。int 上限 ≈ 2.147 × 10⁹， 2 × 10⁹ × 2 × 10⁹ = 4 × 10¹⁸ — 乘法當下就 溢位。 

// EXAMPLE (int 溢位)
P_a = (10⁹, 0)
P_b = (-10⁹, 0)
dx  = 10⁹ - (-10⁹) = 2 × 10⁹

int:    dx * dx = 4 × 10¹⁸  ❌ 溢位 (int 最大 2.1 × 10⁹)
long long: dx * dx = 4 × 10¹⁸  ✓ (long long 最大 ~9.2 × 10¹⁸)

所以 Point.x、Point.y、dist2() 全部用 long long。

A5 · 最佳化

瓶頸是每層 merge 都對 strip 做 sort by y，每層 O(N log N) → 總 O(N log² N)。

消掉這個 log 的辦法：

  • 進入遞迴前，建立一份「按 y 排序」的索引副本。
  • 遞迴每層回傳「該子集合按 y 排序好的結果」。
  • 合併兩個已 y-排序的子集合：用 merge sort O(N) 合併。
  • strip 從這個 y-排序好的合併結果線性掃出來即可。

單層工作降為 O(N)，總 T(N) = 2 T(N/2) + O(N) = O(N log N)。

A · App 1

所有點 x 相同（極端 case）時，strip 條件 |x − mid_x|² = 0 < d 對所有點都成立 ⇒ strip = 全體 N 點。

看起來像爆炸，但實際還是 OK：strip 按 y 排序後，每點只看 7 個鄰居 ⇒ 仍是 O(N) merge。 worst-case 仍然 O(N log² N)，安全通過。

A · App 2

log₂(2 × 10⁵) ≈ 17.6，所以最深約 18 層。 遠遠在預設 stack 大小可承受範圍內，不需要改 stack 或改 iterative。

A · App 3

三個改動：

  • Point.x 與 .y 改為 double。
  • dist2 仍可用，回傳 double。比較 dx*dx < d 仍正確。
  • 但浮點比較有誤差 — 嚴格的題目可能要 d + ε 或 sqrt 後比。

實務上整數座標就用整數；只在輸入本身是浮點時才改 — 能用整數就用整數。
