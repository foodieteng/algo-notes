01

題目簡介// Brief

QUESTION

無向帶權圖（N 城市、M 條道路計劃，計劃 i 連 uᵢ–vᵢ 花費 wᵢ）。 資芽國要用最小總花費把全部城市連通（= 最小生成樹 MST）。但達到最小花費的方法可能不只一種 （多棵 MST）。小 Y 想讓「最終建造方式唯一」，又不能改變最小花費本身。 問：最少要刪掉幾條道路計劃，才能讓 MST 變唯一？

1 ≤ N ≤ 10⁵ · 0 ≤ M ≤ 3×10⁵ · 1 ≤ wᵢ ≤ 10⁹ · uᵢ ≠ vᵢ，保證圖連通。 換句話說：刪最少邊使「最小花費不變」且「MST 唯一」。

△ REJECT · 太慢

同權重邊一起處理：候選邊數 − 實際合併數 = 該類別冗餘邊，全部加起來就是答案。

關鍵觀察：MST 不唯一只會發生在同權重邊之間的選擇。 把邊按權重分組、依 Kruskal 順序處理，每一組裡「能連接不同連通塊但用不到的多餘候選邊」 就是製造選擇的元兇 —— 數出來、刪掉它們即可。

03

核心策略 · 每個權重類別做三件事// Algorithm Outline

把邊按權重由小到大排序、分組。處理某個權重 w 的那一組時：

  1. 找候選邊 · 在「處理完所有更輕邊」的 DSU 狀態下，挑出這組裡 find(u) ≠ find(v) 的邊（兩端在不同連通塊）—— 這些才有資格進 MST。
  2. 合併並數冗餘 · 對候選邊逐一 union；真正造成合併的數量 = merges。 候選數 − merges = 這組要刪的冗餘邊。
  3. 累加 · 把每組的冗餘邊數加總，就是答案（讓 MST 唯一所需的最少刪除數）。

為何「候選 − merges」就是冗餘 · 一組裡若有 c 條候選邊但只需 m 次合併， 多出來的 c − m 條都是「可換可不換」的選擇來源（平行邊或在收縮圖上成環）⇒ 全刪掉才唯一。

05

為何非唯一只發生在同權重之間// Why Equal Weights Only

這題的正確性建立在 MST 的兩個性質上。

MST 唯一性 · 兩個關鍵事實
  1. 每個權重類別的「合併次數」是固定的。 不管選哪些等權邊，處理完權重 ≤ w 的所有邊後， 連通塊結構完全相同 ⇒ 每個權重在 MST 裡貢獻的邊數固定（這也是「最小花費不變」的保證）。
  2. 選擇只來自同權重組內。 跨權重不會有選擇：更輕的邊永遠優先，Kruskal 不會猶豫。 非唯一只可能是「同權重的多條候選邊，連接同一批連通塊，挑誰都行」。
  3. 所以在處理權重 w 的那組時，把已收縮的連通塊當點，這組候選邊在上面若有多餘 （平行邊、或成環），就是選擇來源。需要 merges 次合併、卻有 c 條候選 ⇒ 多餘 c − merges 條。
  4. 刪掉這 c − merges 條後，這組剛好只剩「每次合併唯一可用的邊」⇒ 無從選擇 ⇒ 該層唯一。 各層都唯一 ⇒ 整棵 MST 唯一，且因為合併次數不變、最小花費不變。

實測佐證：對 2900+ 組隨機圖，「分權重類別數冗餘」與「窮舉刪最少邊使 MST 唯一」 答案完全一致、零反例，且 N=10⁵、M=3×10⁵ 在 1 秒內跑完。
