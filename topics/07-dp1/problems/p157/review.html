01

重點回顧// Concept Recap

關鍵點內容
問題轉譯 第 i 類選 aᵢ 位鄰居 = C(M, aᵢ)；各類獨立，不限「至少一個」時總數 = ∏ C(M, aᵢ)。
為何要扣 ∏ C(M, aᵢ) 多算了「有鄰居完全沒拿到」的分法。
容斥公式 答案 = Σ (−1)ʲ C(M,j) ∏ C(M−j, aᵢ)，j = 排除的鄰居數。
邊界 aᵢ > M−j ⇒ C(M−j, aᵢ)=0，整項自動歸零。
模數陷阱 1000007 非質數 ⇒ C(n,k) 必用巴斯卡三角（禁用逆元）。
複雜度 O(M·N)；建表 O(M²) 全程一次。

觀念題// Concept Questions

Q1 建模

為什麼第 i 類的分法數是 C(M, aᵢ)？哪兩個題目條件分別對應「選」與「不重複」？

Q2 為何要容斥

∏ C(M, aᵢ) 算出來的數，比正確答案多算了哪些分法？為什麼不能只減一次？

Q3 公式推導

容斥第 j 項 C(M,j)·∏C(M−j,aᵢ) 中，C(M,j) 和 C(M−j,aᵢ) 各代表什麼？

Q4 模數

為什麼這題不能用「階乘 + 逆元」算組合數？換成質數模（如 1e9+7）就可以了嗎？

Q5 邊界

當 aᵢ > M−j 時這一項應該是多少？這在程式裡是怎麼自然處理的？

範例 Trace · N=3, M=3, a=[1,2,2]// Worked Trace

對應動畫那筆（答案 21）。需要的組合數：C(3,0)=1, C(3,1)=3, C(3,2)=3, C(2,1)=2, C(2,2)=1, C(1,2)=0。

// INPUT3 3
1 2 2       // 第1類1個、第2類2個、第3類2個；M=3 位鄰居
// j = 0  (不排除，剩 3 位)  → 加
C(3,0) · C(3,1) · C(3,2) · C(3,2)
  =  1   ·   3   ·   3   ·   3   = 27
// j = 1  (排除 1 位，剩 2 位)  → 減
C(3,1) · C(2,1) · C(2,2) · C(2,2)
  =  3   ·   2   ·   1   ·   1   = 6
// j = 2  (排除 2 位，剩 1 位)  → 加
C(3,2) · C(1,1) · C(1,2) · C(1,2)
  =  3   ·   1   ·   0   · …       = 0   (a₂=2 > 剩 1 位 ⇒ 歸零)
// j = 3  (排除 3 位，剩 0 位)  → 減
C(3,3) · C(0,1) · …
  =  1   ·   0   · …             = 0   (a₁=1 > 剩 0 位 ⇒ 歸零)
// OUTPUT27 − 6 + 0 − 0 = 21
觀察：j=0 的 27 是「不管至少一個」的全集；j=1 的 −6 扣掉「至少有 1 位被晾著」的分法； j≥2 因為剩的鄰居不夠分（某類要的 aᵢ 比剩的人多）自然歸零。最終 21。
04

應用題// Quick Calculations

App 1 手算

對 N=2, M=3, a=[2,1]（答案 3），逐項寫出容斥的每一項並加總。

App 2 特例

若所有 aᵢ = M（每類都要分給全部鄰居），答案是多少？用公式說明為什麼。

App 3 變體

若把限制改成「每位鄰居剛好拿到 1 個」（而非至少 1 個），這題還能用同一套容斥嗎？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · 建模

第 i 類有 aᵢ 個、同類視為相同，且同一位鄰居最多拿 1 個同類。 所以分配第 i 類就是「從 M 位鄰居中挑出 aᵢ 位各給一個」，順序、個體相同性都已涵蓋 ⇒ C(M, aᵢ)。

其中「挑 aᵢ 位」對應同類視為相同（不是排列而是組合）；「每位最多 1 個同類」對應不重複選取（同一位不會被選兩次）。

Q2 · 為何要容斥

∏ C(M, aᵢ) 完全沒管「每位鄰居至少一個」，所以把某些鄰居完全沒被任何一類選到的分法也算了進去 — 那些違反條件，要扣掉。

不能只減一次，因為「鄰居 A 沒拿到」與「鄰居 B 沒拿到」的分法集合會重疊（同時 A、B 都沒拿到的分法被減了兩次）。 容斥的交錯正負號正是把這些被多扣／少扣的部分逐層修正回來。

Q3 · 公式推導

C(M, j) = 「指定哪 j 位鄰居被排除（強制不分給他們）」的選法數。

∏ C(M−j, aᵢ) = 在剩下的 M−j 位裡，第 i 類仍要選 aᵢ 位來給的分法數，各類獨立相乘。 兩者相乘是「排除這特定 j 位」的分法總和；再乘 (−1)ʲ 交錯，即容斥。

Q4 · 模數

1000007 = 29 × 34483 是合數。逆元法要算 (k!)⁻¹，但當 k! 與模數不互質時逆元不存在 ⇒ 整套失效。 本題只能用巴斯卡三角加法遞推 C[n][k]=C[n−1][k−1]+C[n−1][k]，全程只用加法，對任何模數都成立。

若換成質數模（如 1e9+7），任何小於模數的非零數都互質、逆元都存在 ⇒ 就可以用階乘 + 逆元的 O(1) 查詢法。 本題 N,M ≤ 100，巴斯卡三角已足夠。

Q5 · 邊界

當 aᵢ > M−j：剩下的 M−j 位鄰居根本不夠選出 aᵢ 位 ⇒ C(M−j, aᵢ) = 0，整項乘積為 0。

程式裡這由 comb 表自然回 0（上三角外都是 0），或如本題碼提早 break 並令 term=0 — 兩種都正確，後者省乘法且避免讀表外。

App 1 · 手算 [2,1], M=3 
// 需要：C(3,0)=1 C(3,1)=3 C(3,2)=3 C(2,2)=1 C(2,1)=2 C(1,2)=0
j=0 (+): C(3,0)·C(3,2)·C(3,1) = 1·3·3 = 9
j=1 (−): C(3,1)·C(2,2)·C(2,1) = 3·1·2 = 6
j=2 (+): C(3,2)·C(1,2)·…      = 3·0   = 0   (a=2 > 剩1)
j=3 (−): C(3,3)·C(0,2)·…      = 0
合計 = 9 − 6 + 0 − 0 = 3 ✓
App 2 · 所有 aᵢ = M

每一類都必須分給全部 M 位鄰居 ⇒ 每位鄰居每類都拿到一個 ⇒ 只有一種分法，答案 = 1。

用公式看：j=0 項 = ∏ C(M, M) = 1；任何 j ≥ 1 時剩 M−j < M，而 aᵢ = M > M−j ⇒ 該項為 0。 故總和 = 1 − 0 + … = 1。（對應 Sample 末筆 3 3 / 3 3 3 → 1。）

App 3 · 剛好 1 個的變體

「剛好 1 個」是更強的限制：總共要分出的高棕櫚數必須恰好 = M（Σ aᵢ = M），否則無解（0 種）。

在 Σ aᵢ = M 時，等於把 M 個有顏色的位置（各類各 aᵢ 個、同類同色）排到 M 位不同鄰居上、每人剛好一個 ⇒ 多項式係數 M! / (a₁! a₂! … a_N!)。這不再需要容斥，是另一個計數模型 — 條件一改，工具就換。
