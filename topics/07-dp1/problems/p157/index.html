分送高棕櫚

// 組合計數 · 容斥原理（Inclusion–Exclusion）

★ ★ ★ ★ ☆ 核心 容斥 + 巴斯卡三角預處理 C(n,k) · 時間 O(M·N)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

高棕櫚分成 N 類，第 i 類有 aᵢ 個（同類視為相同）。要分送給 M 位鄰居（每位鄰居是不同個體），且：

  • 每位鄰居至少分到一個高棕櫚（任何一類都算）。
  • 同一位鄰居不會拿到兩個同類的高棕櫚。

問共有幾種分配方式？輸出 mod 1000007。 多筆 T ≤ 10，1 ≤ N, M ≤ 100，1 ≤ aᵢ ≤ M。

關鍵轉譯：第 i 類有 aᵢ 個、每位鄰居最多拿 1 個同類 ⇒ 第 i 類就是「從 M 位鄰居中選 aᵢ 位來給」， 共 C(M, aᵢ) 種。各類獨立 ⇒ 若不管「每人至少一個」，總數是 ∏ C(M, aᵢ)。但這會把有人完全沒拿到的分法也算進去 — 那正是要扣掉的。

02

為什麼用容斥// Why Inclusion–Exclusion

A 枚舉每位鄰居拿哪些類 O(2^(NM))
每個 (類, 鄰居) 格子要不要給是 0/1，共 NM 個格子 ⇒ 2^(NM) 種，再驗證限制。N=M=100 完全爆炸。
× REJECT · TLE
B 直接算 ∏ C(M, aᵢ) O(N)
忽略「每人至少一個」⇒ 多算了某些鄰居完全沒被選到的分法。少了下界限制，答案偏大。
△ REJECT · OVERCOUNT
BOTH FAIL · ADOPT
C 容斥扣掉「有人沒拿到」 O(M·N)
以「被排除的鄰居數 j」做容斥：硬性不給某 j 位，剩 M−j 位自由分 = ∏ C(M−j, aᵢ)；交錯加減把重複扣回來。
● ADOPT · STABLE

為何要容斥而非直接扣？ 「至少 1 位沒拿到」的分法們彼此重疊（A 沒拿到、B 沒拿到的情況會重複計）， 不能單純減一次。容斥的交錯正負號正是把這些重複精確抵銷的標準工具。

03

核心策略 · 三步驟// Strategy

設 U = 所有分法（不管至少一個），A_t = 「鄰居 t 完全沒拿到」的分法集合。 要算的是不落入任何 A_t 的分法數。

答案 = Σj=0M (−1)j · C(M, j) · ∏i C(M−j, aᵢ)
固定排除 j 位鄰居（哪 j 位有 C(M,j) 種選法）；剩 M−j 位裡，第 i 類選 aᵢ 位 = C(M−j, aᵢ)； 各類相乘。交錯正負號 (−1)ʲ 即容斥。
  1. 預處理 · 用巴斯卡三角建 C(n,k) 表（n,k ≤ 100），全程 mod 1000007。
  2. 枚舉 j = 0..M：算這一項 C(M,j) · ∏ C(M−j, aᵢ)，若某 aᵢ > M−j 則 C=0，整項歸零。
  3. 交錯累加 · j 為偶數加、奇數減（減完 +MOD 再取模，避免負數）。

05

白癡版：球放盒子（先懂這個就懂高棕櫚）// Warm-up

先用一個小到能用手數的例子把排容講清楚，再回頭看公式怎麼長出來。 4 顆不同的球（1,2,3,4）放進 3 個不同的盒子（A,B,C），每個盒子都不能空，問幾種放法？

① 先別管「不能空」：每顆球可選 A/B/C 共 3 種，4 顆獨立 ⇒ 3⁴ = 81 種。但這裡面混進了「有盒子空」的壞情況（如 4 顆全進 A）。

② 天真地扣「某盒空」：A 空 ⇒ 球只能進 B/C ⇒ 2⁴=16；B 空、C 空同理。 天真算法 81 − 16×3 = 33 — 但這是錯的！

// 重複扣的陷阱（排容的核心）
  1. 「A 空」那 16 種裡，包含了「A、B 同時空」（4 顆全進 C）；「B 空」那 16 種裡也包含它。
  2. 所以減「A 空」又減「B 空」時，「A、B 同時空」被扣了兩次 ⇒ 多扣了 ⇒ 要加回來一次。
  3. 「兩個指定盒子空」⇒ 球全進剩下 1 個 ⇒ 1⁴=1；挑哪 2 個盒子空有 C(3,2)=3 種 ⇒ 加回 3×1=3。
  4. 「三個盒子全空」⇒ 沒地方放 4 顆球 ⇒ 0⁴=0，不可能。
81 − 3×16 + 3×1 − 1×0 = 81 − 48 + 3 − 0 = 36
減太多就加回、加太多再減掉 ⇒ 正負交錯，讓每個「合法放法」剛好被算 1 次、每個「有空盒」剛好 0 次 — 這就是排容原理

把四項用統一寫法（令 k = 強制幾個盒子空）：

k（強制空）挑哪 k 個空每球剩幾種該項值符號
0C(3,0)=133⁴ = 81+
1C(3,1)=322⁴ = 16−
2C(3,2)=311⁴ = 1+
3C(3,3)=100⁴ = 0−
每項 = (−1)ᵏ × C(3,k) × (3−k)⁴　⇒　Σk=03 (−1)ᵏ C(3,k)(3−k)⁴ = 36
(−1)ᵏ = 減/加交錯的符號 · C(3,k) = 挑哪 k 個盒子強制空 · (3−k)⁴ = 4 顆球各自只能在剩 3−k 個盒子裡選
和高棕櫚一模一樣，只換掉「每顆球的放法數」：
球放盒子：每球在剩盒有 (3−k) 種 → 連乘 (3−k)⁴
高棕櫚：每類在剩 M−k 位鄰居有 C(M−k, aᵢ) 種 → 連乘 ∏ C(M−k, aᵢ)。同一招、換個外衣而已。
06

為什麼這個式子是對的// Correctness

// INCLUSION–EXCLUSION · 扣掉「有人完全沒拿到」
  1. 令 A_t = 「鄰居 t 完全沒分到任何高棕櫚」的分法集合。我們要的是不在任何 A_t 裡的分法數，即 |U| − |A₁∪…∪A_M|。
  2. 容斥：|A₁∪…∪A_M| = Σ|A_t| − Σ|A_t∩A_s| + …。第 j 項是「指定 j 位鄰居全沒拿到」的所有組合之和。
  3. 「指定某 j 位鄰居全沒拿到」⇒ 每一類只能分給剩下的 M−j 位 ⇒ 第 i 類有 C(M−j, aᵢ) 種；各類獨立相乘 = ∏ C(M−j, aᵢ)。
  4. 「哪 j 位被指定」有 C(M, j) 種，且每一種貢獻相同（對稱）⇒ 第 j 項總和 = C(M,j) · ∏ C(M−j, aᵢ)。
  5. 合併符號：把 |U| 視為 j=0 那一項（C(M,0)·∏C(M,aᵢ)），整體就是 Σ (−1)ʲ C(M,j) ∏ C(M−j, aᵢ)。

邊界自洽：當 aᵢ > M−j 時 C(M−j, aᵢ)=0，整項自動歸零 — 因為「只剩 M−j 位卻要從中選出 aᵢ 位」根本不可能，符合直覺。

07

複雜度// Complexity

M+1 項 × 每項算 N 個 C 相乘 = O(M·N) 每筆
C(n,k) 表用巴斯卡三角一次 O(M²) 建好（全測資共用）；N,M ≤ 100 ⇒ 每筆 ~10⁴，T≤10 秒過
項目數量備註
TimeO(M·N) 每筆建表 O(M²) 全程一次
SpaceO(M²)C[n][k] 巴斯卡三角表
遞迴深度0（迭代）純枚舉 j
Constant factor低每項一次乘法鏈

⚠ 模數陷阱：1000007 = 29 × 34483 不是質數 ⇒ 不能用「除法 = 乘逆元」算 C(n,k)！ 必須用巴斯卡三角加法遞推 C[n][k]=C[n−1][k−1]+C[n−1][k]（只用加法，永遠合法）。

08

一句洞察// One-Sentence Insight

每類獨立選 aᵢ 位鄰居 ⇒ 不限「至少一個」的總數是 ∏ C(M, aᵢ)；
用容斥「排除 j 位鄰居」把多算的扣掉 —
答案 = Σ (−1)ʲ C(M,j) ∏ C(M−j, aᵢ)，且因模數非質數，C(n,k) 必用巴斯卡三角。
