01

完整實作 · 排序 + 逐秒模擬// Full C++

咒術用平行陣列存（門檻 need[i]=pᵢ·M、傷害 dealt[i]=cᵢ），依門檻由大到小排序。 逐秒模擬，已解鎖的咒術丟進最大堆，每秒彈最強來放。題目每筆一組 N M R（無測資數），讀到 EOF 為止。

p326.cpp C++17 
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

const int MAXN = 100005;
ll need[MAXN];      // need[i] = pᵢ × M：當 hp×100 ≤ need[i] 時可施放
ll dealt[MAXN];     // dealt[i] = cᵢ：每秒造成的傷害
int order[MAXN];     // 咒術索引，依 need 由大到小排

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    ll M, R;
    while (cin >> N >> M >> R) {
        for (int i = 0; i < N; i++) {
            ll pi, ci;
            cin >> pi >> ci;
            need[i] = pi * M;
            dealt[i] = ci;
            order[i] = i;
        }
        // 依解鎖門檻由大到小：pᵢ 大者血量還高時就能放
        sort(order, order + N,
             [](int a, int b) { return need[a] > need[b]; });

        priority_queue<ll> ready;   // 已解鎖、未施放的咒術，以 c 為鍵的最大堆
        int ptr = 0;
        ll hp = M, dps = 0, sec = 0, answer = -1;
        ll guard = M + N + 5;          // 有限答案 ≤ M+N，超過即視為打不死

        while (true) {
            hp -= dps;                       // ① 已施放咒術一起扣血
            hp += R; if (hp > M) hp = M;   // ② 回血，封頂 M
            if (hp <= 0) { answer = sec; break; }   // ③ 判死

            // 血量已降到門檻的咒術，全部解鎖進堆
            while (ptr < N && need[order[ptr]] >= hp * 100) {
                ready.push(dealt[order[ptr]]);
                ptr++;
            }
            if (!ready.empty()) {              // ④ 放最強的
                dps += ready.top();
                ready.pop();
            } else if (dps <= R) {          // 沒咒可放且扣血≤回血 ⇒ 永遠打不死
                answer = -1; break;
            }

            sec++;
            if (sec > guard) { answer = -1; break; }
        }
        cout << answer << '\n';
    }
    return 0;
}

為什麼用 ptr 單調掃描？ 血量只在 dps>R 後單調下降，門檻已排序 ⇒ 每個咒術一生只會被 ptr 掃過、進堆一次， 整體解鎖成本 O(N)。堆只負責「在已解鎖的之中挑最強」。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
門檻方向 是「生命值 ≤ pᵢ%·M 才可放」（題目說生命值大於 pᵢ% 時無法施放）。別寫反成 ≥。
百分比用整數比較 用 hp × 100 ≤ pᵢ × M，不要 hp ≤ pᵢ/100 × M（浮點誤差會在邊界出錯）。
第 0 秒就能放 流程從第 0 秒起：扣血(此時 dps 可能=0)→回血→判死→施放。滿血時 p=100 的咒術即可放（hp×100 = 100M ≤ 100·M）。
判 −1 沒咒可放且 dps ≤ R ⇒ 血量不再下降，永遠打不死。漏判會死迴圈。guard=M+N 是雙保險。
回血封頂 回血後 hp = min(hp, M)；早期 dps 小時血會頂在 M，別讓它超過。
型別 dps 累加可達 N·c ≈ 10¹⁰、need=pᵢ·M ≈ 10⁷ ⇒ 一律 long long。
輸入格式 本題沒有測資數 T，第一行直接是 N M R；用 while(cin>>N>>M>>R) 讀到 EOF。

細節走讀// Section Walkthrough

① 每秒的四步順序

// per second
hp -= dps            // 1. 已施放咒術扣血
hp = min(hp+R, M)    // 2. 回血、封頂
if hp <= 0: 打敗於 sec// 3. 判死
解鎖 need >= hp*100 的咒術進堆；放堆中 c 最大者 // 4. 施放

② 解鎖：單調指標 + 最大堆

// unlock
咒術依 need(=pᵢ·M) 由大到小排序
血量下降時，ptr 往後把所有 need >= hp*100 的推進堆
堆以 c 為鍵 → top() 就是當前可放的最強咒術

③ 終止：打敗 / −1 / guard

// terminate
hp<=0            → 答案 = sec
無咒可放且 dps<=R → 答案 = -1（血不再降）
sec > M+N        → -1（保險；有限答案必 <= M+N）
