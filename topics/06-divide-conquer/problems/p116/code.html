完整實作// Full C++

互動式題目：實作 solve(N, M)，由 judge 提供 GetVal(row, col) 與 Report(col)。不需 main()。

p116.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const int N_MAX = 1000005;
int answer[N_MAX];   // answer[row] = 該列峰的欄位 (1-indexed)

// 在 row 中、欄位範圍 [colLo, colHi] 內 brute 找峰，寫入 answer[row]。
void scan(int row, int colLo, int colHi) {
    int bestCol = colLo;
    long long bestVal = GetVal(row, colLo);
    for (int c = colLo + 1; c <= colHi; ++c) {
        long long v = GetVal(row, c);
        if (v > bestVal) { bestVal = v; bestCol = c; }
    }
    answer[row] = bestCol;
}

// 在列範圍 [rowLo, rowHi]、欄位範圍 [colLo, colHi] 內遞迴找每列峰。
void dnc(int rowLo, int rowHi, int colLo, int colHi) {
    if (rowLo > rowHi) return;
    int mid = (rowLo + rowHi) / 2;

    scan(mid, colLo, colHi);             // 找 row mid 的峰
    int m = answer[mid];

    dnc(rowLo, mid - 1, colLo, m);      // 上半：列峰必 ≤ m
    dnc(mid + 1, rowHi, m, colHi);      // 下半：列峰必 ≥ m
}

void solve(int N, int M) {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    dnc(1, N, 1, M);

    // 遞迴順序不是 row 1..N — 必須依 row 序輸出
    for (int i = 1; i <= N; ++i) Report(answer[i]);
}

複雜度分析// Complexity Derivation

把 dnc() 一層的工作量拆開：

步驟內容單呼叫成本
取 mid整數運算O(1)
scan(mid, colLo, colHi)brute force 掃過 colHi − colLo + 1 個欄位，每格 1 次 GetValO(colHi − colLo + 1)
遞迴上下2 個子問題（列數對半切）2 · T(N/2)

每層欄位總長：同一層所有子問題的 [colLo, colHi] 加起來 — 相鄰子問題只在端點 m 重疊 1 格，所以總長 ≤ M + 子問題數 ≤ O(M)。

T(N) = 2 · T(N/2) + O(M_layer)  且  ∑ M_layer ≤ O(M) per level
共 O(log N) 層 ⇒ Total = O((N + M) log N)
項目數量備註
TimeO((N + M) log N)每層欄位總長 O(M) + 子問題遞迴呼叫 O(N)
SpaceO(N)answer[1..N] 全域陣列
Recursion DepthO(log N)N = 10⁶ ⇒ 約 20 層
GetVal Calls≤ (N + M) log₂ NN = M = 10⁶ ⇒ 2 × 10⁶ × 20 ≈ 4 × 10⁷ < 10⁸ ✓

子任務對照： N ≤ 10 ⇒ 100 次以內； N ≤ 100 ⇒ 10⁴ 次以內； N ≤ M ≤ 10⁶ 滿分 ⇒ ~4 × 10⁷ 次。三檔都遠在預算內。

03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
GetVal 呼叫上限 10⁸ 暴力 O(N·M) = 10¹² 直接爆。分治壓到 (N+M) log N ≈ 4 × 10⁷。不是時間限制，是呼叫次數限制。
不要在遞迴中 Report 遞迴順序是 row 2 → row 1 → row 3 → row 4（mid-first），不是 row 1..N。必須先 answer[i] 全部存好，最後再依 i 順序 Report。
欄位範圍端點包含 m 傳給上半的是 [colLo, m]、下半是 [m, colHi]，端點 m 兩邊都包含。題目「嚴格遞增」其實保證 p_i ≤ m − 1，但程式裡寫 m 更乾淨，不會錯（最差情況該欄位被多看 1 次）。
1-indexed GetVal(row, col) 與 Report(col) 都是 1-indexed。answer[] 開到 [1..N]，dnc(1, N, 1, M)。
用 long long 比較 題目數值範圍未明示，保險用 long long bestVal。即使 spec 是 int，也不會吃虧。
遞迴 base case 不必特判 n=1 rowLo == rowHi 時 mid = rowLo，scan 一次後上下半皆空 — 自然收斂，不需要寫 if (n == 1)。
不要包含 lib0116.h 那是 judge 端的標頭。提交時不寫 include，函式直接呼叫 — judge 會幫忙連結。

細節走讀// Section Walkthrough

① SCAN MID ROW

// SCAN MID ROW
mid = (rowLo + rowHi) / 2
scan(mid, colLo, colHi):
    bestCol = colLo
    bestVal = GetVal(mid, colLo)
    for c in [colLo + 1, colHi]:
        v = GetVal(mid, c)
        if v > bestVal:
            bestVal = v
            bestCol = c
    answer[mid] = bestCol

→ 共 (colHi − colLo + 1) 次 GetVal。

② RECURSE UPPER (上半列：列峰必 ≤ m)

// RECURSE UPPER
m = answer[mid]
dnc(rowLo, mid - 1, colLo, m)

→ 上半列數 = (mid - 1) - rowLo + 1 ≤ ⌈N/2⌉
→ 上半欄位範圍縮為 [colLo, m]
→ rowLo > rowHi 即 stop（mid 已是最上時 rowLo = mid, rowHi = mid - 1）

③ RECURSE LOWER (下半列：列峰必 ≥ m)

// RECURSE LOWER
dnc(mid + 1, rowHi, m, colHi)

→ 下半列數 = rowHi - (mid + 1) + 1 ≤ ⌊N/2⌋
→ 下半欄位範圍縮為 [m, colHi]
→ 同樣 rowLo > rowHi 即 stop

④ REPORT IN ORDER

// REPORT IN ORDER
dnc 完成後，answer[1..N] 已全部填好。
for i in [1, N]:
    Report(answer[i])

→ 遞迴順序是 mid 先（不是 row 1..N）；
→ 必須等全部 scan 完，再依 i 順序輸出。
