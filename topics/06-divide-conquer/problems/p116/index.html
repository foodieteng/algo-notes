01

題目簡介// Brief

QUESTION

給 N × M 防禦力矩陣（1 < N ≤ M ≤ 10⁶）。對每一列找重要守護點 — 該列唯一的最大值欄位。 題目保證：每列的最大值欄位嚴格大於上一列的最大值欄位（i.e. row-peak 欄位嚴格遞增）。

互動式：實作 solve(N, M)，judge 提供 int GetVal(row, col) 與 void Report(col)。 GetVal 呼叫上限 10⁸。

暴力 N · M = 10¹² 完全爆 — 必須利用「列峰嚴格遞增」這條結構性條件，把搜尋範圍逐列縮小。

03

核心策略 · 分治 on rows// Algorithm Outline

想像一下：每一列都有一位太陽軍團隊長（該列的最大值），題目保證隊長們的位置從上到下排成一條嚴格往右下走的階梯（第 i 列的隊長一定比第 i−1 列的隊長更靠右）。

暴力法每列都掃整排 M 個格子才能找出隊長，N 列就要 N · M 次 GetVal — 一定爆。但「隊長階梯」這個結構給了我們一個免費資訊：只要知道中間那位隊長站在哪一欄，就能立刻砍掉上下半的搜尋範圍。

具體做法（當前要處理 rowLo..rowHi 列、欄位範圍 colLo..colHi）：

  1. 挑正中間那列：mid = (rowLo + rowHi) / 2
  2. 派偵察兵掃一次找出 mid 列的隊長：在 cols [colLo, colHi] 內逐格比大小，找到峰欄 m，記下 answer[mid] = m
  3. 上半的隊長一定在 m 左邊（階梯往右下走）⇒ 之後上半只需在 cols [colLo, m−1] 裡找。遞迴呼叫 dnc(rowLo, mid−1, colLo, m−1)
  4. 下半的隊長一定在 m 右邊 ⇒ 之後下半只需在 cols [m+1, colHi] 裡找。遞迴呼叫 dnc(mid+1, rowHi, m+1, colHi)

一句話：掃中間列找隊長 → 上下半各自的搜尋區間被砍成更窄的兩塊 → 對上下半遞迴繼續砍。階梯結構讓「找上半隊長」與「找下半隊長」這兩個子問題的欄位範圍互不重疊，所以每層遞迴所有子問題加起來掃的總欄位數不超過 M。

Base case：rowLo > rowHi 直接 return（沒有列要處理）。因為遞迴是「中間 → 上半 → 下半」的順序，而不是 row 1..N，所以 Report 要等遞迴全部結束，再依 answer[] 順序輸出。

06

為什麼這樣切是對的// Correctness

// CORRECTNESS · 階梯結構保證範圍切得乾淨
  1. 掃完中間列得到隊長位置 m（這列的答案就鎖定了）。
  2. 上半列（i < mid）的隊長一定在 m 左邊 ⇒ 上半之後只能在 cols [colLo, m−1] 找。m 本身已經被中間列搶走，不會再是上半的答案。
  3. 下半列（i > mid）的隊長一定在 m 右邊 ⇒ 下半之後只能在 cols [m+1, colHi] 找。
  4. 每次遞迴把「列數對半切」（mid 切上下兩半），所以最多切 O(log N) 層就見底。
  5. 邊界情況：若 m == colLo，上半的欄位範圍變成空 — dnc 入口的 colLo > colHi 守備會自動 return，不會有問題。m == colHi 同理（下半變空）。

換句話說：「列峰嚴格遞增」就像一條從左上往右下走的階梯 — 知道中間那階站在哪，就立刻把整個搜尋空間切成「上半（只能在階梯左邊）」與「下半（只能在階梯右邊）」兩塊互不重疊的小區。

07

複雜度// Complexity

每一層遞迴的所有子問題加起來掃過的欄位總長 ≤ M — 因為相鄰子問題的欄位範圍互不重疊（端點 m 已被中間 row 取走，上下半從 m−1 / m+1 起算）。

T(N, M) ≤ 2 · T(N/2, ?) + O(M_layer)
每層欄位總長 = O(M)，共 O(log N) 層 ⇒ Total = O((N + M) log N)
項目數量備註
TimeO((N + M) log N)遞迴每層的欄位總長 ≤ M（子問題互不重疊）
SpaceO(N)answer[1..N] 暫存；遞迴堆疊 O(log N)
Recursion DepthO(log N)N = 10⁶ ⇒ 約 20 層
GetVal Calls≤ (N + M) log₂ NN = M = 10⁶ ⇒ ≈ 2×10⁶ × 20 = 4 × 10⁷，遠在 10⁸ 上限內

GetVal 預算才是這題的真正限制 — 不是時間複雜度。10⁸ 的限制意味著就算單列 brute O(M) 也只能用 100 次（N ≤ 100 的子任務），全題必須靠遞迴縮減欄位範圍。

08

一句洞察// One-Sentence Insight

「列峰嚴格遞增」這條結構性條件，把每層遞迴的欄位總長壓在 O(M)。
所以 D&C on rows 不是「砍列數」，而是用列峰位置把欄位範圍切開 — 砍的是欄位，省的是 GetVal。
