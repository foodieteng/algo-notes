01

完整實作 · 一維 0/1 背包// Full C++

一維 dp[]。容量上界取 cap = min(M, ΣAᵢ)。外層逐物品、內層 w 由 cap 倒掃到 Aᵢ。 答案是 dp[cap]。

p143.cpp C++17 
#include <bits/stdc++.h>
using namespace std;

const int MAXM = 1000005;
long long dp[MAXM];        // dp[w] = 飽足感 ≤ w 時的最大滿足感
int satiety[105];         // satiety[i] = 第 i 個的飽足感 Aᵢ（重量）
int joy[105];             // joy[i]     = 第 i 個的滿足感 Bᵢ（價值）

void solve() {
    int N;
    long long M;
    cin >> N >> M;

    long long sumSat = 0;
    for (int i = 0; i < N; i++) {
        cin >> satiety[i] >> joy[i];
        sumSat += satiety[i];
    }
    long long cap = min(M, sumSat);      // 容量超過總重量沒意義

    for (long long w = 0; w <= cap; w++) dp[w] = 0;

    for (int i = 0; i < N; i++) {
        // w 由大到小 ⇒ dp[w − satiety[i]] 仍是「沒放第 i 個」的值 ⇒ 每個只拿一次
        for (long long w = cap; w >= satiety[i]; w--)
            dp[w] = max(dp[w], dp[w - satiety[i]] + joy[i]);
    }

    cout << dp[cap] << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) solve();

    return 0;
}

cap=min(M,ΣAᵢ) 的好處：當所有高棕櫚的飽足感加起來都 < M 時，胃根本撐不滿，dp 表只需開到 ΣAᵢ，省時又省記憶體。

02

複雜度分析// Complexity Derivation

步驟內容成本
初始化 dp歸零 cap+1 格O(M)
外層物品i = 0..N−1O(N)
內層容量倒掃w = cap..AᵢO(M)
每格轉移一次 maxO(1)
單筆合計—O(N·M)
N 物品 × cap 容量 = O(N · min(M, ΣAᵢ))
最壞 N=100、M=10⁶ ⇒ 10⁸，但每格只一次 max，實測 < 0.5s
項目數量備註
TimeO(N·M) 每筆兩層迴圈
SpaceO(M)一維 dp[]（二維會 MLE）
遞迴深度0（迭代）純填表
Constant factor低每格一次比較取大
03

邊界與陷阱// Pitfalls

陷阱說明 / 解法
容量方向 內層 w 必須由大到小。寫成由小到大會變「完全背包」，每個物品被重複拿 ⇒ 答案偏大、錯。
二維會 MLE dp[100][10⁶] = 10⁸ 個 long long ≈ 800MB，爆記憶體。一定要壓成一維滾動。
容量上界 開 dp[M] 沒錯，但用 cap=min(M,ΣAᵢ) 當上界更省；別忘了答案輸出 dp[cap] 不是 dp[M]。
每筆重置 dp 多筆測資要重新歸零 dp[0..cap]；只清到 cap 即可（前一筆若 cap 更大要小心殘值，保險可清到上一輪用過的範圍）。
型別 M 可達 10⁶ 用 long long 當索引安全；滿足感總和 ≤ N·100 = 10⁴，int 也夠，但統一 long long 省心。
多筆 T 第一行是 T，之後每筆 N M + N 行。別漏讀 T 或把第一筆的 N 當成 T。
04

細節走讀// Section Walkthrough

① 狀態定義

// state
dp[w] = 在飽足感(重量) ≤ w 的限制下，能拿到的最大滿足感(價值)
初值 dp[w] = 0（什麼都沒吃）
答案 = dp[cap]，cap = min(M, ΣAᵢ)

② 轉移（拿 or 不拿第 i 個）

// transition
for i in 0..N-1:
  for w from cap down to A[i]:        // 倒序！
    dp[w] = max( dp[w],               // 不吃第 i 個
                 dp[w - A[i]] + B[i] )// 吃第 i 個

③ 為什麼倒序

// 0/1 vs 完全
倒序：dp[w-A[i]] 還沒被本輪改 → 是「沒放第 i 個」的值 → 只拿一次（0/1）✓
正序：dp[w-A[i]] 本輪已被改 → 可能含第 i 個 → 重複拿（完全背包）✗

ONE-LINE SUMMARY
一維 0/1 背包：dp[w]=max(dp[w], dp[w−Aᵢ]+Bᵢ)，w 倒掃，cap=min(M,ΣAᵢ)，輸出 dp[cap]。
倒序保證每物只拿一次；二維會 MLE；多筆記得重置 dp；long long 防索引溢位。
