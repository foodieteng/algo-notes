01

重點回顧// Concept Recap

關鍵點內容
問題轉譯 飽足感 = 重量、滿足感 = 價值、胃容量 M = 背包 ⇒ 0/1 背包。
狀態 dp[w] = 飽足感 ≤ w 時的最大滿足感。
轉移 dp[w] = max(dp[w], dp[w−Aᵢ]+Bᵢ)，w 由大到小。
倒序原因 確保 dp[w−Aᵢ] 是「沒放第 i 個」的值 ⇒ 每物只拿一次。
容量上界 cap = min(M, ΣAᵢ)，答案 dp[cap]。
複雜度 O(N·M) 時間、O(M) 空間（一維滾動）。

Q2 倒序 vs 正序

內層 w 由大到小掃和由小到大掃，結果差在哪？為什麼正序會把一個物品重複拿？

Q3 壓維

二維 dp[i][w] 為什麼能壓成一維 dp[w]？它只依賴上一列的哪些格子？

Q4 容量上界

為什麼可以用 cap = min(M, ΣAᵢ) 取代 M？什麼情況下這個優化省很多？

Q5 轉移意義

轉移式裡 dp[w]（保留）與 dp[w−Aᵢ]+Bᵢ（更新）分別代表什麼決策？

03

範例 Trace · M=5, 物品 (2,4)(3,3)// Worked Trace

cap = min(5, 2+3) = 5。dp 一條長度 6（索引 0..5），逐物品 w 由 5 倒掃：

// INPUT2 5
2 4         // 物品0：飽足 A=2、滿足 B=4
3 3         // 物品1：飽足 A=3、滿足 B=3
// init
索引 w :  0   1   2   3   4   5
dp[w] :  0   0   0   0   0   0
// 物品0 (A=2,B=4)  w: 5→2
w=5: max(dp[5], dp[3]+4)=max(0,4)=4
w=4: max(dp[4], dp[2]+4)=max(0,4)=4
w=3: max(dp[3], dp[1]+4)=max(0,4)=4
w=2: max(dp[2], dp[0]+4)=max(0,4)=4
dp :  0   0   4   4   4   4
// 物品1 (A=3,B=3)  w: 5→3
w=5: max(dp[5], dp[2]+3)=max(4, 4+3)=7   ← 兩個都吃！
w=4: max(dp[4], dp[1]+3)=max(4, 0+3)=4
w=3: max(dp[3], dp[0]+3)=max(4, 0+3)=4
dp :  0   0   4   4   4   7
// OUTPUTdp[cap=5] = 7
觀察：w=5 那格用 dp[2]+3，而 dp[2]=4 是「只吃物品0」的值（飽足 2、滿足 4）； 再吃物品1（飽足 +3 剛好到 5、滿足 +3）⇒ 4+3=7。倒序讓 dp[2] 還停在「沒吃物品1」的狀態，物品1 才沒被重複加。
04

應用題// Quick Calculations

App 1 第二筆

範例第二筆 M=4、同樣物品 (2,4)(3,3)，答案為何是 4 而非 7？手動跑一次 dp。

App 2 完全背包

若每個高棕櫚可以無限吃（吃幾次都行），程式要怎麼改？只動一個地方。

App 3 小 A 限制

30% 測資保證 Aᵢ ≤ 100。這對複雜度或解法有什麼啟發（部分分怎麼拿）？

Q1 · 建模

背包容量 = 胃容量 M（飽足感上限）；物品重量 = 飽足感 Aᵢ；物品價值 = 滿足感 Bᵢ。 每個高棕櫚只能吃一次 ⇒ 每個物品拿/不拿，正是 0/1 背包。

Q2 · 倒序 vs 正序

倒序（w 大→小）：更新 dp[w] 時 dp[w−Aᵢ]（較小索引）本輪還沒被改 ⇒ 是「沒放第 i 個」的值 ⇒ 物品只被拿一次（0/1）。

正序（w 小→大）：dp[w−Aᵢ] 本輪已先被更新、可能已含第 i 個 ⇒ 再拿一次第 i 個 ⇒ 同一物品重複計入，變成「完全背包」，這題會答錯。

Q3 · 壓維

二維 dp[i][w] = max(dp[i−1][w], dp[i−1][w−Aᵢ]+Bᵢ) 只依賴上一列 i−1 的兩個格子。 既然只用前一列，就不必保留整張表，用一條 dp[w] 原地更新即可，空間從 O(N·M) 降到 O(M)。

原地更新的代價是必須倒序，才能讓被讀的 dp[w−Aᵢ] 維持「上一列」的語意。

Q4 · 容量上界

全部高棕櫚的飽足感加起來才 ΣAᵢ，胃容量再大也吃不超過這個總和 ⇒ 容量超過 ΣAᵢ 的部分永遠用不到， 答案 dp[min(M,ΣAᵢ)] 等於 dp[M]。

當 ΣAᵢ ≪ M（東西不多但 M 開很大）時，dp 表從 M 縮到 ΣAᵢ，時間與記憶體都大省。本題 ΣAᵢ ≤ 100×10⁵ = 10⁷，但 M ≤ 10⁶，故 cap ≤ 10⁶。

Q5 · 轉移意義

dp[w]（等號右邊的舊值）= 不吃第 i 個高棕櫚，沿用上一輪在容量 w 下的最佳滿足感。

dp[w−Aᵢ]+Bᵢ = 吃第 i 個：先在剩餘容量 w−Aᵢ 拿到最佳滿足感，再加上這顆的 Bᵢ。兩者取大 = 這格的最佳決策。

App 1 · 第二筆 M=4 
// cap = min(4, 5) = 4
init:            dp = [0,0,0,0,0]
物品0(A=2,B=4):  w=4..2 → dp = [0,0,4,4,4]
物品1(A=3,B=3):  w=4: max(dp[4],dp[1]+3)=max(4,3)=4
                 w=3: max(dp[3],dp[0]+3)=max(4,3)=4
                 dp = [0,0,4,4,4]
答案 dp[4] = 4

容量只有 4，吃不下「飽足 2+3=5」的組合，只能擇一 ⇒ 取較大的滿足感 max(4,3)=4。

App 2 · 完全背包

把內層迴圈方向從倒序改成正序（for w = Aᵢ to cap）即可。 正序時 dp[w−Aᵢ] 已可能含第 i 個，於是同一物品能被重複拿 ⇒ 正好是「無限吃」的完全背包。其餘程式不變。

App 3 · 小 A 限制

Aᵢ ≤ 100 ⇒ 總飽足 ΣAᵢ ≤ 100×100 = 10⁴，於是 cap = min(M, ΣAᵢ) ≤ 10⁴。 dp 表只需開到 10⁴、複雜度降到 O(N·ΣAᵢ) ≤ 100×10⁴ = 10⁶，輕鬆過。

啟發：容量上界用 min(M, ΣAᵢ) 的寫法天生就吃下這個部分分——同一份程式在 Aᵢ 小時自動變快，不必特判。
