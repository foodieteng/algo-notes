最近點對

// Closest Pair of Points · 平面分治經典

★ ★ ★ ★ ★ 核心 Divide & Conquer · Strip Trick · 時間 O(N log² N)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

平面上 N ≤ 2 × 10⁵ 個整數座標的點 (|x|, |y| ≤ 10⁹)。 請找出距離最近的兩個點，輸出最小距離平方 d²。 用平方避開浮點誤差。

之所以採距離平方，是因為兩個距離 a < b 的點，也滿足 a² < b²，大小關係不變， 不必開根號即可比較。整題能用純整數運算 (long long) 完成。

02

為什麼用分治// Why D&C

A暴力 O(N²)
枚舉所有 C(N, 2) 對 + O(1) 算距離。N = 2 × 10⁵ ⇒ ~2 × 10¹⁰ 運算，TLE。
BHash / 格子
期望 O(N) 但 worst-case 退化；浮點切格子又有精度風險，需動態調整。工程麻煩。
C分治 O(N log² N)
按 x 切半 → 遞迴左右 → 合併時只看「中線附近 strip」內每點下 7 個鄰居。穩定且純整數。
03

核心策略 · 三步驟// Algorithm Outline

對所有點先按 x 預排序。遞迴函式 closest(lo, hi) 做這 3 件事：

  1. 切一半：取中位索引 mid = (lo + hi) / 2，記下中線 mid_x = pts[mid].x。
  2. 遞迴左右：d_L = closest(lo, mid)、d_R = closest(mid+1, hi)，取 d = min(d_L, d_R)。
  3. 合併 strip：收集所有 |x − mid_x| < √d 的點，按 y 排序後每點只跟後續 7 個比，更新 d。

Base case：n ≤ 3 時直接暴力。最多比 3 對，遞迴下去 mid 也會落在唯一一點上、不便切。

06

為什麼只比 7 個鄰居就夠// Pigeonhole Bound

令 d 為遞迴左右後的最小距離。strip 內任兩點若要 < d，那它們的 y 軸差也必 < d。 考慮在 strip 任意「d 高度視窗」內最多能塞幾個點：

Pigeonhole · 鴿籠論證
  1. 把 2d × d 區域切成 4 個 (d/2) × (d/2) 子格（左 2 個、右 2 個）。
  2. 每個子格內兩點對角距離為 d / √2 ≈ 0.71 d < d。
  3. 但左半已遞迴出 d_L ≥ d ⇒ 左半任兩點距離 ≥ d ⇒ 左半 2 個子格各最多 1 點 ⇒ 左半最多 2 點。
  4. 右半同理最多 2 點 ⇒ 整個 2d × d 區域最多 4 點。
  5. 擴成 2d × 2d 視窗（兩矩形上下疊）⇒ 最多 8 點（含自己）。

所以按 y 排序的 strip 中，每點之後至多有 7 個 鄰居會跟它構成 < d 的距離 — 檢查這 7 個即可。實作多半再加 if (Δy² ≥ d) break; 提前終止。

07

複雜度// Complexity

T(N) = 2 · T(N/2) + O(N log N)
合併階段需要對 strip 排序 ⇒ O(N log N)
總複雜度 = O(N log² N)

若改用 merge sort 在遞迴中同步維護「y 已排序」的副本，可降到 O(N log N)。 多數教學版採 O(N log² N) — 程式更直觀，N = 2 × 10⁵ 仍秒過。

08

一句洞察// One-Sentence Insight

D&C 解這題的價值不在「切半」，而在「合併」： 左右遞迴得到 d 後，畫一條寬 2d 的 strip 就把「可能更近的跨界 pair」候選縮到 O(N)； strip 按 y 排序、每點只看 7 個鄰居又把比對縮到 O(N)。 整題的關鍵是幾何 + pigeonhole，而不是遞迴本身。
