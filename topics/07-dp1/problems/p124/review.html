重點回顧// Concept Recap

關鍵點內容
問題本質 3×N 用 1×2 骨牌鋪滿的方法數（domino tiling）。
奇偶 3N 為奇數（N 奇）時鋪不滿 ⇒ 答案 0。
狀態 f[n] 平邊界、g[n] 缺一角（凹）邊界。
遞推 f[n]=f[n−2]+2g[n−1]、g[n]=f[n−1]+g[n−2]。
單一式 消去 g ⇒ f[n]=4f[n−2]−f[n−4]（減法補 MOD）。
多筆 一次預處理 f[1..maxN]，查表 O(1)；mod 1000007。

觀念題// Concept Questions

Q1 奇偶

為什麼 N 是奇數時答案一定是 0？用格子數說明。

Q2 狀態意義

g[n]「缺一角」是什麼意思？為什麼只需要「平 / 缺一角」兩種狀態、不會有更多型態？

Q3 轉移 ×2

f[n]=f[n−2]+2·g[n−1] 裡的係數 2 從哪來？

Q4 單一遞迴

單一式 f[n]=4f[n−2]−f[n−4] 比雙狀態版方便在哪？又多了什麼要小心？

Q5 f[0]=1

為什麼 base 要設 f[0]=1（空走廊算 1 種）而不是 0？

03

範例 Trace · 雙狀態推到 f[4]=11// Worked Trace

用 f[n]=f[n−2]+2g[n−1]、g[n]=f[n−1]+g[n−2] 一路推（base 如下）：

// base
f[0]=1  f[1]=0
g[0]=0  g[1]=1
// n = 2
f[2] = f[0] + 2·g[1] = 1 + 2·1 = 3      ← 範例 N=2 → 3 ✓
g[2] = f[1] + g[0]   = 0 + 0   = 0
// n = 3
f[3] = f[1] + 2·g[2] = 0 + 0 = 0        ← 奇數 → 0 ✓
g[3] = f[2] + g[1]   = 3 + 1 = 4
// n = 4
f[4] = f[2] + 2·g[3] = 3 + 2·4 = 11     ← 範例 N=4 → 11 ✓
g[4] = f[3] + g[2]   = 0 + 0   = 0
// 對照單一遞迴
f[4] = 4·f[2] − f[0] = 4·3 − 1 = 11     ← 同樣得 11 ✓
// OUTPUT (若 N=4)11
觀察：g 在偶數 n 都是 0、奇數 n 才有值（缺一角會破壞奇偶平衡）。 兩條鏈交錯餵養：偶數的 f 靠奇數的 g、奇數的 g 靠偶數的 f。 消去 g 後就只剩偶數項的 f[n]=4f[n−2]−f[n−4]。

應用題// Quick Calculations

App 1 2×N

若走廊寬度改成 2（2×N），鋪法數是什麼數列？遞迴長怎樣？

App 2 一般寬度

寬度 m（m×N）一般情形怎麼做？為什麼 m 大時要用「broken profile / 輪廓 DP」？

App 3 巨大 N

若 N 可達 10¹⁸（但只問一個 N），O(N) 預處理就不行了，怎麼辦？

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

Q1 · 奇偶

3×N 共 3N 格。每塊 1×2 骨牌恰蓋 2 格，鋪滿需要總格數是偶數。 3N 為偶 ⇔ N 為偶。所以 N 奇數時 3N 是奇數，無論怎麼放都剩一格 ⇒ 0 種。

Q2 · 狀態意義

逐欄往右鋪時，已鋪滿區域的「右邊界輪廓」決定後面怎麼接。寬度只有 3， 邊界要嘛三格都齊（平），要嘛最後一欄缺了頂端或底端一格（缺一角、凹進去）。

「缺中間一格」或「缺兩格」其實都能歸到這兩類（上下對稱、或等價於更早的狀態）， 所以兩個狀態 f / g 就足以描述所有邊界，遞迴才會封閉。

Q3 · 轉移 ×2

2·g[n−1] 的 2 來自上下對稱：缺角可以缺在「上面那格」或「下面那格」， 兩種鏡像情形數量相同，各貢獻一份 ⇒ 乘 2。

Q4 · 單一遞迴

方便：只需一條陣列 f[]，不用同時維護 g，程式更短。

要小心：式子有減法（−f[n−4]），取餘後可能變負 ⇒ 一定要 (x % MOD + MOD) % MOD。雙狀態 f/g 版全是加法、反而不會踩這個雷。

Q5 · f[0]=1

f[0] 代表「長度 0 的空走廊」的鋪法數。空走廊有唯一一種「鋪法」——什麼都不放 —— 所以是 1。這個 1 是遞迴的乘法/加法基底：例如 f[2]=f[0]+2g[1]=1+2=3 就靠它。

若誤設 f[0]=0，f[2] 會變 2、整條鏈全錯。

App 1 · 2×N

2×N 用 1×2 骨牌的鋪法數是費氏數列：1, 1, 2, 3, 5, 8, …， 遞迴 a[n] = a[n−1] + a[n−2]（最右一欄放一個直的 ⇒ a[n−1]；放兩個橫的 ⇒ a[n−2]）。 比 3×N 簡單，因為邊界只有「平」一種、不需要 g。

App 2 · 一般寬度

寬度 m 一般情形用輪廓 DP（broken profile / bitmask DP）： 狀態是「目前這一欄的填充情形」用 m 位元的 bitmask 表示， 逐格決定放橫 / 直骨牌，轉移到下一個 bitmask。

複雜度約 O(N · 2^m)（或 O(N · 4^m) 視轉移寫法）。 m 小（如 3）可手推成上面的封閉遞迴；m 大就一定要 bitmask DP，因為邊界型態變成指數多種。

App 3 · 巨大 N

遞迴 f[n]=4f[n−2]−f[n−4] 是線性常係數遞迴 ⇒ 可用矩陣快速冪 把狀態向量 (f[n], f[n−2])（或 f/g 向量）寫成轉移矩陣的冪次。 

// matrix power
把 (f[n], f[n-2]) = M · (f[n-2], f[n-4]) 寫成 2×2 矩陣 M
求 M^(N/2) ⇒ O(2³ · log N) = O(log N)
可處理 N 高達 10^18（仍對 1000007 取餘）
