陣線推進

// Lexicographically Smallest Topological Order

★ ★ ★ ☆ ☆ 核心 拓樸排序 · Kahn + min-heap · 時間 O((n + m) log n)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

n 個陣地（編號 0 ~ n−1），m 條依賴關係。每條 a b 代表 攻破 b 之前必須先攻破 a。請輸出一個合法的攻破順序，且要是字典序最小的那一個； 若根本不存在合法順序（出現循環依賴），輸出 QAQ。

1 ≤ n ≤ 10⁵ · 0 ≤ m ≤ 3×10⁵，共 T ≤ 5 組測資。 「a b」就是一條有向邊 a → b（a 是 b 的前置）— 這整題的骨架就是 有向圖的拓樸排序（topological sort）。

02

為什麼用 min-heap Kahn// Why min-heap Kahn

A 枚舉全排列 O(n!)
試每個排列、檢查合法又比字典序。n = 10⁵ 直接爆炸。
× REJECT · TLE
B DFS 拓樸排序 O(n + m)
後序反轉雖能拓樸，但無法保證字典序最小。
△ REJECT · WRONG ORDER
BOTH FAIL · ADOPT
C Kahn + min-heap O((n+m) log n)
用 priority_queue 維護「目前可選」的集合，每次貪心取編號最小。順手判循環。
● ADOPT · OPTIMAL

普通 Kahn 用一般 queue 只保證「某個」合法序；把 queue 換成 min-heap（小根堆）， 就把它升級成「字典序最小」的合法序。改動只有一行，這就是本題的全部精華。

03

核心策略 · 三步驟// Algorithm Outline

建好「鄰接表 adj」與「入度 indeg」後：

  1. 初始化堆 · 把所有 indeg == 0 的陣地（沒有前置、現在就能打）丟進 min-heap。
  2. 反覆取最小 · 從堆取出編號最小的 u，加入答案序列；對每條 u → w 做 --indeg[w]，若降到 0 就把 w 推進堆。
  3. 判循環 · 堆空後若已輸出 n 個 ⇒ 合法；否則剩下的點卡在循環裡 ⇒ 輸出 QAQ。

關鍵 · 「貪心取目前可選的最小」不是「取全場最小」— 一個小編號若還有前置沒打完， 就得乖乖等。下面的動畫示範這個「被迫延後」的時刻。

05

為何「每次取最小」就是字典序最小// Greedy Correctness

直覺上好像理所當然，但要證明「貪心 = 全域最佳」需要一個交換論證 （exchange argument）。重點是：第一個位置一定要放「目前 indeg=0 中最小的那個」， 之後每一格遞迴同理。

交換論證 · 為何第一格非最小不可

設 v = 目前所有 indeg=0 的點裡編號最小者。考慮任何一個合法拓樸序 S：

  1. v 沒有任何前置（indeg=0），所以放在序列最前面不會違反任何依賴。
  2. 若 S 的第一個元素 S₁ ≠ v，那 v 一定出現在 S 的後面某處。 把 v 抽出來、移到最前面 ⇒ 因為 v 無前置，移動後仍是合法序。
  3. 新序列第一格變成 v < S₁ ⇒ 字典序更小。 ⇒ 任何「第一格不是 v」的序，都能被改造得更小 ⇒ 最佳序的第一格必是 v。
  4. 固定第一格為 v 後，把 v 移除、更新入度，剩下的子問題結構完全一樣 ⇒ 第二格同理取「現在可選的最小」⇒ 一路歸納到底。

⇒ 「每步取堆頂最小」逐格都是最優選擇 ⇒ 整體就是字典序最小的拓樸序。

常見誤區：以為「先把答案排序」或「DFS 後序反轉」能拿到字典序最小 — 都不行。 字典序由最前面的元素主導，必須在每個前綴都貪心取最小可選，這正是 min-heap 在做的事。

06

複雜度// Complexity

每個點進、出堆各一次 ⇒ O(n log n)；每條邊在「鬆弛入度」時被掃一次 ⇒ O(m) （配合可能的入堆是 O(m log n)）。合起來：

T = O((n + m) log n)
n = 10⁵、m = 3×10⁵ ⇒ 約 4×10⁵ × 17 ≈ 7×10⁶ 次操作，每組測資都秒過
項目數量備註
TimeO((n+m) log n)log 來自 priority_queue 的進出
SpaceO(n + m)鄰接表 + 入度陣列 + 堆
多筆測資× T (≤ 5)每組獨立重建圖，記得清空
Constant factor中堆操作常數略高，但量級內輕鬆
