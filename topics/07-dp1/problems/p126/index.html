// 計數 DP · 相鄰限制塗色

★ ★ ★ ☆ ☆ 核心 線性 DP（末位顏色）+ 固定首色 · 時間 O(maxN + T)
PAGE 1 / 3  ·  概念
01

題目簡介// Brief

QUESTION

把 N 個零錢排成一列，每個塗紅 / 綠 / 藍其中一色。限制： 藍色和綠色不能相鄰，而且頭尾也不能一藍一綠。 問共有幾種塗色方法？答案對 1000007 取餘數。多筆 T ≤ 10⁵，1 ≤ N ≤ 10⁵。

範例：N=1 → 3、N=2 → 7、N=3 → 15、N=100000 → 918955。 「頭尾不能一藍一綠」等於把 B/G 的相鄰限制繞成環。

02

為什麼用 DP// Why DP

A 暴力枚舉 O(3ᴺ)
列出每種塗法再檢查 ⇒ N=100000 根本天文數字，連 N=30 都跑不完。
× REJECT · TLE
B 每筆各自 DP O(T · N)
每筆測資重跑一次 O(N) DP ⇒ T·N 最壞 10⁵×10⁵ = 10¹⁰，仍 TLE。
△ REJECT · TLE
BOTH FAIL · ADOPT
C 線性 DP + 預處理 O(maxN + T)
先用末位顏色處理相鄰，再固定首色補頭尾；一次把 ans[1..10⁵] 全算好。
● ADOPT · STABLE

關鍵觀察：一般相鄰是否合法，只跟前一個 coin 的顏色有關（綠藍不相鄰）。 頭尾那一條額外限制則靠固定第一個顏色補回。多筆測資 ⇒ 一次預處理所有 N，查表回答。

03

狀態設計 · 末位顏色// State Design

合法只看前一個 coin 的顏色（綠藍不相鄰）⇒ 用「末位顏色」當狀態的線性 DP：

r[i] = r[i−1] + g[i−1] + b[i−1]
g[i] = r[i−1] + g[i−1]
b[i] = r[i−1] + b[i−1]
R 可接任何色；G 不可接 B；B 不可接 G。base：r[1]=g[1]=b[1]=1

但這只擋了「相鄰」，還沒擋「頭尾不一藍一綠」。乾淨的處理是固定第一個 coin 的顏色 first， 對每個 first 各跑一次上面的 DP，最後加總時只收「末位 last 跟 first 不形成一藍一綠」的那些：

ans[N] = Σfirst　Σlast：ok(first,last)　dpfirst[N][last]
ok(x,y) = ¬( {x,y} = {G,B} )；即只排除 (G,B) 與 (B,G) 這兩種頭尾
  1. 跑 3 次 · first = R / G / B 各跑一次末位 DP。
  2. 頭尾過濾 · 末位用 ok(first,last) 取捨，直接加總 ⇒ 不用再做減法。
  3. 等價捷徑 · 也可「線性總數 − 頭尾恰為 {G,B} 的數量」，或用三項遞迴 a(N)=3a(N−1)−a(N−2)−a(N−3)（見 §05）。

05

頭尾修正 · 把環的限制補回來// Endpoint Fix

// 做法 · 固定第一個顏色，再依頭尾規則取捨末位
  1. 線性 DP 只保證「相鄰不違規」，但沒擋頭尾一藍一綠。
  2. 把 DP 改成記住第一個 coin 的顏色：dp[首色][末色]。
  3. 最後加總時，跳過 {首,末} = {G,B} 的兩種組合（G…B 與 B…G）。
  4. 等價的簡單算法：答案 = 線性總數 L(N) − (頭尾恰為 {G,B} 的數量)。
  5. 本題更漂亮：答案序列滿足三項遞迴（見下），一次預處理全部 N。
// endpoint rule · ok(first, last)
固定 first 後，最後一格 last 能不能計入答案：
  last = R：一定可以
  last = G：first 不能是 B
  last = B：first 不能是 G
也就是只排除 (first,last) = (G,B) 和 (B,G)。

N = 3 直接套（這就是 code 在做的事）：每個 first 跑到 n=3 的末位 (R,G,B) 分佈，再用 ok() 過濾加總：

// N = 3 · 三個 first 各自貢獻
first=R：末位(R,G,B)=(3,2,2) → 三色全收 → 3+2+2 = 7
first=G：末位(R,G,B)=(2,2,1) → 丟末位 B   → 2+2   = 4
first=B：末位(R,G,B)=(2,1,2) → 丟末位 G   → 2+2   = 4
ans[3] = 7 + 4 + 4 = 15  ✓（不用任何減法）

若把「固定首色」的狀態遞推整理掉中間變數，答案數列 a(N) = 3, 7, 15, 35, 83, … 還有一條極簡式：

a(N) = 3·a(N−1) − a(N−2) − a(N−3)
base a(1)=3, a(2)=7, a(3)=15；全部對 1000007 取餘。一次跑到 10⁵ 即可查表。

（可由固定首色 DP / 轉移矩陣推出；用它預處理最省事，O(maxN) 一次填滿 ans[]。）

07

一句洞察// One-Sentence Insight

相鄰限制看上一格，頭尾限制看第一格。
所以用「末位顏色 DP」處理相鄰，再用「固定首色 + 末位取捨」補回頭尾；
多筆測資就一次預處理所有 N，查表 O(1)。
