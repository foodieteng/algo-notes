糟糕陣列

// Bad Array · XOR 1-因子分解

★ ★ ★ ★ ☆ 核心 構造 · 組合數學 · N = 2^K ≤ 1024 · 時間 O(N²) · 空間 O(1)
PAGE 1 / 3  ·  CONCEPT
01

題目簡介// Brief

QUESTION

給 N = 2^K（N ≤ 1024），構造一個 N × N 陣列 a，每格填 1..2N−1 之間的整數（可重複），且滿足： 對每個 i，第 i 列（row i）和第 i 行（col i）合起來的值，聯集恰好是 {1, 2, ..., 2N−1}。

互動式：實作 solve(N)，由 judge 提供 void Report(int)， 依 row-major 順序呼叫 N² 次。答案不唯一。

先數一個關鍵的數字：row i 有 N 格、col i 有 N 格，但它們共用對角線 (i, i) 那一格， 所以合起來是 N + N − 1 = 2N − 1 格。而 {1..2N−1} 剛好有 2N − 1 個數 — 格數 = 數量，代表這個十字裡每個數恰好出現一次、不重不漏。

把「row i + col i」這個十字形叫做 index i 的十字 (cross)。 整題就是一句話：每個十字都要把 1..2N−1 不重不漏地放滿。

A 暴力枚舉所有陣列 O((2N−1)^(N²))
指數爆。N = 1024 ⇒ (2047)^(10⁶) — 完全不可行。
× REJECT · INFEASIBLE
B 一般偶數的圓圈法 round-robin
1-因子分解對任何偶數都存在，但 round-robin 構造要轉指標、繞圈，程式碼瑣碎易錯。
△ REJECT · MESSY

● ADOPT · O(1) 空間
03

核心推導 · 四步化約// Derivation Chain

  1. 對角線全放 N。對角格 (i, i) 同時屬於 row i 和 col i，是十字的中心。所有對角線放同一個常數 N（正中間最對稱），剩下 2N−2 格要放 {1..N−1} ∪ {N+1..2N−1}。
  2. 剩下的數兩兩配成「和為 2N」的互補對：{1, 2N−1}, {2, 2N−2}, …, {N−1, N+1}，共 N−1 組。目標變成：讓 (i, c) 和 (c, i) 這兩格剛好放某一組互補對，即 a[i][c] + a[c][i] = 2N。
  3. 化約成圖著色。定義 color(i, c) = 這格屬於第幾組互補對。因為 (i,c) 與 (c,i) 同組 ⇒ color 對稱。十字 i 不重不漏 ⟺ 當 c 跑遍其他 index，color 要把 1..N−1 每個編號各碰一次 ⇒ 這就是 K_N 的 1-因子分解。
  4. N = 2^K ⇒ 用 XOR。令 color(i, j) = i XOR j：固定 i 時 i XOR j 掃過 1..N−1 每個非零值各一次（雙射）、對稱、每色構成完美配對 — 三條件全中，一行解決。
i == j → N  ·  i < j → i ^ j  ·  i > j → 2N − (i ^ j)
用 i 與 j 的大小定方向（小的拿 i^j、大的拿 2N−(i^j)）， a[i][j] + a[j][i] = 2N 自動成立。0-indexed。

05

為什麼 XOR 成立// Why XOR Works

// LEMMA · color(i, j) = i XOR j 是 K_N 的 1-因子分解

把 index 想成「點」、color(i, j) 想成「邊 {i, j} 的顏色」。要證三件事：

  1. 每個點看到全部 N−1 種顏色：固定 i，當 j 跑遍 j ≠ i，i XOR j 是一個雙射（XOR 同一個常數是可逆的），值域是 {1..N−1} 的全部非零值 — 每色各一次。這正是「十字不重不漏」。
  2. 對稱：i XOR j = j XOR i ⇒ (i,j) 與 (j,i) 同色 ⇒ 同屬一組互補對。
  3. 每色是完美配對：顏色 k 的邊把 i 配給 i XOR k。因為 (i XOR k) XOR k = i，這是把每個點兩兩配對、無重疊 — 一個 1-因子（perfect matching）。

三條件全中 ⇒ 顏色 0..N−1（扣掉 0 給對角）把 K_N 的邊分成 N−1 個完美配對 = 1-因子分解。N = 2^K 保證 XOR 在 [0, N) 上封閉，所以一定成立。Q.E.D.

換句話說：XOR 把一個看似要動用組合設計（round-robin / 圓圈法）的構造，壓成一行位元運算。 之後遇到「N 是 2 的次方 + 要做兩兩配對 / 不重複覆蓋」的題目，XOR 1-因子分解都值得第一個試。

06

定方向 · 互補對的兩半// Orientation

每組互補對是 {k, 2N − k}，其中 k = i XOR j。 還要決定 (i, j) 拿 k 還是 2N − k — 用 i 跟 j 的大小當分界：

情況填值說明
i == jN對角中心，所有十字共用
i < ji ^ j互補對較小的一半
i > j2N − (i ^ j)較大的一半，和為 2N

因為 i < j 與 i > j 是對稱兩格、i^j 相同，所以 a[i][j] + a[j][i] = (i^j) + (2N − (i^j)) = 2N 永遠成立 — 互補對的兩半永遠分居對角線兩側。

07

複雜度// Complexity

沒有遞迴、沒有陣列 — 一個雙層迴圈，每格直接算出值並 Report，邊算邊輸出。

每格 O(1) 位元運算 ⇒ Time = O(N²)，Space = O(1)
這已是理論下限：光是輸出就有 N² 個數要產生，不可能比 O(N²) 更快。 和遞迴版同樣 O(N²) 時間，但空間從 O(N²) 降到 O(1)。
項目數量備註
TimeO(N²)每格一次 XOR + 比較；N = 1024 ⇒ ~10⁶ 次
SpaceO(1)不存陣列，邊算邊 Report
Recursion Depth—無遞迴
Report CallsN²row-major 順序輸出

瓶頸是輸出 — N² 個 Report 呼叫無法再省。建構本身被「藏進」每次輸出，零額外開銷。

08

一句洞察// One-Sentence Insight

「每個十字不重不漏」= K_N 的 1-因子分解，而 N = 2^K 讓 i XOR j 免費給出完美配對。
對角線固定成 N，其餘配成「和為 2N」的互補對， XOR 把組合設計壓成一行位元運算 — 這是最該記住的一步。
