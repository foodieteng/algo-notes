INITIAL · 4×4 board · special = (3, 4)

遞迴順序對照

下面是 5 個 L 對應的 tile() 呼叫順序（在實際 code 中，遞迴是 DFS，所以中央 L 會「最先 Report」）：

StepL 名稱覆蓋的格子來自
1L1 · Center(2,2), (2,3), (3,2)tile(1,1,4,3,4) 中央放 L
2L2 · TL(1,1), (1,2), (2,1)tile(1,1,2,2,2) base case
3L3 · TR(1,3), (1,4), (2,4)tile(1,3,2,2,3) base case
4L4 · BL(3,1), (4,1), (4,2)tile(3,1,2,3,2) base case
5L5 · BR(3,3), (4,3), (4,4)tile(3,3,2,3,4) base case
// 觀察

L1 在 TL / TR / BL 三個象限的最角落格上各放一個 cell — 這 3 個 cell 在子問題裡就是該象限的「new special」。 BR 象限因為「原 special (3,4)」就在裡面，所以不需要中央 L 補一個 cell — 直接遞迴用原 special 解。
