02

為什麼分治 — 不是純枚舉// Why D&C

A暴力 O(N²)
枚舉所有 C(N, 2) 對 + O(1) 距離。 N = 2 × 10⁵ 時要算 ~2 × 10¹⁰ 次，TLE。30% 範圍才勉強過。
BHash / 格子
把平面切 d × d 格子，每點只看鄰近格 — 期望 O(N) 但 worst-case (點集中) 退化；需動態調整 d，工程麻煩。
C分治 O(N log² N)
按 x 切半 → 遞迴左右 → 合併時只看「中線附近 strip」內每點下 7 個鄰居。穩定且純整數運算。
03

核心策略 — 三步驟// Algorithm Outline

對 x 預先排序好的 N 個點，遞迴函式 closest(lo, hi) 做三件事：

  1. 切一半：取中位 index mid = (lo + hi) / 2，記下中線 mid_x = pts[mid].x。
  2. 遞迴左右：d_L = closest(lo, mid)、d_R = closest(mid+1, hi)，取 d = min(d_L, d_R)。
  3. 合併 strip：收集所有距中線 |x − mid_x| < √d 的點，按 y 排序後，每個點只跟後續 7 個比，更新 d。

Base case：n ≤ 3 時直接暴力（最多 3 對），避免遞迴深度過深。

04

視覺化 — strip 的形狀// The Strip

遞迴左右算完後得到 d。畫一條寬 2d 的垂直 strip 跨在中線上 — 答案如果是「左半 + 右半」的跨界對，這兩個點一定都落在 strip 裡（否則它們的 Δx 就 ≥ d，d² 不可能更小）。

關鍵：在 strip 之外的點根本不可能跟另一邊的點構成 < d 的距離 — 直接濾掉，省下大量比較。 strip 內的點再按 y 軸排序，後面只要看每個點的「下 7 個」鄰居。
05

為什麼只比 7 個鄰居就夠// Pigeonhole Bound

令 d 為當前已知最小距離。考慮 strip 內任意兩點要 < d，y 軸差也必 < d。 現在計算 strip 任意 d-高度 視窗中最多容納多少點：

PIGEONHOLE
把 2d × d 矩形切成 4 個 (d/2) × (d/2) 子格 (左半 2 個、右半 2 個)。
每個子格內兩點最遠距離為對角線 d√2 / 2 ≈ 0.707 d < d。
但左半已遞迴出 d_L ≥ d，意指左半任兩點距離 ≥ d ⇒ 左半的 2 個子格各最多 1 點 ⇒ 左半最多 2 點。
右半同理最多 2 點 ⇒ 整個 2d × d 矩形最多 4 點。
擴成 2d × 2d (兩個矩形上下疊) ⇒ 最多 8 點 (含自己)。

所以在按 y 排序的 strip 中，strip[i] 之後只要看 strip[i+1], …, strip[i+7] 即可 —— 若有更近的 pair，一定在這 7 個之內。每點 O(1) 鄰居 ⇒ 合併階段 O(n)。

實作時可額外加 if (Δy² ≥ d) break; 提前終止，平均常數更低。

程式碼參考// C++ reference

#include <bits/stdc++.h>
using namespace std;

struct Point { long long x, y; };

// 距離平方 (純整數，避開浮點)
long long dist2(const Point& a, const Point& b) {
    long long dx = a.x - b.x, dy = a.y - b.y;
    return dx * dx + dy * dy;
}

bool by_y(const Point& a, const Point& b) { return a.y < b.y; }

long long closest(vector<Point>& pts, int lo, int hi) {
    int n = hi - lo + 1;
    if (n <= 3) {                              // base case: brute
        long long d = LLONG_MAX;
        for (int i = lo; i < hi; ++i)
            for (int j = i + 1; j <= hi; ++j)
                d = min(d, dist2(pts[i], pts[j]));
        return d;
    }

    int mid = (lo + hi) / 2;
    long long mid_x = pts[mid].x;

    long long dL = closest(pts, lo, mid);
    long long dR = closest(pts, mid + 1, hi);
    long long d  = min(dL, dR);

    // 收集 strip：|x − mid_x|² < d
    vector<Point> strip;
    for (int i = lo; i <= hi; ++i) {
        long long dx = pts[i].x - mid_x;
        if (dx * dx < d) strip.push_back(pts[i]);
    }

    // strip 按 y 排序，每點只看下 7 個 (pigeonhole 保證)
    sort(strip.begin(), strip.end(), by_y);
    for (int i = 0; i < (int)strip.size(); ++i) {
        for (int j = i + 1; j < (int)strip.size() && j - i < 8; ++j) {
            long long dy = strip[j].y - strip[i].y;
            if (dy * dy >= d) break;          // 提前終止
            d = min(d, dist2(strip[i], strip[j]));
        }
    }
    return d;
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);

    int n; cin >> n;
    vector<Point> pts(n);
    for (int i = 0; i < n; ++i) cin >> pts[i].x >> pts[i].y;

    sort(pts.begin(), pts.end(), [](const Point& a, const Point& b){
        return a.x < b.x;
    });

    cout << closest(pts, 0, n - 1) << '\n';
}

重點 Q&A// Quick Review

問題解答 / 重點
為何用 long long？ 座標 |x|, |y| ≤ 10⁹，Δx² 可達 4 × 10¹⁸ — 接近 long long 上限 (9.2 × 10¹⁸)。int 只到 2.1 × 10⁹，乘法溢位。
strip 條件用 dx² < d 而非 |dx| < √d？ 避開 sqrt 浮點誤差，整題只用整數平方比較。dx² < d 等價於 |dx| < √d。
為什麼 base case 設 n ≤ 3？ n = 2 或 3 時 brute 只 1 或 3 對；遞迴下去 mid 會落在唯一點上，左半變空。設 ≤ 3 邏輯簡單也夠快。
如何降到 O(N log N)？ 不要每層 strip 內 sort by y — 改用 merge sort 把「y 排序」也分治處理 (兩個已 y 排序的子陣列 O(n) 合併)。少一個 log。
ONE-LINE SUMMARY
中位線切割 · 寬 2d 的 strip · 內排 y · 候選 7 個常數 · pigeonhole 證明 · O(N log² N)。
closest = min(d_L, d_R, strip 跨界 pair)  ·  整題純整數 (long long)
